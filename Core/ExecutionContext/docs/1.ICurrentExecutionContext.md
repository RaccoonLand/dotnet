# 1. ICurrentExecutionContext

`ICurrentExecutionContext` is the host-independent contract for reading information about the **current
execution scope** — who is acting, in which tenant, and how the operation correlates to upstream work.

- **Package:** `RaccoonLand.Core.ExecutionContext.Abstractions`
- **Namespace:** `RaccoonLand.Core.ExecutionContext.Abstractions`
- **Analogy:** like `ICurrentCultureProvider` in the Localization module, this package ships **only the
  contract** (and a null implementation). A host-specific package (ASP.NET Core, a message consumer, a worker)
  is responsible for populating the context per scope.

## Surface

```csharp
public interface ICurrentExecutionContext
{
    bool IsAvailable { get; }
    string? UserId { get; }
    string? TenantId { get; }
    string? CorrelationId { get; }
}
```

| Member | Description |
| --- | --- |
| `IsAvailable` | `true` when meaningful context values are present for this scope; `false` when no context exists (see [`NullCurrentExecutionContext`](2.NullCurrentExecutionContext.md)). |
| `UserId` | Identifier of the user on whose behalf the operation runs. `null` when unknown or unavailable. |
| `TenantId` | Identifier of the tenant (organization/account) in scope. `null` for single-tenant hosts or when unknown. |
| `CorrelationId` | Trace/correlation id propagated across services and logs. `null` when not supplied by the host. |

## Host independence

The execution scope may originate from:

- an **HTTP request** (user/tenant from claims, correlation from a header),
- a **message consumer** (user/tenant from message metadata, correlation from message id),
- a **background job** (system user, tenant from job payload),
- or **any other host**.

Application code depends only on `ICurrentExecutionContext` — never on `HttpContext` or transport types.

## Typical usage

Inject `ICurrentExecutionContext` where cross-cutting logic needs ambient identity:

```csharp
internal sealed class AuditMiddleware(ICurrentExecutionContext executionContext)
{
    public async Task InvokeAsync(/* … */)
    {
        if (executionContext.IsAvailable)
        {
            var userId = executionContext.UserId;
            var tenantId = executionContext.TenantId;
            // …
        }
    }
}
```

Always check `IsAvailable` (or tolerate `null` identifiers) before assuming context is populated.

## Registration

This abstractions package does **not** register services. A host package registers a scoped (or async-local)
implementation and falls back to [`NullCurrentExecutionContext.Instance`](2.NullCurrentExecutionContext.md)
when no scope is active.

See also: [NullCurrentExecutionContext](2.NullCurrentExecutionContext.md).
