# 2. NullCurrentExecutionContext

`NullCurrentExecutionContext` is the **null object** for [`ICurrentExecutionContext`](1.ICurrentExecutionContext.md).
It is used when no execution scope is active — for example before a host adapter has established context, in
unit tests, or in console tools that run without user/tenant information.

- **Package:** `RaccoonLand.Core.ExecutionContext.Abstractions`
- **Namespace:** `RaccoonLand.Core.ExecutionContext.Abstractions`

## Surface

```csharp
public sealed class NullCurrentExecutionContext : ICurrentExecutionContext
{
    public static NullCurrentExecutionContext Instance { get; }
    public bool IsAvailable => false;
    public string? UserId => null;
    public string? TenantId => null;
    public string? CorrelationId => null;
}
```

| Member | Description |
| --- | --- |
| `Instance` | Singleton null object. Prefer this over `new()` — the constructor is private. |
| `IsAvailable` | Always `false`. |
| `UserId`, `TenantId`, `CorrelationId` | Always `null`. |

## When to use

| Scenario | Usage |
| --- | --- |
| Default DI fallback | Register `NullCurrentExecutionContext.Instance` when no host-specific provider is configured. |
| Tests | Pass `Instance` (or a test double) without spinning up HTTP or messaging infrastructure. |
| Library code | Return or accept `ICurrentExecutionContext` with a safe default when context is optional. |

## Example

```csharp
services.AddSingleton<ICurrentExecutionContext>(
    NullCurrentExecutionContext.Instance);
```

A production host replaces this with a scoped implementation that reads from the current request, message, or
job and sets `IsAvailable = true` with the appropriate identifiers.

## Design notes

- Sealed with a private constructor — exactly one shared instance, no allocation on the hot path.
- Identifiers are `string?` so hosts can use opaque ids (GUID strings, subject ids, slugs) without forcing a
  particular storage type at the abstraction layer.

See also: [ICurrentExecutionContext](1.ICurrentExecutionContext.md).
