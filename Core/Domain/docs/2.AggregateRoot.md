# 2. AggregateRoot

`AggregateRoot<TKey>` extends `Entity<TKey>` and is the consistency boundary of the domain. It also implements the non-generic `IAggregateRoot`, which lets infrastructure discover aggregates without knowing `TKey`.

- Namespace: `RaccoonLand.Core.Domain.Abstractions`
- Package: `RaccoonLand.Core.Domain`

## Responsibilities

1. **Domain events** — collects `DomainEvent` instances to be dispatched/persisted via the Outbox pattern when EF Core saves the aggregate.
2. **Service events** — collects `ServiceEvent` (cross-service integration events), also persisted via the Outbox.
3. **Optimistic concurrency** — owns a `ConcurrencyToken` (GUID) regenerated on each save (by infrastructure through `IAggregateRootMutations`) to detect concurrent modifications.

## Public surface (read-only)

| Member | Description |
|--------|-------------|
| `ConcurrencyToken` | Sequential GUID; configured as an EF Core concurrency token. Regenerated by the save interceptor. |
| `DomainEvents` | Read-only collection of raised domain events. |
| `ServiceEvents` | Read-only collection of raised service events. |
| `RaiseDomainEvent(...)` | `protected` — raise a domain event from within the aggregate. |
| `RaiseServiceEvent(...)` | `protected` — raise a service event from within the aggregate. |

The public surface deliberately does **not** expose any way to clear or remove events, or to regenerate the concurrency token. Those are infrastructure concerns (see below) and business code must never invoke them.

## IAggregateRoot (read-only marker)

The non-generic contract exposes only the read-only surface used by infrastructure to discover and inspect aggregates: `BusinessKey`, `ConcurrencyToken`, `DomainEvents`, `ServiceEvents`. Interceptors, outbox writers, and any other cross-cutting infrastructure depends on this interface rather than the generic base.

## IAggregateRootMutations (internal, infrastructure-only)

The mutating surface lives on the internal interface `IAggregateRootMutations` in the same namespace:

| Member | Description |
|--------|-------------|
| `ClearDomainEvents()` / `ClearServiceEvents()` | Remove every pending event. Only appropriate after successful persistence. |
| `RemoveDomainEvents(ids)` / `RemoveServiceEvents(ids)` | Surgically remove events whose id is in the set (used by the outbox interceptor). |
| `RegenerateConcurrencyToken()` | Called by the save interceptor on modified aggregates. |

- The interface is `internal` and visible only to assemblies granted `InternalsVisibleTo` on `RaccoonLand.Core.Domain` (currently the domain test project and `RaccoonLand.Modules.Persistence.SqlServer.Commands`).
- Every `AggregateRoot<TKey>` implements it via explicit interface implementations, so the methods are **not** callable through an `AggregateRoot<TKey>` or `IAggregateRoot` reference. Infrastructure obtains the surface with `if (entity is IAggregateRootMutations mutations) { ... }`.
- Rationale: prevents application code from accidentally calling `Clear*Events()` on an aggregate before it is persisted, which would silently drop unpublished domain and integration events.

## Example

```csharp
public sealed class Order : AggregateRoot<Guid>
{
    public Order(Guid id) : base(id) { }

    public void Submit()
    {
        // ... state change ...
        RaiseDomainEvent(new OrderSubmitted(BusinessKey));
    }
}

public sealed record OrderSubmitted(Guid OrderBusinessKey) : DomainEvent
{
    public override string EventType => "order.submitted";
}
```

## Notes

- `ConcurrencyToken` must be included in every query (read model) output so updates can round-trip it.
- Events are append-only within a unit of work; infrastructure removes them via `RemoveDomainEvents(ids)` / `RemoveServiceEvents(ids)` after writing to the outbox.
