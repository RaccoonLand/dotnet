# 6. DomainException

`DomainException` is the exception type used to signal one or more business-rule violations.

- Namespace: `RaccoonLand.Core.Domain.Exceptions`
- Package: `RaccoonLand.Core.Domain`

## Purpose

`DomainException` carries one or more [`DomainError`](7.DomainError.md) values. Each error's `Code` is a machine-readable contract for the consumer; each error's `Message` is a localization template key.

This type is host-agnostic: it does not define HTTP status codes or web middleware behavior. Workers, consoles, and APIs can all throw and observe it the same way.

## Constructors

### Single error

```csharp
public DomainException(string code, string message, params object?[] parameters)
```

| Parameter | Required | Description |
|-----------|----------|-------------|
| `code` | yes | Error code for the consumer's programmed reaction. Not null/whitespace. |
| `message` | yes | Localization template key. Used as the fallback user-facing text when no `IMessageLocalization` is registered. Not null/whitespace. |
| `parameters` | no | Positional values inserted into the localized template (never null; defaults to empty). |

Internally this creates one [`DomainError`](7.DomainError.md) and delegates to the multi-error constructor.

### Multiple errors

```csharp
public DomainException(params DomainError[] errors)
```

| Parameter | Required | Description |
|-----------|----------|-------------|
| `errors` | yes | One or more domain errors. At least one is required. The array must not be `null` (`ArgumentNullException`), must not be empty (`ArgumentException`), and **no element may be `null`** (`ArgumentException` — the index of the offending element is included in the message). |

## Surface

| Member | Type | Description |
|--------|------|-------------|
| `Errors` | `IReadOnlyList<DomainError>` | All errors carried by this exception (never empty). |
| `Message` | `string` | All error template keys joined with `"; "` (inherited from `Exception`). |

## Examples

```csharp
// Single error (existing pattern)
if (product is null)
{
    throw new DomainException(
        code: "product.not_found",
        message: "PRODUCT_NOT_FOUND",
        parameters: productId);
}

// Multiple errors
throw new DomainException(
    new DomainError("person.email_invalid", "EMAIL_INVALID"),
    new DomainError("person.mobile_invalid", "MOBILE_INVALID"));
// exception.Message == "EMAIL_INVALID; MOBILE_INVALID"
```

## Host mapping (not part of Domain)

How a host turns `DomainException` into a transport response is an integration concern:

- In the RaccoonLand request pipeline, [`ExceptionHandlingMiddleware`](../../../Modules/Middlewares/ExceptionHandlingMiddleware/docs/1.ExceptionHandlingMiddleware.md) maps every entry in `Errors` to a [`PipelineMessage`](../../RequestProcessing/RequestProcessing.Abstractions/docs/7.PipelineResponse.md) (and localizes when `IMessageLocalization` is registered).
- An ASP.NET host may further map that envelope to an HTTP status (commonly 400 for business-rule failures). Other hosts (workers, consoles) handle or log the exception without HTTP semantics.

## Notes

- `Code` and `Message` on each [`DomainError`](7.DomainError.md) are validated with `ArgumentException.ThrowIfNullOrWhiteSpace`.
- Null elements in the `errors` array are rejected up front with a clear `ArgumentException` — consumers never observe a `NullReferenceException` from downstream message composition.
- Derive from `DomainException` for more specific business errors while keeping the same error-carrying contract.
