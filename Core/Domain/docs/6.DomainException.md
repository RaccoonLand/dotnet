# 6. DomainException

`DomainException` is the exception type used to signal one or more business-rule violations.

- Namespace: `RaccoonLand.Core.Domain.Exceptions`
- Package: `RaccoonLand.Core.Domain`

## Purpose

When a `DomainException` is thrown, the pipeline exception-handling middleware exposes each error's `Code` as a machine-readable contract for the consumer. Each error's `Message`
is treated as a localization template key.

## Constructors

### Single error 

```csharp
public DomainException(string code, string message, params object?[] parameters)
```

| Parameter | Required | Description |
|-----------|----------|-------------|
| `code` | yes | Error code for the consumer's programmed reaction. Not null/whitespace. |
| `message` | yes | Localization template key. Used as the fallback user-facing text when no `IMessageLocalization` is registered. Not null/whitespace. |
| `parameters` | no | Positional values inserted into the localized template (never null; defaults to empty). |

Internally this creates one [`DomainError`](7.DomainError.md) and delegates to the multi-error constructor.

### Multiple errors

```csharp
public DomainException(params DomainError[] errors)
```

| Parameter | Required | Description |
|-----------|----------|-------------|
| `errors` | yes | One or more domain errors. At least one is required. |

## Surface

| Member | Type | Description |
|--------|------|-------------|
| `Errors` | `IReadOnlyList<DomainError>` | All errors carried by this exception (never empty). |
| `Message` | `string` | All error template keys joined with `"; "` (inherited from `Exception`). |

## Examples

```csharp
// Single error (existing pattern)
if (product is null)
{
    throw new DomainException(
        code: "product.not_found",
        message: "PRODUCT_NOT_FOUND",
        parameters: productId);
}

// Multiple errors
throw new DomainException(
    new DomainError("person.email_invalid", "EMAIL_INVALID"),
    new DomainError("person.mobile_invalid", "MOBILE_INVALID"));
// exception.Message == "EMAIL_INVALID; MOBILE_INVALID"
```

## Notes

- `Code` and `Message` on each [`DomainError`](7.DomainError.md) are validated with `ArgumentException.ThrowIfNullOrWhiteSpace`.
- [`ExceptionHandlingMiddleware`](../../../Modules/Middlewares/ExceptionHandlingMiddleware/docs/1.ExceptionHandlingMiddleware.md) maps **every** entry in `Errors` to a [`PipelineMessage`](../../RequestProcessing/RequestProcessing.Abstractions/docs/7.PipelineResponse.md).
- Derive from `DomainException` for more specific business errors while keeping the same contract behavior.
