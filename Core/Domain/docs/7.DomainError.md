# 7. DomainError

`DomainError` is a single coded domain error: a machine-readable `Code`, a localization template key in `Message`,
and optional positional `Parameters`. Use it with the multi-error [`DomainException`](6.DomainException.md)
constructor when more than one business-rule violation should be reported at once.

- Namespace: `RaccoonLand.Core.Domain.Exceptions`
- Package: `RaccoonLand.Core.Domain`

## Constructor

```csharp
public DomainError(string code, string message, params object?[] parameters)
```

| Parameter | Required | Description |
|-----------|----------|-------------|
| `code` | yes | Error code for the consumer's programmed reaction. Not null/whitespace. |
| `message` | yes | Localization template key. Used as the fallback user-facing text when no `IMessageLocalization` is registered. Not null/whitespace. |
| `parameters` | no | Positional values inserted into the localized template (copied on construction; never null). |

## Surface

| Member | Type | Description |
|--------|------|-------------|
| `Code` | `string` | Machine-readable error code. |
| `Message` | `string` | Localization template key. |
| `Parameters` | `IReadOnlyList<object?>` | Positional localization parameters (never null; defensive copy of the constructor args). |

## Example

```csharp
throw new DomainException(
    new DomainError("order.line_required", "ORDER_LINE_REQUIRED"),
    new DomainError("order.quantity_invalid", "ORDER_QUANTITY_INVALID", requestedQuantity));
```

## Notes

- The single-error `DomainException(code, message, parameters)` constructor creates a `DomainError` internally — you
  only need this type when throwing multiple errors in one exception.
- Constructor arguments are copied so later mutation of the caller's array cannot change this instance.
- Hosts that localize (for example
  [`ExceptionHandlingMiddleware`](../../../Modules/Middlewares/ExceptionHandlingMiddleware/docs/1.ExceptionHandlingMiddleware.md))
  resolve each `DomainError` independently when `IMessageLocalization` is registered.
