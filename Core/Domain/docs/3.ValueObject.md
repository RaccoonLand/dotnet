# 3. ValueObject

`ValueObject` is the base class for immutable, identity-less domain values whose equality is based on their components.

- Namespace: `RaccoonLand.Core.Domain.Abstractions`
- Package: `RaccoonLand.Core.Domain`

## Responsibilities

- Provide structural equality based on a sequence of components.
- Derived types only implement `GetEqualityComponents()` and return their identifying values in order.

## Surface

| Member | Description |
|--------|-------------|
| `GetEqualityComponents()` | `protected abstract` — yields the values that define equality, in order. |
| `Equals` / `GetHashCode` | Implemented in terms of the components. |
| `==` / `!=` | Value equality operators. |

## Equality semantics

- Reference equal ⇒ equal.
- Different runtime type ⇒ not equal.
- Otherwise equal when the component sequences are equal (`SequenceEqual`).
- `GetHashCode` combines all components.

## Example

```csharp
public sealed class Money : ValueObject
{
    public Money(decimal amount, string currency)
    {
        Amount = amount;
        Currency = currency;
    }

    public decimal Amount { get; }
    public string Currency { get; }

    protected override IEnumerable<object?> GetEqualityComponents()
    {
        yield return Amount;
        yield return Currency;
    }
}
```

## Notes

- Order matters: components are compared positionally.
- Keep value objects immutable; expose new instances instead of mutating.
