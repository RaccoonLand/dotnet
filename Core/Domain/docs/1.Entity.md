# 1. Entity

`Entity<TKey>` is the base class for every domain entity in RaccoonLand.

- Namespace: `RaccoonLand.Core.Domain.Abstractions`
- Package: `RaccoonLand.Core.Domain`

## Responsibilities

1. **Identity & equality** — two entities are equal when they have the same runtime type and the same `Id`.
2. **Stable business key** — a sequential GUID (`BusinessKey`) assigned at construction, available before the database allocates a key, so it can be referenced from events.
3. **Auditing** — exposes create/modify audit data, populated by infrastructure (an EF Core interceptor) via the explicitly implemented `IAuditable`.

## Surface

| Member | Type | Notes |
|--------|------|-------|
| `Id` | `TKey` | Primary key. `protected init` — assigned once at construction or by the ORM materializer; never mutated afterwards. |
| `BusinessKey` | `Guid` | Sequential GUID (`Guid.CreateVersion7()`). `protected init` — same immutability rule as `Id`. |
| `CreatedAtUtc` / `CreatedBy` | `DateTimeOffset` / `string?` | Set on insert. |
| `ModifiedAtUtc` / `ModifiedBy` | `DateTimeOffset?` / `string?` | Set on update. |

`TKey` is fully generic (`Guid`, `int`, `long`, ...).

## Equality semantics

- Reference equal ⇒ equal.
- Different runtime type ⇒ not equal.
- **Transient** entities (with a default `Id`) are equal only by reference — two unsaved entities are never considered equal by key.
- `==` / `!=` operators and `GetHashCode` follow the same rules.

## Why `Id` and `BusinessKey` are `init`-only

`GetHashCode` for a persisted entity is `HashCode.Combine(GetType(), Id)`. If `Id` could be reassigned after the entity is placed in a `HashSet<T>` or a `Dictionary<T, V>`, the entity's hash bucket would silently go stale and lookups would fail — a classic hard-to-reproduce bug. Marking `Id` (and, for symmetry, `BusinessKey`) as `protected init` moves that invariant from a documentation footnote to a compile-time constraint:

- Assignable only in a constructor (`base(id)` or a derived-class ctor body such as `Id = Guid.CreateVersion7();`).
- Assignable by ORMs that support init-only setters (EF Core 5+).
- Not assignable from method bodies, `set;`, or reflection-free consumer code.

## Example

```csharp
public sealed class Product : Entity<Guid>
{
    private Product() { }

    public Product(Guid id, string name) : base(id) => Name = name;

    public string Name { get; private set; } = default!;
}
```

## Notes

- The parameterless constructor exists for EF Core materialization.
- Audit mutators are explicit interface implementations (`IAuditable.SetCreatedAudit` / `SetModifiedAudit`) so only infrastructure can set them, keeping the domain-facing API clean.
