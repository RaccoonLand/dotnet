# 9. Pagination (PagedQuery & PageResponse)

Base types for paginated CQRS queries and their responses.

- **Package:** `RaccoonLand.Core.RequestProcessing.Abstractions`
- **Namespace:** `RaccoonLand.Core.RequestProcessing.Abstractions.Cqrs`

## PagedQuery&lt;TResponse&gt;

```csharp
public abstract class PagedQuery<TResponse> : IQuery<TResponse>
{
    public int Page { get; init; } = 1;
    public int PageSize { get; init; } = 20;
    public bool IncludeTotalCount { get; init; }
}
```

| Member | Description |
| --- | --- |
| `Page` | 1-based page number. Default `1`. |
| `PageSize` | Items per page. Default `20`. Max clamping belongs in the validator or endpoint. |
| `IncludeTotalCount` | When `true`, the handler should populate `PageResponse<TItem>.TotalCount`. |

## PageResponse&lt;TItem&gt;

```csharp
public abstract record PageResponse<TItem>
{
    public IReadOnlyList<TItem> Items { get; init; } = [];
    public int? TotalCount { get; init; }
    public int Page { get; init; }
    public int PageSize { get; init; }
    public int TotalPages { get; }
}
```

| Member | Description |
| --- | --- |
| `Items` | Items in the current page. |
| `TotalCount` | Total matching items when requested; `null` when `IncludeTotalCount` was `false`. |
| `Page` | Effective 1-based page number (after any clamping in the handler). |
| `PageSize` | Effective page size used for this result. |
| `TotalPages` | Computed from `TotalCount` and `PageSize`. Zero when `TotalCount` is `null` or `PageSize` is not positive. |

## Example

```csharp
public sealed class SearchPeopleQuery : PagedQuery<SearchPeopleResponse>
{
    public string? LastName { get; init; }
}

public sealed record SearchPeopleResponse : PageResponse<PersonListItem>;

internal sealed class SearchPeopleEndpoint(IPeopleRepository repository)
    : IEndpoint<SearchPeopleQuery, SearchPeopleResponse>
{
    public async Task<Result<SearchPeopleResponse>> ExecuteAsync(
        SearchPeopleQuery request,
        CancellationToken cancellationToken)
    {
        var page = Math.Max(request.Page, 1);
        var pageSize = Math.Clamp(request.PageSize, 1, 100);

        var (items, totalCount) = await repository.SearchAsync(
            request.LastName,
            page,
            pageSize,
            request.IncludeTotalCount,
            cancellationToken);

        return Result<SearchPeopleResponse>.Success(new SearchPeopleResponse
        {
            Items = items,
            TotalCount = request.IncludeTotalCount ? totalCount : null,
            Page = page,
            PageSize = pageSize,
        });
    }
}
```

## Design notes

- `PagedQuery` is an **abstract class** so feature queries can use `class` or `record` as needed.
- `PageResponse` is an **abstract record** so derived responses stay concise (`record SearchPeopleResponse : PageResponse<Item>`).
- Pagination is a **query-only** concern; commands do not use these bases.
- The abstractions do not enforce validation or EF patterns — only the shared shape.
