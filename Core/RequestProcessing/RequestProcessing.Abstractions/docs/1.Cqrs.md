# 1. CQRS contracts (IRequest, ICommand, IQuery, IEndpoint)

These are the foundational markers and handler contracts you use to **author** requests and their handlers.
The pipeline then processes them. They have no dependency on any host or transport.

- **Package:** `RaccoonLand.Core.RequestProcessing.Abstractions`
- **Namespace:** `RaccoonLand.Core.RequestProcessing.Abstractions.Cqrs`

## Request markers

```csharp
public interface IRequestBase;

public interface IRequest : IRequestBase;
public interface IRequest<TResponse> : IRequestBase;

public interface ICommand : IRequest;
public interface ICommand<TResponse> : IRequest<TResponse>;

public interface IQuery<TResponse> : IRequest<TResponse>;
```

| Marker | Meaning |
| --- | --- |
| `IRequestBase` | Common root for every request type. Used by [`PipelineContext`](3.PipelineContext.md) and [`IRequestDispatcher`](8.IRequestDispatcher.md) so pipelines are type-safe instead of accepting `object`. |
| `IRequest` | A request that produces **no** response. |
| `IRequest<TResponse>` | A request that produces a `TResponse`. |
| `ICommand` | A command that **changes state** and produces no response. |
| `ICommand<TResponse>` | A command that changes state and produces a `TResponse`. |
| `IQuery<TResponse>` | A query that **reads state** and produces a `TResponse`. |

For paginated queries, derive from [`PagedQuery<TResponse>`](9.Pagination.md) and return a type derived from
[`PageResponse<TItem>`](9.Pagination.md).

`ICommand`/`IQuery` are intent markers built on top of `IRequest`. They map naturally to
[`RequestKind`](3.PipelineContext.md) (`Command` / `Query`), letting a pipeline apply command‑only concerns
(such as a transaction) while treating queries as side‑effect‑free.

## Endpoint (handler) contracts

An endpoint returns a [`Result`](2.Result.md) / [`Result<TResponse>`](2.Result.md) rather than a bare value,
so the handler can surface **errors and warnings** itself (without throwing):

```csharp
public interface IEndpoint<TRequest>
    where TRequest : IRequestBase
{
    Task<Result> ExecuteAsync(TRequest request, CancellationToken cancellationToken);
}

public interface IEndpoint<TRequest, TResponse>
    where TRequest : IRequestBase
{
    Task<Result<TResponse>> ExecuteAsync(TRequest request, CancellationToken cancellationToken);
}
```

| Contract | For |
| --- | --- |
| `IEndpoint<TRequest>` | Handling a request that returns no payload (`Result`). |
| `IEndpoint<TRequest, TResponse>` | Handling a request that returns a `TResponse` (`Result<TResponse>`). |

An endpoint is the **terminal** of the pipeline: the [`IPipelineBuilder.Build()`](5.IPipelineBuilder.md)
result invokes the matching endpoint for `context.Request`, then maps the returned `Result` to a
[`PipelineResponse`](7.PipelineResponse.md) assigned to `context.Response`.

## Example

```csharp
public sealed record CreateProductCommand(string Name, decimal Price) : ICommand<Guid>;

internal sealed class CreateProductEndpoint(IProductRepository repository)
    : IEndpoint<CreateProductCommand, Guid>
{
    public async Task<Result<Guid>> ExecuteAsync(CreateProductCommand request, CancellationToken cancellationToken)
    {
        if (await repository.ExistsByNameAsync(request.Name, cancellationToken))
        {
            return Result<Guid>.Failure(new PipelineMessage("PRODUCT_NAME_TAKEN", "The product name is already in use."));
        }

        var product = Product.Create(request.Name, request.Price);
        await repository.AddAsync(product, cancellationToken);
        return product.Id; // implicit success
    }
}
```

```csharp
public sealed record GetProductByIdQuery(Guid Id) : IQuery<ProductDto>;

internal sealed class GetProductByIdEndpoint(IProductReader reader)
    : IEndpoint<GetProductByIdQuery, ProductDto>
{
    public async Task<Result<ProductDto>> ExecuteAsync(GetProductByIdQuery request, CancellationToken cancellationToken)
    {
        var dto = await reader.GetAsync(request.Id, cancellationToken);
        return dto is null
            ? Result<ProductDto>.Failure(new PipelineMessage("PRODUCT_NOT_FOUND", "The product was not found."))
            : dto;
    }
}
```

See [Result & Result\<TValue>](2.Result.md) for the full surface (warnings, failures, mapping).

## Design notes

- The markers are intentionally empty: they carry intent for dispatch/registration, not behavior.
- Keeping the request/handler contracts here (rather than in a host package) means the same command/query and
  endpoint definitions are reusable from an API, a worker, or tests — consistent with the host‑independent
  [`PipelineContext`](3.PipelineContext.md).
