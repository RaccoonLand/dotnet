# 6. Registration (AddRaccoonLandRequestProcessing)

`AddRaccoonLandRequestProcessing` is the one-call startup helper that scans assemblies for endpoint
implementations, registers them in DI and the
[`EndpointInvokerRegistry`](3.EndpointInvokerRegistry.md), and builds the command and query pipelines.

- **Package:** `RaccoonLand.Core.RequestProcessing`
- **Namespace:** `RaccoonLand.Core.RequestProcessing.DependencyInjection`

## Surface

```csharp
public static IServiceCollection AddRaccoonLandRequestProcessing(
    this IServiceCollection services,
    Action<IPipelineBuilder>? configureCommandPipeline = null,
    Action<IPipelineBuilder>? configureQueryPipeline = null,
    params Assembly[] scanAssemblies);
```

| Parameter | Description |
| --- | --- |
| `services` | The service collection. |
| `configureCommandPipeline` | Optional callback to add middleware to the command pipeline before it is built. |
| `configureQueryPipeline` | Optional callback to add middleware to the query pipeline before it is built. |
| `scanAssemblies` | Assemblies to scan for `IEndpoint<>` implementations. When omitted, the **calling assembly** is scanned. |

## What it registers

| Service | Lifetime | Description |
| --- | --- | --- |
| `EndpointInvokerRegistry` | Singleton | Populated during the scan; holds invokers and `RequestKind` per request type. |
| `CompiledPipelines` | Singleton | Built immediately with optional middleware; holds the two compiled delegates. |
| `IRequestDispatcher` → `RequestDispatcher` | Singleton | The run-time entry point. |
| Each discovered endpoint type | Scoped | Registered via `AddScoped(endpointType)`. |

## Scan algorithm

For every concrete, non-generic type in the supplied assemblies:

1. Detect whether it implements `IEndpoint<TRequest>` or `IEndpoint<TRequest, TResponse>`.
2. Skip types whose `TRequest` does not implement [`IRequestBase`](../../RequestProcessing.Abstractions/docs/1.Cqrs.md).
3. Register the endpoint type as scoped in DI.
4. Classify the request type as `Command` or `Query` (implements `IQuery<>` → Query).
5. Call `RegisterVoid` or `RegisterResponse` on the registry with the classified `RequestKind`.

After the scan, both pipelines are built and stored in `CompiledPipelines`.

## Example

```csharp
builder.Services.AddRaccoonLandRequestProcessing(
    configureCommandPipeline: pipeline => pipeline
        .Use(/* logging */)
        .Use(/* validation */),
    configureQueryPipeline: pipeline => pipeline
        .Use(/* logging */)
        .Use(/* caching */),
    typeof(Program).Assembly,
    typeof(SomeFeatureEndpoints).Assembly);
```

At run time, inject [`IRequestDispatcher`](../../RequestProcessing.Abstractions/docs/8.IRequestDispatcher.md)
and dispatch any [`IRequestBase`](../../RequestProcessing.Abstractions/docs/1.Cqrs.md):

```csharp
await dispatcher.DispatchAsync(command, scope.ServiceProvider, cancellationToken);
```

## Design notes

- Reflection is used **only at startup** during the assembly scan and invoker compilation — not on the request
  hot path.
- Pipeline middleware is configured via the optional callbacks; if omitted, each pipeline consists of the
  terminal endpoint invoker only. Middleware must be registered as **singleton** and remain **stateless**;
  request-scoped services are resolved from `PipelineContext.RequestServices` — see
  [PipelineDelegate & IPipelineMiddleware](../../RequestProcessing.Abstractions/docs/4.PipelineDelegate.md).
- This extension has no dependency on ASP.NET Core. An API host calls it from `Program.cs`; a worker calls it
  from its host builder.
