# 1. PipelineBuilder

`PipelineBuilder` is the default, concrete implementation of
[`IPipelineBuilder`](../../RequestProcessing.Abstractions/docs/5.IPipelineBuilder.md). It composes the
middleware components registered via `Use(...)` into a single executable `PipelineDelegate`, around a fixed
**terminal** delegate supplied by the derived builder.

- **Package:** `RaccoonLand.Core.RequestProcessing`
- **Namespace:** `RaccoonLand.Core.RequestProcessing.Pipeline`

## Surface

```csharp
public abstract class PipelineBuilder : IPipelineBuilder
{
    protected PipelineBuilder(IServiceProvider applicationServices, PipelineDelegate terminal);

    public IServiceProvider ApplicationServices { get; }
    public IPipelineBuilder Use(Func<PipelineDelegate, PipelineDelegate> middleware);
    public PipelineDelegate Build();
}
```

| Member | Description |
| --- | --- |
| `.ctor(applicationServices, terminal)` | The application (root) service provider and the **terminal** delegate that runs after all middleware (typically the endpoint invoker). |
| `ApplicationServices` | Root service provider, available while composing the pipeline. Not the request scope. |
| `Use(middleware)` | Appends a component expressed as `next => context => { … }`. Returns the builder for chaining. |
| `Build()` | Composes all components around the terminal into a single `PipelineDelegate`. |

## Build algorithm

The algorithm is identical to ASP.NET Core's application builder. `Build()` starts from the terminal and
wraps it with each registered component **in reverse order**, so the first component registered becomes the
outermost one:

```csharp
public PipelineDelegate Build()
{
    var app = _terminal;
    for (var i = _components.Count - 1; i >= 0; i--)
    {
        app = _components[i](app);
    }
    return app;
}
```

Given `builder.Use(A).Use(B).Use(C)`, execution nests as `A → B → C → terminal → C → B → A`.

## Why it is abstract

`PipelineBuilder` itself does not know what "the end of the pipeline" is — that is the terminal delegate.
Concrete builders ([`CommandPipelineBuilder` / `QueryPipelineBuilder`](2.CommandAndQueryPipelineBuilders.md))
supply a terminal that resolves and invokes the right endpoint via the
[`EndpointInvokerRegistry`](3.EndpointInvokerRegistry.md).

## Host independence

`PipelineBuilder` depends only on the abstractions (`IPipelineBuilder`, `PipelineDelegate`) and
`IServiceProvider`. It has no reference to ASP.NET Core, so the same builder is used to compose pipelines in
an API host or a worker.

See also: [Command & Query pipeline builders](2.CommandAndQueryPipelineBuilders.md).
