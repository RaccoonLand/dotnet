# 4. CompiledPipelines

`CompiledPipelines` is a tiny holder for the two `PipelineDelegate`s produced by
[`Build()`](1.PipelineBuilder.md) after the application has configured its command and query pipelines. The
[`RequestDispatcher`](5.RequestDispatcher.md) reads from it at run time to pick the right pipeline per request.

- **Package:** `RaccoonLand.Core.RequestProcessing`
- **Namespace:** `RaccoonLand.Core.RequestProcessing.Pipeline`

## Surface

```csharp
public sealed class CompiledPipelines
{
    public CompiledPipelines(PipelineDelegate command, PipelineDelegate query);

    public PipelineDelegate Command { get; }
    public PipelineDelegate Query { get; }
}
```

| Member | Description |
| --- | --- |
| Constructor | Requires non-null `command` and `query` (`ArgumentNullException` if either is null). |
| `Command` | The compiled command pipeline. |
| `Query` | The compiled query pipeline. |

The holder is immutable after construction — properties are get-only. This prevents accidental runtime swaps of
a singleton pipeline.

## Lifecycle

1. **Startup** — register `CompiledPipelines` as a singleton, typically built inside the DI factory:

```csharp
return new CompiledPipelines(
    commandBuilder.Build(),
    queryBuilder.Build());
```

2. **Run time** — the [`RequestDispatcher`](5.RequestDispatcher.md) selects `Command` or `Query` based on the
   request type's pre-registered kind (via [`EndpointInvokerRegistry`](3.EndpointInvokerRegistry.md)) and
   invokes it. The dispatcher also null-checks both delegates when it is constructed.

## Why a singleton holder

Pipelines are composed **once** at startup but consumed on **every** request. Storing the two delegates in an
immutable singleton lets the dispatcher (also a singleton) read them with zero per-request allocation, and keeps
the build step (one-time) cleanly separated from dispatch (hot path).

## Host independence

`CompiledPipelines` stores plain `PipelineDelegate`s and has no host dependency. The same holder works whether
the pipelines were configured by an API host or a worker.

See also: [PipelineBuilder](1.PipelineBuilder.md), [RequestDispatcher](5.RequestDispatcher.md).
