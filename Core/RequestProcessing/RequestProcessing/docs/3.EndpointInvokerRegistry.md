# 3. EndpointInvokerRegistry

`EndpointInvokerRegistry` maps a **request type** to the terminal `PipelineDelegate` that resolves the
request's endpoint and invokes it, and records the request's [`RequestKind`](../../RequestProcessing.Abstractions/docs/3.PipelineContext.md)
so the [`RequestDispatcher`](5.RequestDispatcher.md) can pick the right pipeline without run-time reflection.

- **Package:** `RaccoonLand.Core.RequestProcessing`
- **Namespace:** `RaccoonLand.Core.RequestProcessing.Dispatch`

## Surface

```csharp
public sealed class EndpointInvokerRegistry
{
    public void RegisterResponse(Type requestType, Type responseType, Type endpointType, RequestKind kind);
    public void RegisterVoid(Type requestType, Type endpointType, RequestKind kind);
    public PipelineDelegate Resolve(Type requestType);
    public RequestKind ResolveKind(Type requestType);
}
```

| Member | Description |
| --- | --- |
| `RegisterResponse` | Registers an endpoint that returns a payload — handles an [`IEndpoint<TRequest, TResponse>`](../../RequestProcessing.Abstractions/docs/1.Cqrs.md). |
| `RegisterVoid` | Registers an endpoint with no payload — handles an `IEndpoint<TRequest>`. |
| `Resolve` | Returns the invoker for `requestType`, or throws if none is registered. |
| `ResolveKind` | Returns the pre-registered `RequestKind` (`Command` or `Query`) for `requestType`. |

## How an invoker works

Registration uses reflection once (at startup) to build a strongly-typed invoker delegate. At run time the
invoker resolves the endpoint from the **request scope**, executes it, and publishes the outcome onto
`context.Response`:

```csharp
private static PipelineDelegate BuildResponseInvoker<TRequest, TResponse>(Type endpointType)
    where TRequest : IRequestBase
    => async context =>
    {
        var endpoint = (IEndpoint<TRequest, TResponse>)context.RequestServices.GetRequiredService(endpointType);
        var result = await endpoint.ExecuteAsync((TRequest)context.Request, context.CancellationToken);
        context.Response = result.ToPipelineResponse();
    };
```

The void variant is identical except it resolves `IEndpoint<TRequest>` and maps the payload-less
[`Result`](../../RequestProcessing.Abstractions/docs/2.Result.md).

## RequestKind at registration time

When an endpoint is registered (via [`AddRaccoonLandRequestProcessing`](6.Registration.md) or manually),
the caller supplies the request's `RequestKind`. The registration extension classifies each request type
once at startup: types implementing [`IQuery<>`](../../RequestProcessing.Abstractions/docs/1.Cqrs.md) are
`Query`; everything else is `Command`. The dispatcher then calls `ResolveKind` on the hot path — no
interface inspection per request.

## Mapping Result → PipelineResponse

Endpoints return a [`Result`](../../RequestProcessing.Abstractions/docs/2.Result.md) /
`Result<TResponse>`. The invoker calls `ToPipelineResponse()` so that both success (payload + warnings) and
failure (errors) are surfaced uniformly on
[`PipelineContext.Response`](../../RequestProcessing.Abstractions/docs/3.PipelineContext.md) as a
[`PipelineResponse`](../../RequestProcessing.Abstractions/docs/7.PipelineResponse.md).

## Resolution from the request scope

The endpoint is resolved from `context.RequestServices`, **not** from the application provider. This means a
fresh, scoped endpoint instance (and its scoped dependencies) is used per request.

See also: [Registration](6.Registration.md), [RequestDispatcher](5.RequestDispatcher.md).
