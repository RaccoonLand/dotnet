# 1. RaccoonLandController

`RaccoonLandController` is the thin ASP.NET Core base class for feature controllers. It exposes
`DispatchAsync` helpers that send a CQRS request through [`IRequestDispatcher`](../../RequestProcessing/RequestProcessing.Abstractions/docs/8.IRequestDispatcher.md)
and map the resulting [`PipelineResponse`](../../RequestProcessing/RequestProcessing.Abstractions/docs/7.PipelineResponse.md)
to an `IActionResult`.

- **Package:** `RaccoonLand.Core.Hosting.AspNetCore`
- **Namespace:** `RaccoonLand.Core.Hosting.AspNetCore.Controllers`
- **Folder:** `src/Controllers/`

## Surface

```csharp
public abstract class RaccoonLandController : ControllerBase
{
    protected IRequestDispatcher    Dispatcher     { get; }
    protected IPipelineResponseMapper ResponseMapper { get; }

    protected Task<IActionResult> DispatchAsync(
        IRequest request,
        CancellationToken cancellationToken = default);

    protected Task<IActionResult> DispatchAsync<TResponse>(
        IRequest<TResponse> request,
        CancellationToken cancellationToken = default);
}
```

Both `Dispatcher` and `ResponseMapper` are resolved from `HttpContext.RequestServices` on each access —
no constructor injection is needed and no `using Microsoft.Extensions.DependencyInjection;` is required
in the controller file.

## Responsibilities

| Layer | Responsibility |
| --- | --- |
| **Controller** | HTTP binding (route/query/body), call `DispatchAsync`, return `IActionResult`. |
| **`IRequestDispatcher`** | Run the request through the command or query pipeline. |
| **`IEndpoint` handler** | Business logic; returns [`Result`](../../RequestProcessing/RequestProcessing.Abstractions/docs/2.Result.md). |
| **Pipeline middleware** | Validation, exception handling, caching, … |
| **`IPipelineResponseMapper`** | Translate `PipelineResponse` → HTTP status + JSON body. |

The controller **must not** contain business logic, direct DbContext access, or exception handling for
`DomainException` — those belong in handlers and pipeline middleware.

## Request scope

`DispatchAsync` passes `HttpContext.RequestServices` to the dispatcher. That is the ASP.NET Core per-request
scope, so scoped handlers, validators, and DbContexts resolve correctly.

## Cancellation

`DispatchAsync` accepts an optional `CancellationToken`. When the caller passes an explicit token
(the usual MVC pattern where the action method accepts and forwards a `CancellationToken`
parameter), it is used unchanged. When the caller omits it — a common oversight — `DispatchAsync`
falls back to [`HttpContext.RequestAborted`](https://learn.microsoft.com/dotnet/api/microsoft.aspnetcore.http.httpcontext.requestaborted)
so a client disconnect still cancels the pipeline and in-flight DB / HTTP / cache work stops.
There is never a silent "no cancellation" path.

The `Dispatcher` property does **not** apply this fallback — that is the responsibility of the
`DispatchAsync` helper. Advanced callers that dispatch through the property directly should either
forward the action's `CancellationToken` or pass `HttpContext.RequestAborted` explicitly.

## Advanced scenarios: `Dispatcher` + `ResponseMapper`

Prefer `DispatchAsync` for the standard "dispatch + envelope-to-HTTP" flow — it is the 90% path.
Reach for `Dispatcher` and `ResponseMapper` only when the default envelope mapping does not fit,
for example a file download where the success case is `File(...)` but errors and `StatusHint` should
still flow through the default mapper.

```csharp
[HttpGet("{id:int}/photo")]
public async Task<IActionResult> DownloadPhoto(
    [FromRoute] int id,
    CancellationToken cancellationToken)
{
    var response = await Dispatcher.DispatchAsync(
        new DownloadPersonPhotoQuery { Id = id },
        HttpContext.RequestServices,
        cancellationToken);

    if (response?.Result is DownloadPersonFileResult file)
    {
        return File(file.Content, file.ContentType ?? "application/octet-stream", file.FileName);
    }

    if (response is { Errors.Count: 0 })
        return NotFound();

    return ResponseMapper.Map(response); // errors / StatusHint go through the default policy
}
```

## Example

```csharp
using RaccoonLand.Core.Hosting.AspNetCore.Controllers;

[ApiController]
[Route("api/persons")]
public sealed class PersonsController : RaccoonLandController
{
    [HttpPost]
    public Task<IActionResult> Create(
        [FromBody] CreatePersonCommand command,
        CancellationToken cancellationToken)
        => DispatchAsync(command, cancellationToken);

    [HttpGet("{id:guid}")]
    public Task<IActionResult> GetById(
        [FromRoute] GetPersonByIdQuery query,
        CancellationToken cancellationToken)
        => DispatchAsync(query, cancellationToken);
}
```
