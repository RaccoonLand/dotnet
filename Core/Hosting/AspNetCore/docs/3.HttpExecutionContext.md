# 3. HttpExecutionContext

HTTP-scoped implementation of
[`ICurrentExecutionContext`](../../ExecutionContext/docs/1.ICurrentExecutionContext.md). Values are resolved from
JWT/OIDC claims and request headers according to [`HttpExecutionContextOptions`](#options),
then exposed to audit, outbox, and handlers for the lifetime of the request.

- **Package:** `RaccoonLand.Core.Hosting.AspNetCore`
- **Namespace:** `RaccoonLand.Core.Hosting.AspNetCore.HttpExecutionContext`
- **Folder:** `src/HttpExecutionContext/` (all types and extensions for this feature)

## Components

| Type | Role |
| --- | --- |
| `HttpExecutionContextOptions` | Claim/header mapping (appsettings + `Action` configure). |
| `MultiValueHeaderMode` | How to read multi-value request headers for tenant / correlation. |
| `HttpExecutionContext` | Scoped `ICurrentExecutionContext` implementation. |
| `HttpExecutionContextPopulator` | Internal claim/header resolution. |
| `HttpExecutionContextMiddleware` | Populates the context once per HTTP request. |
| `AddRaccoonLandHttpExecutionContext` | DI registration. |
| `UseRaccoonLandHttpExecutionContext` | HTTP pipeline middleware (after `UseAuthentication`). |

## Options

```csharp
public enum MultiValueHeaderMode
{
    FirstValue = 0,       // first non-empty value
    SingleValueOnly = 1,  // exactly one non-empty value; else missing (default)
    Join = 2,             // comma-joined (legacy / permissive)
}

public sealed class HttpExecutionContextOptions
{
    public const string SectionName = "HttpExecutionContext";

    public string UserIdClaim { get; set; } = "sub";
    public string? TenantIdClaim { get; set; }
    public string? TenantIdHeader { get; set; }
    public MultiValueHeaderMode TenantIdHeaderMultiValueMode { get; set; } = MultiValueHeaderMode.SingleValueOnly;
    public string CorrelationIdHeader { get; set; } = "X-Correlation-Id";
    public MultiValueHeaderMode CorrelationIdHeaderMultiValueMode { get; set; } = MultiValueHeaderMode.SingleValueOnly;
    public bool GenerateCorrelationIdWhenMissing { get; set; } = true;
    public bool EchoCorrelationIdInResponse { get; set; } = true;
}
```

`TenantIdHeaderMultiValueMode` and `CorrelationIdHeaderMultiValueMode` are configured **separately** so each
identifier can use a different multi-value policy.

### Resolution order

| Field | Sources (in order) |
| --- | --- |
| `UserId` | Claim `UserIdClaim` |
| `TenantId` | Claim `TenantIdClaim`, then header `TenantIdHeader` (using `TenantIdHeaderMultiValueMode`) |
| `CorrelationId` | Header `CorrelationIdHeader` (using `CorrelationIdHeaderMultiValueMode`), then `Activity.Current.TraceId`, then generated GUID v7 when enabled |

`IsAvailable` is `true` when at least one of `UserId`, `TenantId`, or `CorrelationId` is non-null.

### DI ownership of `ICurrentExecutionContext`

`AddRaccoonLandHttpExecutionContext` treats its `ICurrentExecutionContext` registration as
**authoritative**: any prior registration (for example a `NullCurrentExecutionContext` added by
another module) is removed and replaced. This prevents a silent divergence where the middleware
would populate `HttpExecutionContext` while consumers — audit, outbox, handlers — kept injecting
the older implementation and always saw a null context. If you need to override the HTTP-scoped
context (for tests, or a custom hosting scenario), register your replacement **after**
`AddRaccoonLandHttpExecutionContext` / `AddRaccoonLandAspNetCore`.

### Correlation id defaults (opt-out)

`GenerateCorrelationIdWhenMissing` and `EchoCorrelationIdInResponse` default to `true` for tracing DX.
Echoing writes an external-facing response header (`CorrelationIdHeader`). For stricter API shape or privacy,
set one or both to `false` in configuration or code.

## Registration

### From appsettings + optional Action

```json
{
  "HttpExecutionContext": {
    "UserIdClaim": "sub",
    "TenantIdClaim": "tenant_id",
    "TenantIdHeader": "X-Tenant-Id",
    "TenantIdHeaderMultiValueMode": "SingleValueOnly",
    "CorrelationIdHeader": "X-Correlation-Id",
    "CorrelationIdHeaderMultiValueMode": "SingleValueOnly",
    "GenerateCorrelationIdWhenMissing": true,
    "EchoCorrelationIdInResponse": true
  }
}
```

```csharp
builder.Services.AddRaccoonLandHttpExecutionContext(
    builder.Configuration,
    configureOptions: o => o.TenantIdHeader = "X-Tenant-Id");

// or via AddRaccoonLandAspNetCore when configuration is passed:
builder.Services.AddRaccoonLandAspNetCore(builder.Configuration);
```

### Code-only

```csharp
builder.Services.AddRaccoonLandHttpExecutionContext(o =>
{
    o.UserIdClaim = "oid";
    o.TenantIdHeader = "X-Tenant-Id";
    o.TenantIdHeaderMultiValueMode = MultiValueHeaderMode.FirstValue;
    o.EchoCorrelationIdInResponse = false;
});
```

### HTTP pipeline

```csharp
app.UseAuthentication();
app.UseRaccoonLandHttpExecutionContext();
app.UseAuthorization();
```
