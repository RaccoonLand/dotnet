# 4. HttpExceptionHandling

ASP.NET Core HTTP middleware that catches unhandled exceptions from controllers and downstream middleware,
shaping them into HTTP responses.

- **Package:** `RaccoonLand.Core.Hosting.AspNetCore`
- **Namespace:** `RaccoonLand.Core.Hosting.AspNetCore.HttpExceptionHandling`
- **Folder:** `src/HttpExceptionHandling/`
- **Type:** `HttpExceptionHandlingMiddleware` (not to be confused with
  [`ExceptionHandlingMiddleware`](../../../Modules/Middlewares/ExceptionHandlingMiddleware/docs/1.ExceptionHandlingMiddleware.md)
  on the RaccoonLand request pipeline)

## Surface

```csharp
// DI — registers HttpExceptionHandlingOptions
public static IServiceCollection AddRaccoonLandHttpExceptionHandling(
    this IServiceCollection services,
    Action<HttpExceptionHandlingOptions>? configure = null);

// HTTP pipeline — add early so controller and pipeline failures are caught
public static IApplicationBuilder UseRaccoonLandHttpExceptionHandling(this IApplicationBuilder app);
```

| --- | --- |
| `HttpExceptionHandlingMiddleware` | `ExceptionHandlingMiddleware` |
| `HttpExceptionHandlingOptions` | `ExceptionHandlingOptions` |
| `UseRaccoonLandHttpExceptionHandling()` | `UseMiddleware<ExceptionHandlingMiddleware>()` |

## Resolution order

When an unhandled exception escapes downstream HTTP components:

1. **Custom handlers** — registered via `HttpExceptionHandlingOptions.On<T>`, in **registration order**,
   matched with `Type.IsInstanceOfType`. Register **more specific** exception types **before** broader ones
   (for example `On<DbUpdateException>` before `On<Exception>`). An early `On<Exception>` can shadow later
   specific handlers.
2. **Any unhandled exception** — HTTP **500** with a minimal `ProblemDetails` (`Status`, `Title`,
   `Type = about:blank`, `Instance = Request.Path`). Exception details are logged, not leaked in the body.

There is **no built-in `DomainException` handling**. Register a custom handler or use
[`ExceptionHandlingMiddleware`](../../../Modules/Middlewares/ExceptionHandlingMiddleware/docs/1.ExceptionHandlingMiddleware.md)
on the RaccoonLand request pipeline.

If the response has already started, the exception is logged and **rethrown**.

### Fallback response guarantees

The built-in 500 fallback:

- Calls `HttpResponse.Clear()` before writing. Any headers a downstream action set before it
  threw — `Content-Disposition`, `Cache-Control`, `Set-Cookie`, `ETag`, `Vary`, custom headers —
  and any partial body that was buffered are removed. Without this a browser might download the
  JSON error as `orders.csv`, or a CDN might cache the 500 for 5 minutes.
- Serves the body as **`application/problem+json`** (RFC 7807), so clients that route responses
  by content-type (Refit, Swagger UI, API gateways) recognize it as a problem document.

## Custom handler contract (`Task<bool>`)

Returning `true` means the middleware treats the exception as **handled** and will **not** run the next
handler or the 500 fallback. Therefore:

- Return `true` only after writing a complete response (at least a status code, and a body when appropriate).
- Return `false` if you did not handle the exception (so another handler or the fallback can run).

## Example

```csharp
using RaccoonLand.Core.Hosting.AspNetCore.HttpExceptionHandling;
using RaccoonLand.Core.RequestProcessing.DependencyInjection;
using RaccoonLand.Modules.Middlewares.FluentValidationMiddleware;

builder.Services.AddRaccoonLandHttpExceptionHandling();

builder.Services.AddRaccoonLandRequestProcessing(
    configureCommandPipeline: p => p.UseMiddleware<FluentValidationMiddleware>(),
    configureQueryPipeline: p => p.UseMiddleware<FluentValidationMiddleware>(),
    typeof(Program).Assembly);

app.UseRaccoonLandHttpExceptionHandling();
```

## Custom handlers

```csharp
using RaccoonLand.Core.Domain.Exceptions;
using RaccoonLand.Core.RequestProcessing.Abstractions.Responses;

builder.Services.AddRaccoonLandHttpExceptionHandling(options =>
{
    // Specific before general — registration order matters.
    options.On<DomainException>(async (httpContext, ex) =>
    {
        var envelope = new PipelineResponse
        {
            Errors = [.. ex.Errors.Select(e => new PipelineMessage(e.Code, e.Message))],
        };

        httpContext.Response.StatusCode = StatusCodes.Status400BadRequest;
        await httpContext.Response.WriteAsJsonAsync(envelope);
        return true; // only after status + body were written
    });
});
```
