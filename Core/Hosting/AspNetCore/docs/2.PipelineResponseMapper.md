# 2. PipelineResponseMapper

`IPipelineResponseMapper` and `DefaultPipelineResponseMapper` translate a host-agnostic
[`PipelineResponse`](../../RequestProcessing/RequestProcessing.Abstractions/docs/7.PipelineResponse.md) into an
ASP.NET Core `IActionResult`.

- **Package:** `RaccoonLand.Core.Hosting.AspNetCore`
- **Namespace:** `RaccoonLand.Core.Hosting.AspNetCore.PipelineResponseMapping`
- **Folder:** `src/PipelineResponseMapping/`

## Surface

```csharp
public interface IPipelineResponseMapper
{
    IActionResult Map(PipelineResponse? response);
}
```

## Default mapping (`DefaultPipelineResponseMapper`)

The **`PipelineResponse` envelope is always the response body** (`Result`, `Errors`, `Warnings`). Only the HTTP
status code varies:

| Condition | HTTP status |
| --- | --- |
| `StatusHint` has a value | that value (for example **401** / **403** from the authorization middleware) |
| Otherwise, `Errors.Count > 0` | **400** Bad Request |
| Otherwise (success, void, or `null` response) | **200** OK |

`StatusHint` takes precedence: a middleware can suggest the exact status code (see
[`PipelineResponse.StatusHint`](../../RequestProcessing/RequestProcessing.Abstractions/docs/7.PipelineResponse.md)).
When it is `null`, the mapper falls back to the errors/success rule above.

When the pipeline returns `null` (for example a void endpoint), the mapper substitutes an empty envelope
(`Result: null`, `Errors: []`, `Warnings: []`) so the JSON shape is always consistent.

Validation failures and `DomainException` handling (via pipeline middleware) arrive as `Errors` on the
envelope; without a `StatusHint` they map to **400**.

## Custom mapper

Register your own implementation before or instead of the default:

```csharp
services.AddSingleton<IPipelineResponseMapper, MyCustomMapper>();
services.AddRaccoonLandAspNetCore(); // skips default when already registered
```

Use a custom mapper when you need different status codes (for example 422 for validation) or ProblemDetails for
errors. The default mapper always keeps the `PipelineResponse` envelope in the body.

See also: [RaccoonLandController](1.RaccoonLandController.md), [Registration](3.Registration.md).
