# 5. Registration (AddRaccoonLandWorker)

`AddRaccoonLandWorker` registers the worker hosting adapter. Pipeline, dispatcher, and endpoint scanning are
registered separately via `AddRaccoonLandRequestProcessing`.

- **Package:** `RaccoonLand.Core.Hosting.Worker`
- **Namespace:** `RaccoonLand.Core.Hosting.Worker`

## Surface

```csharp
public static IServiceCollection AddRaccoonLandWorker(
    this IServiceCollection services,
    bool wireExecutionContext = true);
```

| Service | Lifetime | Description |
| --- | --- | --- |
| `WorkerExecutionContext` | Scoped | Per-job execution context. |
| `WorkerRequestDispatcher` | Singleton | Opens scope and dispatches. |
| `IPipelineResponseHandler` → `DefaultPipelineResponseHandler` | Singleton | Interprets `PipelineResponse`. |
| `ICurrentExecutionContext` → `WorkerExecutionContext` | Scoped | Only when `wireExecutionContext` is `true` and none registered yet. |

## Worker-only host example

```csharp
using RaccoonLand.Core.Hosting.Worker;
using RaccoonLand.Core.RequestProcessing.DependencyInjection;

var builder = Host.CreateApplicationBuilder(args);

builder.Services.AddRaccoonLandWorker();
builder.Services.AddRaccoonLandRequestProcessing(
    configureCommandPipeline: pipeline => { /* middleware */ },
    typeof(Program).Assembly);

builder.Services.AddHostedService<MyWorker>();

var host = builder.Build();
await host.RunAsync();
```

## API + worker in the same process

```csharp
builder.Services.AddRaccoonLandAspNetCore();
builder.Services.AddRaccoonLandWorker(wireExecutionContext: false); // HTTP context already registered
builder.Services.AddRaccoonLandRequestProcessing(/* shared pipelines */, typeof(Program).Assembly);
builder.Services.AddHostedService<MyWorker>();
```
