# 4. PipelineResponseHandler

`IPipelineResponseHandler` interprets a
[`PipelineResponse`](../../RequestProcessing/RequestProcessing.Abstractions/docs/7.PipelineResponse.md) after a
background dispatch. Unlike the ASP.NET Core
[`IPipelineResponseMapper`](../../AspNetCore/docs/2.PipelineResponseMapper.md), it does **not** produce HTTP —
it returns a [`WorkerDispatchResult`](2.WorkerRequestDispatcher.md) for the worker loop (retry, ack, log).

- **Package:** `RaccoonLand.Core.Hosting.Worker`
- **Namespace:** `RaccoonLand.Core.Hosting.Worker`

## Surface

```csharp
public interface IPipelineResponseHandler
{
    WorkerDispatchResult Handle(PipelineResponse? response);
}

public sealed record WorkerDispatchResult
{
    public required bool IsSuccess { get; init; }
    public PipelineResponse Response { get; init; }
}
```

## Default behaviour (`DefaultPipelineResponseHandler`)

| Condition | `IsSuccess` | `Response` |
| --- | --- | --- |
| `Errors.Count > 0` | `false` | Full envelope |
| Otherwise | `true` | Full envelope (`null` pipeline output → empty envelope) |

The **`PipelineResponse` structure is always preserved** on `WorkerDispatchResult.Response`. Only
`IsSuccess` changes — the worker decides what to do (retry, dead-letter, log) based on that flag and the
envelope contents.

## Custom handler

```csharp
services.AddSingleton<IPipelineResponseHandler, MyWorkerResponseHandler>();
services.AddRaccoonLandWorker(); // skips default when already registered
```

See also: [RaccoonLandBackgroundService](1.RaccoonLandBackgroundService.md).
