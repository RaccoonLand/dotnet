# 3. Registration (Messaging.OutboxRelay)

- **Package:** `RaccoonLand.Modules.Messaging.OutboxRelay`

```csharp
public static IServiceCollection AddRaccoonLandOutboxRelay(
    this IServiceCollection services,
    IConfiguration configuration,
    string sectionName = OutboxRelayOptions.SectionName,
    Action<OutboxRelayOptions>? configure = null);

public static IServiceCollection AddRaccoonLandOutboxRelay(
    this IServiceCollection services,
    Action<OutboxRelayOptions>? configure = null);

public static IServiceCollection AddRaccoonLandDomainEventHandler<TEvent, THandler>(
    this IServiceCollection services,
    ServiceLifetime lifetime = ServiceLifetime.Scoped)
    where TEvent : DomainEvent
    where THandler : class, IDomainEventHandler<TEvent>;
```

| Registers | Lifetime |
| --- | --- |
| `OutboxRelayOptions` | Options (`ValidateOnStart`) |
| `DomainEventHandlerRegistry` | Singleton |
| `IDomainEventDispatcher` → `DomainEventDispatcher` | Singleton |
| `OutboxRelayBackgroundService` | Hosted service (singleton) |
| `IDomainEventHandler<TEvent>` / `THandler` | Scoped by default |

## Full example

```csharp
builder.Services.AddRaccoonLandOutboxEventStore(builder.Configuration, configureSql: sql =>
{
    sql.ConnectionString = builder.Configuration.GetConnectionString("Commands")!;
});

builder.Services.AddRaccoonLandDomainEventHandler<OrderSubmitted, OrderSubmittedHandler>();
builder.Services.AddRaccoonLandOutboxRelay(builder.Configuration);
```

Register **handlers before or after** `AddRaccoonLandOutboxRelay` — the registry is built when first resolved and includes all `DomainEventHandlerRegistration` services.

`IOutboxEventStore` must be registered (SqlServer package). When `ProcessServiceEvents` is `true`,
register `AddRaccoonLandRabbitMqServiceEvents` (or another `IServiceEventPublisher`) as well.

See also: [Overview](../Abstractions/docs/0.Overview.md), [RabbitMQ registration](../RabbitMQ/docs/4.Registration.md), [Options](../Abstractions/docs/5.Options.md).
