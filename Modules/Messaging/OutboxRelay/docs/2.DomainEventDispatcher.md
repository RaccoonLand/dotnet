# 2. DomainEventDispatcher

Default [`IDomainEventDispatcher`](../Abstractions/docs/3.IDomainEventHandler.md).

- **Package:** `RaccoonLand.Modules.Messaging.OutboxRelay`
- **Namespace:** `RaccoonLand.Modules.Messaging.OutboxRelay`

## Behaviour

1. Resolve `EventType` → CLR type + `IDomainEventHandler<TEvent>` from `DomainEventHandlerRegistry`.
2. Deserialize `Payload` with `JsonSerializerDefaults.Web` (same family as the write interceptor).
3. Assert `domainEvent.EventId == outboxEvent.EventId` (envelope/outbox key must match payload).
4. Create an async DI scope; resolve **all** handlers for that closed generic service type.
5. Invoke `HandleAsync` sequentially for each handler.

Missing registration, null deserialize, EventId mismatch, or handler exception fail the dispatch (relay does not mark processed). There is **no domain poison queue** — failures retry via outbox claim lease until fixed.

## Handler example

```csharp
public sealed record OrderSubmitted(Guid OrderBusinessKey) : DomainEvent
{
    public override string EventType => "order.submitted";
}

public sealed class OrderSubmittedHandler : IDomainEventHandler<OrderSubmitted>
{
    public Task HandleAsync(
        OrderSubmitted domainEvent,
        DomainEventHandlingContext context,
        CancellationToken cancellationToken)
    {
        // Idempotent side effect using domainEvent.EventId
        return Task.CompletedTask;
    }
}

// Registration
services.AddRaccoonLandDomainEventHandler<OrderSubmitted, OrderSubmittedHandler>();
```

`EventType` for the registry is taken from an uninitialized `TEvent` instance; the override must return a **constant** contract string (no instance state).

See also: [Registration](3.Registration.md).
