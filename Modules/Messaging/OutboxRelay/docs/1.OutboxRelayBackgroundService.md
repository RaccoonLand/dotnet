# 1. OutboxRelayBackgroundService

Hosted worker that polls the outbox and dispatches/publishes claimed rows.

- **Package:** `RaccoonLand.Modules.Messaging.OutboxRelay`
- **Namespace:** `RaccoonLand.Modules.Messaging.OutboxRelay`

## Behaviour

1. Read `OutboxRelayOptions` via `IOptionsMonitor` each cycle.
2. Open a DI scope; resolve `IOutboxEventStore`.
3. `ClaimPendingAsync(batchSize, categoryFilter, ClaimLease)` (optionally filtered to Domain or Service only).
4. For each row:
   - `Domain` → `IDomainEventDispatcher.DispatchAsync`
   - `Service` → `IServiceEventPublisher.PublishAsync` (when enabled)
   - Unknown category → throw (row not marked; lease retry / ops visibility)
   - On success → `MarkProcessedAsync([row.ToClaim()])` (fencing token)
   - On failure → log error; leave claim for lease retry
5. If the batch was empty, delay `PollInterval`; otherwise poll again immediately.

Does **not** inherit `RaccoonLandBackgroundService`: this loop is infrastructure polling, not CQRS job dispatch. Domain handlers may themselves call application services or raise new commands if the app chooses.

## Failure model

At-least-once. Handlers and publishers must tolerate duplicates (`EventId`).
For Service events, a failed or unroutable broker publish must not mark the outbox row processed
(the RabbitMQ publisher uses mandatory + publisher confirms for this).

**Domain rows have no poison/DLQ in this package.** Persistent handler, registration, or deserialize failures
retry forever via claim-lease reclaim. Fix the handler or stop the relay; multi-handler dispatch is
sequential — if handler 2 fails after handler 1 succeeded, the whole set runs again (all handlers must
be idempotent on `EventId`).

## Samples

Default samples enable `ProcessDomainEvents` only (`ProcessServiceEvents = false`). Service-event
publishing requires `AddRaccoonLandRabbitMqServiceEvents` and `ProcessServiceEvents = true`. Consuming
hosts need inbox + `AddRaccoonLandRabbitMqServiceEventConsumer` separately.

See also: [DomainEventDispatcher](2.DomainEventDispatcher.md), [Registration](3.Registration.md).
