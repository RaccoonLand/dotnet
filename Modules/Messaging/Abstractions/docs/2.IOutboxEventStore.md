# 2. IOutboxEventStore

Storage port for claiming and completing aggregate-event outbox rows.

- **Package:** `RaccoonLand.Modules.Messaging.Abstractions`
- **Namespace:** `RaccoonLand.Modules.Messaging.Abstractions`

## Surface

```csharp
public interface IOutboxEventStore
{
    Task<IReadOnlyList<OutboxEventRecord>> ClaimPendingAsync(
        int batchSize,
        string? category,
        TimeSpan claimLease,
        CancellationToken cancellationToken = default);

    Task MarkProcessedAsync(
        IReadOnlyCollection<OutboxClaim> claims,
        CancellationToken cancellationToken = default);
}

public readonly record struct OutboxClaim(Guid EventId, DateTimeOffset ClaimedOnUtc);
```

## Semantics

| Method | Behaviour |
| --- | --- |
| `ClaimPendingAsync` | Atomically claims up to `batchSize` unpublished rows using `claimLease`. Optional `category` must be known; when `null`, only `Domain`/`Service` rows are eligible (unknown categories stay pending). Concurrent workers must not receive the same live claim. |
| `MarkProcessedAsync` | Marks claims processed only when `ClaimedOnUtc` still matches (fencing). Idempotent for the same claim after success; throws when the token no longer owns the row. |

On handler/publisher failure the relay **does not** call `MarkProcessedAsync`. The claim lease expires and another poll may reclaim the row (**at-least-once**).

`OutboxRelayOptions.ClaimLease` is the usual source of `claimLease` for the hosted relay.

## Options

`OutboxEventStoreOptions` (`SectionName` = `OutboxEventStore`): `Database`, `Schema`, `Table` — must match write-side `OutboxOptions`. Identifiers validated by `SqlIdentifier`.

SQL Server registration and connection string live in `RaccoonLand.Modules.Messaging.SqlServer`.
