# 3. IDomainEventHandler & IDomainEventDispatcher

Contracts for **in-service** handling of `DomainEvent` rows from the outbox.

- **Package:** `RaccoonLand.Modules.Messaging.Abstractions`
- **Namespace:** `RaccoonLand.Modules.Messaging.Abstractions`

## IDomainEventHandler&lt;TEvent&gt;

```csharp
public interface IDomainEventHandler<in TEvent> where TEvent : DomainEvent
{
    Task HandleAsync(
        TEvent domainEvent,
        DomainEventHandlingContext context,
        CancellationToken cancellationToken = default);
}
```

- Runs in the **relay worker**, not in the command that raised the event.
- Must be **idempotent** with respect to `domainEvent.EventId` / `context.OutboxEvent.EventId` (the dispatcher requires these ids to match).
- `DomainEventHandlingContext.OutboxEvent` exposes outbox metadata (aggregate type, created-by, …).

Register handlers with `AddRaccoonLandDomainEventHandler<TEvent, THandler>()` in the OutboxRelay package.

## IDomainEventDispatcher

```csharp
public interface IDomainEventDispatcher
{
    Task DispatchAsync(OutboxEventRecord outboxEvent, CancellationToken cancellationToken = default);
}
```

Deserializes `Payload` to the CLR type registered for `EventType` and invokes all matching handlers.
Throws on missing registration, deserialize failure, or handler failure — the relay then leaves the row for retry.

See also: [IServiceEventPublisher](4.IServiceEventPublisher.md), [OutboxRelay](../OutboxRelay/docs/1.OutboxRelayBackgroundService.md).
