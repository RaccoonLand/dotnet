# 5. Options

- **Package:** `RaccoonLand.Modules.Messaging.Abstractions`
- **Namespace:** `RaccoonLand.Modules.Messaging.Abstractions`

## OutboxEventStoreOptions

| Member | Default | Purpose |
| --- | --- | --- |
| `SectionName` | `OutboxEventStore` | Configuration section. |
| `Database` | `null` | Optional three-part naming database. |
| `Schema` | `dbo` | Schema (simple SQL identifier). |
| `Table` | `OutboxEvent` | Table name (must match write-side outbox). |

`QualifiedTableName` bracket-quotes validated identifiers via `SqlIdentifier`.

## OutboxRelayOptions

| Member | Default | Purpose |
| --- | --- | --- |
| `SectionName` | `OutboxRelay` | Configuration section. |
| `BatchSize` | `20` | Max rows per claim. |
| `PollInterval` | `00:00:02` | Delay when a poll returns no work. |
| `ClaimLease` | `00:05:00` | Passed to `IOutboxEventStore.ClaimPendingAsync`. |
| `ProcessDomainEvents` | `true` | Dispatch Domain category via `IDomainEventDispatcher`. |
| `ProcessServiceEvents` | `false` | Publish Service category via `IServiceEventPublisher` (enable after registering RabbitMQ). |

`BatchSize` must be `> 0`. `PollInterval` must be `>= 0`. `ClaimLease` must be `>= 00:00:01` (SQL stores use whole-second `DATEADD`).

SQL-specific store options (`ConnectionString`) are documented under Messaging.SqlServer.

See also: [Registration](../OutboxRelay/docs/3.Registration.md), [SqlServer registration](../SqlServer/docs/3.Registration.md).
