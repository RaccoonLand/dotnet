# 2. ServiceEventMessageHeaders

Stable AMQP application header names published with every service event.

- **Package:** `RaccoonLand.Modules.Messaging.RabbitMQ`
- **Type:** `ServiceEventMessageHeaders`

| Header | Source |
| --- | --- |
| `raccoonland-event-id` | Outbox `EventId` |
| `raccoonland-event-type` | `EventType` |
| `raccoonland-event-version` | `EventVersion` from JSON payload when present |
| `raccoonland-category` | Always `Service` |
| `raccoonland-aggregate-type` | Aggregate CLR type name |
| `raccoonland-aggregate-business-key` | Aggregate business key |
| `raccoonland-occurred-on-utc` | `OccurredOnUtc` (ISO-8601) |
| `raccoonland-created-by` | Optional audit user |
| `raccoonland-delivery-attempt` | Consumer retry policy only (see below) |
| `raccoonland-poisoned` | Set when publishing to the dead-letter exchange |

### `raccoonland-delivery-attempt`

1-based **application** attempt counter written/updated by the **consumer** when it republishes after a handler failure. It is **not** RabbitMQ’s `redelivered` flag or `x-death` count and can diverge from broker-native redelivery.

Standard AMQP properties also set:

| Property | Value |
| --- | --- |
| `message-id` | Event id |
| `type` | `EventType` |
| `content-type` | `application/json` |
| `content-encoding` | `utf-8` |
| `timestamp` | `OccurredOnUtc` (unix seconds) |

The **body** is the same JSON stored in `OutboxEvent.Payload` (the serialized `ServiceEvent`).
