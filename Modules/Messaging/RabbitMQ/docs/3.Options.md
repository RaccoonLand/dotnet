# 3. Options (RabbitMqServiceEventOptions)

- **Package:** `RaccoonLand.Modules.Messaging.RabbitMQ`
- **Section:** `RabbitMqServiceEvents` (`RabbitMqServiceEventOptions.SectionName`)

| Member | Default | Notes |
| --- | --- | --- |
| `Uri` | `null` | When set (`amqp` / `amqps`), overrides discrete host fields. |
| `HostName` | `localhost` | Used when `Uri` is null. |
| `Port` | `5672` | |
| `UserName` / `Password` | `guest` / `guest` | |
| `VirtualHost` | `/` | |
| `ExchangeName` | `raccoonland.service-events` | Required. |
| `ExchangeType` | `topic` | Required. |
| `DurableExchange` | `true` | |
| `DeclareExchange` | `true` | Declare on first publish. |
| `PersistentMessages` | `true` | Persistent delivery mode. |
| `RoutingKeyFormat` | `{EventType}` | Tokens: `{EventType}`, `{Category}`, `{AggregateType}`. |
| `ClientProvidedName` | `raccoonland-service-events-publisher` | Shown in RabbitMQ management. |

Publishing always uses **publisher confirms** + **`mandatory: true`** (not configurable): unroutable messages fail the publish so the outbox row is not marked processed.

Validation runs on host start (`ValidateOnStart`).

## RabbitMqServiceEventConsumerOptions

Section: `RabbitMqServiceEventConsumer`

| Member | Default | Notes |
| --- | --- | --- |
| `Uri` / host credentials | same pattern as publisher | |
| `ExchangeName` | `raccoonland.service-events` | |
| `ExchangeType` | `topic` | |
| `QueueName` | _(required)_ | Per consuming service. |
| `BindingKeys` | `["#"]` | Topic patterns. |
| `PrefetchCount` | `10` | Must be `> 0`. Under claim contention keep modest when delay &gt; 0. |
| `DeclareTopology` | `true` | |
| `InboxClaimLease` | `00:05:00` | Passed to inbox; must be `>= 00:00:01`. |
| `ClaimHeldByOtherRequeueDelay` | `00:00:02` | Backoff before requeue; delivery stays unacked during delay (`>= 0`). |
| `MaxDeliveryAttempts` | `5` | App-level republish retries; `0` falls back to `RequeueOnFailure`. When `> 0`, DLX required. |
| `EnableDeadLetterTopology` | `true` | Declares `{Queue}.dlx` (**fanout**) + `{Queue}.poison`. |
| `DeadLetterExchangeName` | _(derived)_ | Default `{QueueName}.dlx` (fanout). |
| `DeadLetterQueueName` | _(derived)_ | Default `{QueueName}.poison`. |
| `DeadLetterRoutingKey` | `poison` | On poison publish; fanout does not route by key. |
| `RequeueOnFailure` | `true` | Legacy NACK path when `MaxDeliveryAttempts = 0`. |
| `ClientProvidedName` | `raccoonland-service-events-consumer` | |

Retry/poison publishes use **publisher confirms** + `mandatory: true` on the consumer channel before ACK.

See [Consumer](5.Consumer.md) and [Poison / dead-letter](7.PoisonAndDeadLetter.md).
