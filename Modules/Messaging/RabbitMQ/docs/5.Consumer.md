# 5. RabbitMQ service-event consumer

Hosted consumer that binds a queue, applies the inbox gate, and dispatches handlers.

- **Package:** `RaccoonLand.Modules.Messaging.RabbitMQ`
- **Type:** `RabbitMqServiceEventConsumerBackgroundService`

## Delivery semantics

The pipeline is **at-least-once**. Handlers must be **idempotent** on `EventId` (enforced by the inbox).
Retry republish uses **publisher confirms** + `mandatory: true` before ACK. A crash after a confirmed
republish and before ACK can still duplicate; a failed republish NACK-requeues the original (no silent drop).

`raccoonland-delivery-attempt` is an **application** attempt counter for this consumer’s republish policy — not RabbitMQ `redelivered` / `x-death`.

The dispatcher asserts payload `ServiceEvent.EventId` matches the envelope `ServiceEventMessage.EventId`.

## Flow

```text
BasicDeliver
  → map headers/body → ServiceEventMessage
  → IInboxStore.TryClaimAsync
       AlreadyProcessed → ACK
       ClaimHeldByOther → delay (ClaimHeldByOtherRequeueDelay; delivery unacked) → NACK requeue
       Claimed (+ InboxClaimToken) → IServiceEventDispatcher
                 → MarkProcessed(token) → ACK
                 on failure → Release(token) → retry/poison policy
```

Under `ClaimHeldByOther`, prefer a small `PrefetchCount` when `ClaimHeldByOtherRequeueDelay` is non-zero
so sleeping unacked deliveries do not stall the consumer.

## Topology

Declares (when `DeclareTopology = true`):

- Exchange (same defaults as publisher)
- Durable queue (`QueueName` required)
- Bindings from `BindingKeys` (default `#`)
- Dead-letter topology when `EnableDeadLetterTopology` (required if `MaxDeliveryAttempts > 0`)

Manual ACK only (`autoAck: false`). Prefetch from `PrefetchCount`.

## Handler example

```csharp
public sealed record OrderShippedIntegrationEvent(Guid OrderBusinessKey) : ServiceEvent
{
    public override string EventType => "order.shipped";
}

public sealed class OrderShippedHandler : IServiceEventHandler<OrderShippedIntegrationEvent>
{
    public Task HandleAsync(
        OrderShippedIntegrationEvent serviceEvent,
        ServiceEventHandlingContext context,
        CancellationToken cancellationToken)
        => Task.CompletedTask;
}
```
