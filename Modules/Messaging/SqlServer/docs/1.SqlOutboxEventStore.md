# 1. SqlOutboxEventStore

SQL Server implementation of [`IOutboxEventStore`](../Abstractions/docs/2.IOutboxEventStore.md).

- **Package:** `RaccoonLand.Modules.Messaging.SqlServer`
- **Namespace:** `RaccoonLand.Modules.Messaging.SqlServer`

## Claim

Uses a CTE + `UPDATE … OUTPUT` with `UPDLOCK, READPAST, ROWLOCK`:

1. Select top N rows where `ProcessedOnUtc IS NULL` and the claim lease has expired (`ClaimedOnUtc` null or older than `claimLease` from the call).
2. When `category` is null, restrict to `Category IN ('Domain','Service')`; otherwise require that known category.
3. Set `ClaimedOnUtc = SYSUTCDATETIME()`.
4. Return rows with `Id AS EventId` plus `ClaimedOnUtc` (fencing stamp).

`claimLease` must be between **1 second** and **`TimeSpan.FromSeconds(int.MaxValue)`** inclusive. The store converts it to whole seconds via `Ceiling(TotalSeconds)` for `DATEADD(second, …)`.

## Mark processed

```sql
UPDATE … SET ProcessedOnUtc = ISNULL(ProcessedOnUtc, SYSUTCDATETIME())
WHERE Id = @EventId
  AND ClaimedOnUtc = @ClaimedOnUtc;
```

Throws when zero rows are updated (stale fencing token after reclaim).

When multiple claims are passed, all updates run in **one SQL transaction** (all-or-nothing). A fencing failure or other error rolls back the whole batch. The hosted relay typically marks one claim at a time after each successful dispatch/publish.

## Concurrency

Multiple relay instances can poll safely. A crashed worker’s rows become reclaimable after `claimLease` (at-least-once; handlers/publishers must be idempotent). Lease duration is supplied by the caller (typically `OutboxRelayOptions.ClaimLease`).
