# 4. Inbox (SqlInboxStore)

SQL Server implementation of [`IInboxStore`](../Abstractions/docs/6.Inbox.md).

- **Package:** `RaccoonLand.Modules.Messaging.SqlServer`
- **Type:** `SqlInboxStore`

## Schema (required)

Deduplication is keyed by **`EventId`**. The table **must** enforce uniqueness on `EventId`
alone — typically `PRIMARY KEY (EventId)`. Without a PK/UNIQUE constraint on `EventId`, inbox
gating is incorrect and duplicate handling can occur.

`AddRaccoonLandInboxStore` does not auto-create the table. On first inbox operation the store
validates that required columns exist and that `EventId` has a single-column PRIMARY KEY or UNIQUE
constraint; otherwise it throws a clear `InvalidOperationException`.

```sql
CREATE TABLE dbo.InboxEvent
(
    EventId        uniqueidentifier  NOT NULL CONSTRAINT PK_InboxEvent PRIMARY KEY,
    EventType      nvarchar(256)     NOT NULL,
    ClaimedOnUtc   datetimeoffset    NULL,
    ProcessedOnUtc datetimeoffset    NULL,
    ReceivedOnUtc  datetimeoffset    NOT NULL
);
```

On reclaim of an unprocessed row, only `ClaimedOnUtc` is updated; `EventType` from the first insert is
**immutable**. If the same `EventId` is later claimed with a different `EventType`, `TryClaimAsync` throws
(`InvalidOperationException`) instead of overwriting metadata.

## Claim / fencing

`TryClaimAsync` runs in a **single SQL transaction** (`UPDLOCK, HOLDLOCK` held until commit) so concurrent
workers cannot both receive `Claimed` for the same `EventId`. It returns `InboxClaimAttempt` with
`InboxClaimToken(EventId, ClaimedOnUtc)` when claimed. `MarkProcessedAsync` / `ReleaseAsync` update only
when `ClaimedOnUtc` still matches.

`claimLease` must be between **1 second** and **`TimeSpan.FromSeconds(int.MaxValue)`** (same rules as the outbox store / `DATEADD(second, …)`).

## Registration

```csharp
// Connection string from appsettings section InboxStore:ConnectionString
builder.Services.AddRaccoonLandInboxStore(builder.Configuration);

// Or bind section + override ConnectionString from a named connection
builder.Services.AddRaccoonLandInboxStore(builder.Configuration, configureSql: sql =>
{
    sql.ConnectionString = builder.Configuration.GetConnectionString("CommandConnection")!;
});
```

| --- | --- |

### Connection string configuration

With the `IConfiguration` overload:

1. Bind `InboxStore` (or `sectionName`) into both options types.
2. Optional `configureStore` / `configureSql` run **after** bind and **override** bound values.

`ConnectionString` may be in the section, set only via `configureSql`, or section value then overridden by the callback. Startup validation requires a non-empty final `ConnectionString`.

### Validation on start

- `Database` (when set) / `Schema` / `Table`: `SqlIdentifier` + `ValidateOnStart`.
- `ConnectionString`: required + `ValidateOnStart`.
- On first use: required columns + single-column uniqueness on `EventId` (see Schema above).

Claim lease comes from the consumer call (for example `RabbitMqServiceEventConsumerOptions.InboxClaimLease`), not from inbox store options.
