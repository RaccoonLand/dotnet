### Validation on start

- `Database` (when set) / `Schema` / `Table`: simple SQL identifiers via `SqlIdentifier` (`ValidateOnStart`).
- `ConnectionString`: required and non-whitespace (`ValidateOnStart`).
- On first outbox claim/mark: `ClaimedOnUtc` and `ProcessedOnUtc` must exist (see [Schema](2.Schema.md)).

Invalid identifiers fail at host startup. Missing Messaging columns fail on first store use with an explicit message.

### Connection string configuration

With the `IConfiguration` overload:

1. Options bind from section `OutboxEventStore` (or `sectionName`).
2. Optional `configureStore` / `configureSql` run **after** bind and **override** bound values.

So `ConnectionString` may live in `appsettings` under that section, or be set only in `configureSql` (common when reusing `ConnectionStrings:CommandConnection`). At least one path must leave a non-empty connection string.
