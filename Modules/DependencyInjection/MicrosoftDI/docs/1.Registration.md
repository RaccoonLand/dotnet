# 1. Registration

`AddRaccoonLandDependencyInjection` scans assemblies for classes decorated with
[`ServiceAttribute`](../../Abstractions/docs/1.ServiceAttribute.md) and registers them with Scrutor.

- **Package:** `RaccoonLand.Modules.DependencyInjection.MicrosoftDI`
- **Namespace:** `Microsoft.Extensions.DependencyInjection`

## Surface

```csharp
// Preferred: marker types
public static IServiceCollection AddRaccoonLandDependencyInjection(
    this IServiceCollection services,
    Type assemblyMarker,
    params Type[] additionalAssemblyMarkers);

// Explicit assemblies (at least one required)
public static IServiceCollection AddRaccoonLandDependencyInjectionFromAssemblies(
    this IServiceCollection services,
    params Assembly[] assemblies);
```

There is **no** empty-assembly / `Assembly.GetCallingAssembly()` fallback. Always pass a marker type or
at least one assembly.

**Call once** per `IServiceCollection`. The extension registers an internal marker **before** scanning so a
mid-failure cannot be retried on the same collection. A second call (including after a failed attempt)
throws `InvalidOperationException`. After a failure, discard the collection and create a new one.

The two methods have distinct names so `null` / empty `params` do not create overload ambiguity between
`Type` and `Assembly` arrays. Arrays and elements must not be null; duplicate assemblies in one call are
scanned once (`Distinct`).

## Example

```csharp
builder.Services.AddRaccoonLandDependencyInjection(typeof(Program));

builder.Services.AddRaccoonLandDependencyInjection(
    typeof(Program),
    typeof(SomeOtherMarker));
```

Or pass assemblies:

```csharp
builder.Services.AddRaccoonLandDependencyInjectionFromAssemblies(
    typeof(Program).Assembly,
    typeof(SomeOtherMarker).Assembly);
```

## Prerequisites

Reference both packages:

- `RaccoonLand.Modules.DependencyInjection.Abstractions`
- `RaccoonLand.Modules.DependencyInjection.MicrosoftDI`

See also: [2. Scanning Rules](2.ScanningRules.md).
