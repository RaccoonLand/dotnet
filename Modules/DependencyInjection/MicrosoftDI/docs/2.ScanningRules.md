# 2. Scanning Rules

`AddRaccoonLandDependencyInjection` / `AddRaccoonLandDependencyInjectionFromAssemblies` group attributed
types by `(Lifetime, Registration)` and run one Scrutor scan per combination. This keeps per-class
attribute settings accurate without manual registration code.

## Included types

- Public and internal concrete classes (nested: public or `internal` / `NestedAssembly` only)
- Decorated with `ServiceAttribute` (at most one attribute is meaningful; only attributed types are scanned)
- Attribute `Lifetime` and `Registration` must match the current scan group
- Not an open generic type definition

## Excluded types

- Abstract classes
- Classes without `ServiceAttribute`
- Open generic definitions (`IsGenericTypeDefinition`)
- Private / protected nested types

## Registration mapping

| `ServiceRegistration` | Scrutor call | Notes |
|-----------------------|--------------|--------|
| `Self` | `AsSelf()` | Concrete only |
| `MatchingInterface` | `AsMatchingInterface()` | Non-generic `I{ClassName}`; same namespace preferred; validated **before** scan (throws if missing/ambiguous/generic) |
| `ImplementedInterfaces` | `AsImplementedInterfaces()` | All implemented interfaces |
| `SelfAndImplementedInterfaces` | `AsSelfWithInterfaces()` | Concrete + interfaces in **one** registration. **Scoped/Singleton:** resolving self or an interface yields the **same instance**. **Transient:** each resolve creates a new instance (self vs interface are not shared). |

## Duplicate registrations

Assemblies are deduplicated within a single call. The extension may be invoked only **once** per
`IServiceCollection` (marker is registered before scanning).

If two different attributed implementations would expose the **same service type** (for example both
register as `IFoo`), registration **throws** at startup. Keep one attributed implementation per contract.

`IDisposable` / `IAsyncDisposable` are ignored for conflict detection (they are not treated as app service
contracts).

## Design note

This package intentionally does **not** replace explicit registration for framework primitives such as
`IEndpoint<>`, DbContexts, or middleware. It is meant for application services like repositories,
readers, and domain services.
