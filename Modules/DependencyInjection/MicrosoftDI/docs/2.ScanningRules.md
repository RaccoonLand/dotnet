# 2. Scanning Rules

`AddRaccoonLandDependencyInjection` groups attributed types by `(Lifetime, Registration)` and runs one Scrutor
scan per combination. This keeps per-class attribute settings accurate without manual registration code.

## Included types

- Public and internal concrete classes
- Must have exactly one `ServiceAttribute`
- Attribute `Lifetime` and `Registration` must match the current scan group

## Excluded types

- Abstract classes
- Classes without `ServiceAttribute`
- Open generic definitions

## Registration mapping

| `ServiceRegistration` | Scrutor call |
|-----------------------|--------------|
| `Self` | `AsSelf()` |
| `MatchingInterface` | `AsMatchingInterface()` |
| `ImplementedInterfaces` | `AsImplementedInterfaces()` |
| `SelfAndImplementedInterfaces` | `AsSelf()` then `AsImplementedInterfaces()` |

## Duplicate registrations

Scrutor uses its default replacement strategy. If the same service type is discovered twice, the last
registration wins. Prefer one explicit `[Service]` per implementation type.

## Design note

This package intentionally does **not** replace explicit registration for framework primitives such as
`IEndpoint<>`, DbContexts, or middleware. It is meant for application services like repositories,
readers, and domain services.
