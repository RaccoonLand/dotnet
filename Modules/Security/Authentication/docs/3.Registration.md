# 3. Registration (AddRaccoonLandAuthentication)

`AddRaccoonLandAuthentication` registers JWT Bearer and OpenID Connect schemes from configuration. It is
**provider-agnostic** — any standards-compliant IdP works as long as the scheme options are set correctly.

- **Package:** `RaccoonLand.Modules.Security.Authentication`
- **Namespace:** `RaccoonLand.Modules.Security.Authentication`

## Surface

```csharp
public static IServiceCollection AddRaccoonLandAuthentication(
    this IServiceCollection services,
    IConfiguration configuration,
    string sectionName = AuthenticationOptions.SectionName,
    Action<AuthenticationOptions>? configureOptions = null);
```

| Parameter | Description |
| --- | --- |
| `configuration` | Application configuration (`appsettings`, env vars, user secrets, Key Vault, …). |
| `sectionName` | Root section (default `Authentication`). |
| `configureOptions` | Optional code-based override after binding. |

## What it registers

1. `IOptions<AuthenticationOptions>` bound from configuration.
2. `AddAuthentication` with default schemes from [`AuthenticationOptions`](1.AuthenticationOptions.md).
3. One `AddJwtBearer` per entry in `JwtBearer`.
4. One `AddOpenIdConnect` per entry in `OpenIdConnect`.
5. Optional claim-map clearing when `DisableDefaultClaimMapping` is `true`.

## What it does **not** register

The application remains responsible for:

- `app.UseAuthentication()` and `app.UseAuthorization()`
- Authorization policies (`AddAuthorization`, `[Authorize]`, …)
- Swagger / OpenAPI security definitions

This keeps authentication wiring separate from authorization and API documentation.

## Example

```csharp
using RaccoonLand.Modules.Security.Authentication;

var builder = WebApplication.CreateBuilder(args);

builder.Services.AddRaccoonLandAuthentication(builder.Configuration);

builder.Services.AddAuthorization();
// builder.Services.AddAuthorizationBuilder() ...

var app = builder.Build();

app.UseAuthentication();
app.UseAuthorization();
app.MapControllers();
app.Run();
```

## Package dependencies

| Package | Purpose |
| --- | --- |
| `Microsoft.AspNetCore.App` | `AddAuthentication`, hosting integration. |
| `Microsoft.AspNetCore.Authentication.JwtBearer` | JWT Bearer handler. |
| `Microsoft.AspNetCore.Authentication.OpenIdConnect` | OpenID Connect handler. |
