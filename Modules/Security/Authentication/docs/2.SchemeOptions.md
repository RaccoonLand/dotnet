# 2. SchemeOptions (JwtBearer & OpenIdConnect)

Scheme-specific settings inherit the standard ASP.NET Core handler options types so **properties bindable by
`IConfiguration.Bind`** are available from configuration. Runtime-only members (for example event handlers)
are typically set in `configureOptions`.

- **Package:** `RaccoonLand.Modules.Security.Authentication`
- **Namespace:** `RaccoonLand.Modules.Security.Authentication.Configuration`

## Surface

```csharp
public sealed class JwtBearerOptions
    : Microsoft.AspNetCore.Authentication.JwtBearer.JwtBearerOptions;

public sealed class OpenIdConnectOptions
    : Microsoft.AspNetCore.Authentication.OpenIdConnect.OpenIdConnectOptions;
```

## Multi-scheme registration

Schemes are keyed by name under [`AuthenticationOptions`](1.AuthenticationOptions.md):

```json
{
  "Authentication": {
    "JwtBearer": {
      "Bearer": {
        "Authority": "https://login.example.com",
        "Audience": "api"
      },
      "PartnerBearer": {
        "Authority": "https://partner-idp.example.com",
        "Audience": "partner-api"
      }
    },
    "OpenIdConnect": {
      "oidc": {
        "Authority": "https://login.example.com",
        "ClientId": "spa-client",
        "ResponseType": "code"
      }
    }
  }
}
```

Each key (`Bearer`, `PartnerBearer`, `oidc`) becomes the **authentication scheme name** passed to
`AddJwtBearer` / `AddOpenIdConnect`. Keys must be non-empty (not whitespace). This package deliberately
treats names as **case-insensitive** for uniqueness (to avoid ambiguous names): the same name must not appear
twice under one dictionary, nor under both `JwtBearer` and `OpenIdConnect` (for example `Bearer` and `bearer`
collide). ASP.NET Core's scheme provider may still keep ordinal / case-sensitive names.

## Configuration binding

[`AddRaccoonLandAuthentication`](3.Registration.md) binds root scalar properties once, then merges each scheme
child section **once** into a fresh options instance via `IConfigurationSection.Bind`. Scheme dictionaries are
**not** bound by a full `Get<AuthenticationOptions>()` first, so collection properties such as `Scope` are not
bound twice.

Nested bindable properties such as `TokenValidationParameters`, `ClientSecret`, and scopes bind without extra
code. Binding follows the usual `IConfiguration.Bind` rules; it is not a guarantee that every runtime property
on the handler options type is set from JSON.

After binding (and optional `configureOptions`), values are copied onto the ASP.NET Core handler options with a
**shallow** property copy. Incompatible property types fail with a clear configuration error.

## Claim mapping

When [`AuthenticationOptions.DisableDefaultClaimMapping`](1.AuthenticationOptions.md) is `true` (default),
`MapInboundClaims = false` is applied on every scheme registered by this package so inbound JWT claim types
are not rewritten to legacy SOAP URIs. Process-wide static claim maps are cleared only when
`ClearGlobalJwtClaimTypeMaps` is explicitly set to `true` (irreversible for the process lifetime; affects other
JWT handlers in the process).

See also: [Registration](3.Registration.md).
