# 3. Registration

Register the SQL Server authorization provider — binding options from `IConfiguration` or configuring them in
code.

- **Package:** `RaccoonLand.Modules.Security.Authorization.SqlServer`
- **Namespace:** `Microsoft.Extensions.DependencyInjection` (so `AddRaccoonLandSqlServerAuthorization` is available wherever DI is configured)

## Surface

```csharp
public static IServiceCollection AddRaccoonLandSqlServerAuthorization(
    this IServiceCollection services,
    IConfiguration configuration,
    string sectionName = SqlAuthorizationOptions.SectionName);

public static IServiceCollection AddRaccoonLandSqlServerAuthorization(
    this IServiceCollection services,
    Action<SqlAuthorizationOptions> configure);
```

Both overloads register:

| Service | Lifetime | Notes |
| --- | --- | --- |
| `SqlAuthorizationOptions` | options | bound from `configuration` or configured by `configure`; validated (connection string + both procedure names) |
| `SqlAuthorizationRepository` | singleton | `TryAddSingleton` |
| `IAuthorizationProvider` → `SqlAuthorizationProvider` | scoped | `TryAddScoped` |

The host must also register `ICurrentExecutionContext` (it supplies the current user id) and, when caching is
enabled, an `IDistributedCache`.

## From configuration

```csharp
services.AddRaccoonLandAuthorization();                       // middleware package
services.AddRaccoonLandSqlServerAuthorization(configuration); // section "Authorization:SqlServer"
```

```json
{
  "Authorization": {
    "SqlServer": {
      "ConnectionString": "Server=.;Database=Security;Trusted_Connection=True;TrustServerCertificate=True",
      "AnonymousRequestsProcedure": "dbo.GetAnonymousRequests",
      "AllowedRequestsProcedure": "dbo.GetAllowedRequestsForUser",
      "UserIdParameterName": "UserId",
      "EnableCache": true,
      "AnonymousCacheDuration": "00:05:00",
      "UserCacheDuration": "00:01:00"
    }
  }
}
```

## In code

```csharp
services.AddDistributedMemoryCache(); // required only when EnableCache is true
services.AddRaccoonLandSqlServerAuthorization(options =>
{
    options.ConnectionString = connectionString;
    options.AnonymousRequestsProcedure = "dbo.GetAnonymousRequests";
    options.AllowedRequestsProcedure = "dbo.GetAllowedRequestsForUser";
    options.EnableCache = true;
    options.UserCacheDuration = TimeSpan.FromMinutes(1);
});
```

## Non-HTTP hosts

The provider reads the current user id from `ICurrentExecutionContext`, so it has no HTTP dependency. Any host
that populates the execution context (HTTP, message consumer, background job) works without changes.

See also: [SqlAuthorizationProvider](2.SqlAuthorizationProvider.md), [SqlAuthorizationOptions](1.SqlAuthorizationOptions.md).
