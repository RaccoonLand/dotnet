# 2. SqlAuthorizationProvider

An `IAuthorizationProvider` that decides access from two stored procedures (see
[SqlAuthorizationOptions](1.SqlAuthorizationOptions.md)). It returns **status only** — the stable code, message,
and `StatusHint` are the authorization middleware's responsibility.

- **Package:** `RaccoonLand.Modules.Security.Authorization.SqlServer`
- **Namespace:** `RaccoonLand.Modules.Security.Authorization.SqlServer.Provider`

## Decision flow (deny-by-default)

1. Load the anonymous set (cache or `AnonymousRequestsProcedure`). If `RequestName` is in it → **Allowed**.
2. Read the current user id from `ICurrentExecutionContext.UserId`. If it is empty → **Unauthenticated**.
3. Load the user's allowed set (cache or `AllowedRequestsProcedure` with the user id). If `RequestName` is in
   it → **Allowed**; otherwise → **Denied**.

Anything not explicitly anonymous or explicitly allowed is denied.

## Decisions vs infrastructure failures

| Outcome | Meaning |
| --- | --- |
| `Allowed` / `Denied` / `Unauthenticated` | Authorization decision. |
| Exception (connection error, timeout, SQL failure, …) | Infrastructure / data-access failure. **Not** mapped to `Denied`. Propagates to host / exception-handling middleware. |

## User identity

Unlike the claims provider, this provider does not touch `ClaimsPrincipal`. It reads the user id from
`ICurrentExecutionContext` (package `RaccoonLand.Core.ExecutionContext.Abstractions`), so it works the same in an
HTTP request, a message consumer, or a background job — the host only has to populate the execution context.

## Caching

When `SqlAuthorizationOptions.EnableCache` is set, the anonymous set is cached under `{CacheKeyPrefix}anon` and
each user's allowed set under `{CacheKeyPrefix}user:{userId}`, as a JSON array, using `IDistributedCache`. If
caching is enabled but no `IDistributedCache` is registered, the provider throws at construction with a clear
message. With caching disabled, every decision queries the database directly. Cached sets can keep a previous
allow or deny decision until TTL expires after revocation — see
[SqlAuthorizationOptions](1.SqlAuthorizationOptions.md).

## Dependencies

| Service | Lifetime | Notes |
| --- | --- | --- |
| `ISqlAuthorizationRepository` → `SqlAuthorizationRepository` | singleton | Dapper data access to the two procedures. |
| `ICurrentExecutionContext` | host-provided | Supplies the current `UserId`. |
| `IDistributedCache` | optional | Required only when `EnableCache` is set. |

See also: [SqlAuthorizationOptions](1.SqlAuthorizationOptions.md), [Registration](3.Registration.md).
