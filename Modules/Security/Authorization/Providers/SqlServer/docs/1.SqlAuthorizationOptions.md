# 1. SqlAuthorizationOptions

Externally supplied settings for the SQL Server authorization provider. The provider keeps **no** rules of its
own: it asks two configurable stored procedures for the set of anonymous request names and the set of request
names the current user may execute.

- **Package:** `RaccoonLand.Modules.Security.Authorization.SqlServer`
- **Namespace:** `RaccoonLand.Modules.Security.Authorization.SqlServer.Configuration`

## Members

| Member | Default | Description |
| --- | --- | --- |
| `ConnectionString` *(required)* | — | Connection string to the database hosting the procedures. |
| `AnonymousRequestsProcedure` *(required)* | — | Procedure returning the anonymous request full-names (no required parameters). |
| `AllowedRequestsProcedure` *(required)* | — | Procedure returning the request full-names allowed for a user; receives the user id. |
| `UserIdParameterName` | `UserId` | Parameter name (without `@`) passed to `AllowedRequestsProcedure`. |
| `CommandTimeoutSeconds` | `30` | Command timeout applied to each stored-procedure call. |
| `EnableCache` | `false` | Cache the resolved sets in `IDistributedCache`. Requires a registered distributed cache. |
| `CacheKeyPrefix` | `raccoonland:authz:` | Prefix applied to every cache key. |
| `AnonymousCacheDuration` | `00:05:00` | Lifetime of the cached anonymous set. |
| `UserCacheDuration` | `00:01:00` | Lifetime of each user's cached allowed set. |

## Stored-procedure contract

Both procedures are **set-based**: each returns a single column of request full-names (the values of
`request.GetType().FullName`). The provider checks membership in memory.

```sql
-- Anonymous requests: no required parameters.
CREATE PROCEDURE dbo.GetAnonymousRequests
AS
BEGIN
    SET NOCOUNT ON;
    SELECT r.[RequestName]
    FROM dbo.AnonymousRequests AS r;
END;

-- Allowed requests for a user: receives @UserId.
CREATE PROCEDURE dbo.GetAllowedRequestsForUser
    @UserId NVARCHAR(128)
AS
BEGIN
    SET NOCOUNT ON;
    SELECT ra.[RequestName]
    FROM dbo.RequestAccess AS ra
    WHERE ra.[UserId] = @UserId;
END;
```

## Caching trade-off

Caching avoids a database round-trip per request, but a cached set keeps granting (or denying) access until the
entry expires — so **revoking access is not immediate**. Keep `UserCacheDuration` short (the 1-minute default is
a reasonable starting point) and disable caching entirely (`EnableCache = false`) when immediate revocation is
required.
