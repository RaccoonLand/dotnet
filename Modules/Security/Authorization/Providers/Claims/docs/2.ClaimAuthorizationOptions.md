# 2. ClaimAuthorizationOptions

`ClaimAuthorizationOptions` holds the externally supplied access rules for
[`ClaimAuthorizationProvider`](3.ClaimAuthorizationProvider.md), keyed by the request type's full name.

- **Package:** `RaccoonLand.Modules.Security.Authorization.Claims`
- **Namespace:** `RaccoonLand.Modules.Security.Authorization.Claims.Configuration`

## Surface

```csharp
public sealed class ClaimAuthorizationOptions
{
    public ISet<string> AnonymousRequests { get; }
    public IDictionary<string, ClaimAuthorizationPolicy> Policies { get; }

    public ClaimAuthorizationOptions AllowAnonymous(string requestFullName);
    public ClaimAuthorizationOptions RequireAuthenticated(string requestFullName);
    public ClaimAuthorizationOptions RequireClaim(string requestFullName, string claimType, params string[] allowedValues);
    public ClaimAuthorizationOptions RequireAssertion(string requestFullName, Func<ClaimsPrincipal, bool> assertion);
}

public sealed class ClaimAuthorizationPolicy
{
    public IList<ClaimRequirement> Requirements { get; }
    public IList<Func<ClaimsPrincipal, bool>> Assertions { get; }
    public bool IsSatisfiedBy(ClaimsPrincipal principal);
}

public sealed record ClaimRequirement(string ClaimType, IReadOnlyList<string> AllowedValues);
```

## Fluent helpers

| Method | Effect |
| --- | --- |
| `AllowAnonymous(name)` | Marks the request as publicly accessible (no authentication). |
| `RequireAuthenticated(name)` | Requires any authenticated caller (an empty policy). |
| `RequireClaim(name, type, values...)` | Adds a claim requirement. Empty `values` → any value of `type`; otherwise the value must match **one of** `values` (OR). |
| `RequireAssertion(name, assertion)` | Adds a custom predicate over the caller's `ClaimsPrincipal`. |

## How requirements combine — OR within a requirement, AND across them

- **Within one `ClaimRequirement`**, `AllowedValues` are alternatives: the caller satisfies it by having the
  claim with **any one** of the values (OR). This matches `ClaimRequirement.IsSatisfiedBy`, which uses
  `AllowedValues.Any(...)`.
- **Across requirements and assertions** in a policy, **all** must hold (AND) — see
  `ClaimAuthorizationPolicy.IsSatisfiedBy`.

## Deny-by-default

A request is authorized only when it is in `AnonymousRequests` **or** has a policy in `Policies` that the
caller satisfies. A request with neither is **denied**. Adding a new endpoint therefore fails closed until a
rule is added.

## Example (code)

```csharp
services.AddRaccoonLandClaimAuthorization(options =>
{
    options
        .AllowAnonymous("MyApp.Auth.Commands.LoginCommand")
        .RequireAuthenticated("MyApp.People.Queries.GetMyProfileQuery")
        .RequireClaim("MyApp.People.Commands.CreatePersonCommand", "permission", "people.write")
        .RequireClaim("MyApp.People.Commands.DeletePersonCommand", "role", "admin", "owner")
        .RequireAssertion("MyApp.People.Commands.DeletePersonCommand",
            principal => principal.HasClaim("department", "HR"));
});
```

`RequireClaim` with empty values checks only for the presence of the claim type. For configuration-driven
rules, see [Registration](4.Registration.md).

See also: [ClaimAuthorizationProvider](3.ClaimAuthorizationProvider.md), [Registration](4.Registration.md).
