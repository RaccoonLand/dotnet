# 3. ClaimAuthorizationProvider

`ClaimAuthorizationProvider` is an [`IAuthorizationProvider`](../../../Abstractions/docs/1.IAuthorizationProvider.md)
that decides access from [`ClaimAuthorizationOptions`](2.ClaimAuthorizationOptions.md) against the current
caller's `ClaimsPrincipal`. It returns **only a status** — codes and messages are the middleware's job.

- **Package:** `RaccoonLand.Modules.Security.Authorization.Claims`
- **Namespace:** `RaccoonLand.Modules.Security.Authorization.Claims.Provider`

## Decision flow

For a request keyed by its full type name:

1. **Anonymous** — in `AnonymousRequests` → `Allow()`.
2. **Unauthenticated** — no principal or `Identity.IsAuthenticated != true` → `Unauthenticated()`.
3. **No policy** — authenticated but the request has no entry in `Policies` → `Deny()` (deny-by-default).
4. **Policy** — `policy.IsSatisfiedBy(principal)` → `Allow()`, otherwise → `Deny()`.

```csharp
if (options.AnonymousRequests.Contains(requestName))
    return AuthorizationDecision.Allow();

var principal = claimsPrincipalAccessor.Principal;
if (principal?.Identity?.IsAuthenticated != true)
    return AuthorizationDecision.Unauthenticated();

if (!options.Policies.TryGetValue(requestName, out var policy))
    return AuthorizationDecision.Deny();

return policy.IsSatisfiedBy(principal)
    ? AuthorizationDecision.Allow()
    : AuthorizationDecision.Deny();
```

## How a policy is satisfied

A `ClaimAuthorizationPolicy` is satisfied when **every** claim requirement **and** **every** custom assertion
holds (AND). Within a single `ClaimRequirement`, the accepted values are alternatives (OR):

| `ClaimRequirement.AllowedValues` | Satisfied when |
| --- | --- |
| empty | the principal has any claim of `ClaimType` |
| non-empty | the principal has a claim of `ClaimType` whose value is **one of** `AllowedValues` (OR) |

> The matching uses `AllowedValues.Any(...)`, which is an **OR** over the values of a single requirement.
> The **AND** applies only across multiple requirements/assertions in the same policy.

## Dependencies

| Dependency | Role |
| --- | --- |
| `IClaimsPrincipalAccessor` | Supplies the current `ClaimsPrincipal`. |
| `IOptions<ClaimAuthorizationOptions>` | The externally supplied access rules. |

Registered as the active `IAuthorizationProvider` by [`AddRaccoonLandClaimAuthorization`](4.Registration.md).

## Cancellation

`AuthorizeAsync` accepts `CancellationToken` for contract compatibility with
[`IAuthorizationProvider`](../../../Abstractions/docs/1.IAuthorizationProvider.md). This provider is fully
in-memory and synchronous, so the token is **not** observed.

See also: [ClaimAuthorizationOptions](2.ClaimAuthorizationOptions.md), [IClaimsPrincipalAccessor](1.IClaimsPrincipalAccessor.md).
