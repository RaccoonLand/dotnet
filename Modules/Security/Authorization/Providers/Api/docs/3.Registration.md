# 3. Registration

Register the API authorization provider — binding options from `IConfiguration` or configuring them in code.

- **Package:** `RaccoonLand.Modules.Security.Authorization.Api`
- **Namespace:** `Microsoft.Extensions.DependencyInjection` (so `AddRaccoonLandApiAuthorization` is available wherever DI is configured)

## Surface

```csharp
public static IServiceCollection AddRaccoonLandApiAuthorization(
    this IServiceCollection services,
    IConfiguration configuration,
    string sectionName = ApiAuthorizationOptions.SectionName,
    Action<IHttpClientBuilder>? configureClient = null);

public static IServiceCollection AddRaccoonLandApiAuthorization(
    this IServiceCollection services,
    Action<ApiAuthorizationOptions> configure,
    Action<IHttpClientBuilder>? configureClient = null);
```

Both overloads register:

| Service | Lifetime | Notes |
| --- | --- | --- |
| `ApiAuthorizationOptions` | options | bound from `configuration` or configured by `configure`; validated (base address + both paths, `{userId}` placeholder) |
| `AuthorizationApiClient` | typed client | `AddHttpClient` with base address + timeout, plus the auth handler |
| `AuthorizationApiAuthenticationHandler` | transient | `TryAddTransient`; attaches `None`/`ApiKey`/`Bearer` credentials |
| `IAuthorizationProvider` → `ApiAuthorizationProvider` | scoped | `TryAddScoped` |

The host must also register `ICurrentExecutionContext` (it supplies the current user id) and, when caching is
enabled, an `IDistributedCache`.

## From configuration

```csharp
services.AddRaccoonLandAuthorization();                  // middleware package
services.AddRaccoonLandApiAuthorization(configuration);  // section "Authorization:Api"
```

```json
{
  "Authorization": {
    "Api": {
      "BaseAddress": "https://policy.internal/api/",
      "AnonymousRequestsPath": "anonymous-requests",
      "AllowedRequestsPath": "users/{userId}/allowed-requests",
      "AuthenticationMode": "ApiKey",
      "ApiKeyHeaderName": "X-Api-Key",
      "ApiKey": "…",
      "EnableCache": true,
      "UserCacheDuration": "00:01:00"
    }
  }
}
```

## In code

```csharp
services.AddDistributedMemoryCache(); // required only when EnableCache is true
services.AddRaccoonLandApiAuthorization(options =>
{
    options.BaseAddress = new Uri("https://policy.internal/api/");
    options.AuthenticationMode = AuthorizationApiAuthenticationMode.Bearer;
    options.BearerToken = bearerToken;
    options.EnableCache = true;
});
```

## Custom authentication (OAuth2 / token propagation)

Attach your own handler to the named client through `configureClient`:

```csharp
services.AddTransient<ClientCredentialsTokenHandler>();
services.AddRaccoonLandApiAuthorization(
    configuration,
    configureClient: client => client.AddHttpMessageHandler<ClientCredentialsTokenHandler>());
```

The built-in handler runs first; a custom handler can acquire and cache a machine-to-machine token, or propagate
the end-user token, and set the `Authorization` header itself.

## Non-HTTP hosts

The provider reads the current user id from `ICurrentExecutionContext`, so it has no inbound-HTTP dependency. Any
host that populates the execution context works without changes.

See also: [ApiAuthorizationProvider](2.ApiAuthorizationProvider.md), [ApiAuthorizationOptions](1.ApiAuthorizationOptions.md).
