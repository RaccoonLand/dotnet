# 2. ApiAuthorizationProvider

An `IAuthorizationProvider` that decides access by calling an external authorization (policy) API (see
[ApiAuthorizationOptions](1.ApiAuthorizationOptions.md)). It returns **status only** — the stable code, message,
and `StatusHint` are the authorization middleware's responsibility.

- **Package:** `RaccoonLand.Modules.Security.Authorization.Api`
- **Namespace:** `RaccoonLand.Modules.Security.Authorization.Api.Provider`

## Decision flow (deny-by-default)

1. Load the anonymous set (cache or the anonymous-requests endpoint). If `RequestName` is in it → **Allowed**.
2. Read the current user id from `ICurrentExecutionContext.UserId`. If it is empty → **Unauthenticated**.
3. Load the user's allowed set (cache or the allowed-requests endpoint). If `RequestName` is in it →
   **Allowed**; otherwise → **Denied**.

Anything not explicitly anonymous or explicitly allowed is denied.

## HTTP and authentication

Calls go through a typed `AuthorizationApiClient` registered with `IHttpClientFactory`, so socket reuse and
handler lifetimes are managed correctly. A built-in `DelegatingHandler` attaches credentials per
`AuthenticationMode` (`None`, `ApiKey`, `Bearer`); advanced schemes are added through the `configureClient` hook
at registration.

A non-success HTTP response throws (the call fails loudly rather than silently denying); use the middleware and
host error handling to translate transport failures.

## User identity

Like the SQL Server provider and unlike the claims provider, this provider does not touch `ClaimsPrincipal`. It
reads the user id from `ICurrentExecutionContext` (package `RaccoonLand.Core.ExecutionContext.Abstractions`), so
it works the same in an HTTP request, a message consumer, or a background job.

## Caching

When `ApiAuthorizationOptions.EnableCache` is set, the anonymous set is cached under `{CacheKeyPrefix}anon` and
each user's allowed set under `{CacheKeyPrefix}user:{userId}`, as a JSON array, using `IDistributedCache`. If
caching is enabled but no `IDistributedCache` is registered, the provider throws at construction with a clear
message.

## Dependencies

| Service | Lifetime | Notes |
| --- | --- | --- |
| `AuthorizationApiClient` | typed client | HTTP access to the authorization API. |
| `ICurrentExecutionContext` | host-provided | Supplies the current `UserId`. |
| `IDistributedCache` | optional | Required only when `EnableCache` is set. |

See also: [ApiAuthorizationOptions](1.ApiAuthorizationOptions.md), [Registration](3.Registration.md).
