# 1. ApiAuthorizationOptions

Externally supplied settings for the API authorization provider. The provider keeps **no** rules of its own: it
calls an external authorization (policy) API for the set of anonymous request names and the set of request names
the current user may execute.

- **Package:** `RaccoonLand.Modules.Security.Authorization.Api`
- **Namespace:** `RaccoonLand.Modules.Security.Authorization.Api.Configuration`

## Members

| Member | Default | Description |
| --- | --- | --- |
| `BaseAddress` *(required)* | — | Base address of the API; should end with a trailing `/`. |
| `AnonymousRequestsPath` | `anonymous-requests` | Relative path returning the anonymous request names. |
| `AllowedRequestsPath` | `users/{userId}/allowed-requests` | Relative path returning a user's allowed request names; must contain `{userId}`. |
| `TimeoutSeconds` | `30` | Per-call HTTP timeout. |
| `AuthenticationMode` | `None` | `None`, `ApiKey`, or `Bearer`. |
| `ApiKeyHeaderName` | `X-Api-Key` | Header used in `ApiKey` mode. |
| `ApiKey` | — | API key value (`ApiKey` mode). |
| `BearerToken` | — | Static bearer token (`Bearer` mode). |
| `EnableCache` | `false` | Cache the resolved sets in `IDistributedCache`. Requires a registered distributed cache. |
| `CacheKeyPrefix` | `raccoonland:authz:` | Prefix applied to every cache key. |
| `AnonymousCacheDuration` | `00:05:00` | Lifetime of the cached anonymous set. |
| `UserCacheDuration` | `00:01:00` | Lifetime of each user's cached allowed set. |

## API contract

Both endpoints are **set-based**: each returns a JSON object whose `requests` array holds request full-names
(the values of `request.GetType().FullName`). The provider checks membership in memory.

```http
GET {BaseAddress}{AnonymousRequestsPath}
GET {BaseAddress}{AllowedRequestsPath with {userId} replaced}
```

```json
{ "requests": [ "MyApp.People.Commands.CreatePersonCommand", "MyApp.People.Queries.GetMyProfileQuery" ] }
```

The `{userId}` placeholder is replaced with the URL-escaped value of `ICurrentExecutionContext.UserId`.

## Authenticating to the API

These are **service-to-service** calls: the API trusts the calling service and the `userId` it sends. Built-in
modes cover the common cases:

- **None** — internal network or infrastructure-level security (mTLS, service mesh).
- **ApiKey** — `ApiKey` sent in `ApiKeyHeaderName`.
- **Bearer** — static `BearerToken` in the `Authorization` header.

For OAuth2 **client-credentials** or end-user **token propagation**, register your own `DelegatingHandler` on the
named client via the `configureClient` hook (see [Registration](3.Registration.md)).

## Caching trade-off

Caching avoids an API round-trip per request, but a cached set keeps granting (or denying) access until the entry
expires — so **revoking access is not immediate**. Keep `UserCacheDuration` short, and disable caching when
immediate revocation is required.

See also: [ApiAuthorizationProvider](2.ApiAuthorizationProvider.md), [Registration](3.Registration.md).
