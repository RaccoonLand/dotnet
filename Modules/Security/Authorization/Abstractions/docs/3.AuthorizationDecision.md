# 3. AuthorizationDecision

`AuthorizationDecision` is the result an [`IAuthorizationProvider`](1.IAuthorizationProvider.md) returns. It
carries **only** the outcome category — no code and no message.

- **Package:** `RaccoonLand.Modules.Security.Authorization.Abstractions`
- **Namespace:** `RaccoonLand.Modules.Security.Authorization.Abstractions`

## Surface

```csharp
public sealed record AuthorizationDecision
{
    public required AuthorizationStatus Status { get; init; }
    public bool IsAllowed => Status == AuthorizationStatus.Allowed;

    public static AuthorizationDecision Allow();
    public static AuthorizationDecision Deny();
    public static AuthorizationDecision Unauthenticated();
}
```

| Member | Description |
| --- | --- |
| `Status` | The outcome category — see [AuthorizationStatus](4.AuthorizationStatus.md). |
| `IsAllowed` | Convenience: `Status == Allowed`. |

## Providers do not produce messages

A provider's only job is to decide **whether** access is allowed. It does not create user-facing codes or
localized text — that is the caller's concern. In the RaccoonLand pipeline the authorization **middleware** is
the caller: it maps a non-allowed decision to a `PipelineResponse` (stable code, localized message via
`IMessageLocalization`, and `StatusHint`). Keeping providers free of presentation concerns means the same
decision can be rendered differently by different hosts.

## Factory methods

| Method | Status | Typical host mapping |
| --- | --- | --- |
| `Allow()` | `Allowed` | continue the pipeline |
| `Unauthenticated()` | `Unauthenticated` | `StatusHint = 401` |
| `Deny()` | `Denied` | `StatusHint = 403` |

See also: [AuthorizationStatus](4.AuthorizationStatus.md), [IAuthorizationProvider](1.IAuthorizationProvider.md).
