# 2. AuthorizationContext

`AuthorizationContext` carries the information an [`IAuthorizationProvider`](1.IAuthorizationProvider.md) needs
to make a decision.

- **Package:** `RaccoonLand.Modules.Security.Authorization.Abstractions`
- **Namespace:** `RaccoonLand.Modules.Security.Authorization.Abstractions`

## Surface

```csharp
public sealed record AuthorizationContext(string RequestName);
```

| Member | Description |
| --- | --- |
| `RequestName` | The full name of the request type (`request.GetType().FullName`). Unique within the solution; the key providers use to look up access rules. |

## Why the full type name

The full type name is stable and unique per request type across the project, so it works as a portable rule
key in configuration, a database table, or a remote authorization service — without coupling the request to
any authorization concern.

## Extensibility

The context is intentionally minimal. If resource-based authorization is needed later (for example inspecting
the request payload or the target entity id), the record can gain additional members without breaking the
contract's shape.
