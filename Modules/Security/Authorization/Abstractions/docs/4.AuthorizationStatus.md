# 4. AuthorizationStatus

`AuthorizationStatus` is the outcome category of an [`AuthorizationDecision`](3.AuthorizationDecision.md).

- **Package:** `RaccoonLand.Modules.Security.Authorization.Abstractions`
- **Namespace:** `RaccoonLand.Modules.Security.Authorization.Abstractions`

## Surface

```csharp
public enum AuthorizationStatus
{
    Allowed,
    Denied,
    Unauthenticated,
}
```

| Value | Meaning | Host status hint |
| --- | --- | --- |
| `Allowed` | The caller may execute the request (includes anonymous requests). | — (pipeline continues) |
| `Denied` | The caller is authenticated but not permitted. | `403` |
| `Unauthenticated` | No authenticated caller is available; authentication is required first. | `401` |

## Why split Unauthenticated from Denied

Separating the two lets a host respond with the correct status: `401` invites the caller to authenticate,
while `403` signals that authentication succeeded but access is still refused. The numeric hint is carried on
`PipelineResponse.StatusHint` and is transport-agnostic — it uses familiar HTTP-style numbers without coupling
the pipeline to ASP.NET Core.
