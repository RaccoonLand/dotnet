# 3. Message templates

`AuthorizationMessageTemplates` holds the stable keys the middleware produces for non-allowed decisions.

- **Package:** `RaccoonLand.Modules.Security.Authorization.Middlewares`
- **Namespace:** `RaccoonLand.Modules.Security.Authorization.Middlewares`

## Surface

```csharp
public static class AuthorizationMessageTemplates
{
    public const string AuthenticationRequired = "AUTHENTICATION_REQUIRED";
    public const string AccessDenied = "ACCESS_DENIED";
}
```

| Key | Status | `StatusHint` |
| --- | --- | --- |
| `AUTHENTICATION_REQUIRED` | `Unauthenticated` | `401` |
| `ACCESS_DENIED` | `Denied` | `403` |

## How they are used

For a non-allowed decision the middleware emits a single `PipelineMessage` where **both** the `Code` and the
localization template key are this constant:

```csharp
var message = localizer is null ? template : localizer.Get(template);
context.Response = new PipelineResponse
{
    StatusHint = statusHint,
    Errors = [new PipelineMessage(template, message)],
};
```

- `Code` — the stable, loggable key clients branch on (never localized).
- `Message` — the localized text for the key, or the key itself when no `IMessageLocalization` is registered.

## Convention

Keys follow the framework's `UPPER_SNAKE_CASE` template convention (the same style as the validation and
business message templates). Register translations for `AUTHENTICATION_REQUIRED` and `ACCESS_DENIED` in your
localization store to control the user-facing text per culture.
