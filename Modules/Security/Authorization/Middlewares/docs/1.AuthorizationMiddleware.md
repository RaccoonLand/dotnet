# 1. AuthorizationMiddleware

`AuthorizationMiddleware` is a class-based [`IPipelineMiddleware`](../../../../Core/RequestProcessing/RequestProcessing.Abstractions/docs/4.PipelineDelegate.md)
that authorizes the current request before it reaches the terminal endpoint.

- **Package:** `RaccoonLand.Modules.Security.Authorization.Middlewares`
- **Namespace:** `RaccoonLand.Modules.Security.Authorization.Middlewares`

## How it works

1. Computes the request key from `context.Request.GetType().FullName`.
2. Resolves the active [`IAuthorizationProvider`](../../Abstractions/docs/1.IAuthorizationProvider.md) from
   `context.RequestServices`.
3. Calls `AuthorizeAsync(new AuthorizationContext(requestName), ct)`.
4. On `Allowed`, calls `next(context)`.
5. Otherwise short-circuits: builds a [`PipelineResponse`](../../../../Core/RequestProcessing/RequestProcessing.Abstractions/docs/7.PipelineResponse.md)
   error envelope and does **not** call `next`.

```csharp
var requestType = context.Request.GetType();
var requestName = requestType.FullName ?? requestType.Name;

var provider = context.RequestServices.GetRequiredService<IAuthorizationProvider>();
var decision = await provider.AuthorizeAsync(new AuthorizationContext(requestName), context.CancellationToken);

if (decision.IsAllowed)
{
    await next(context);
    return;
}

// The middleware owns the response — the provider returns only a status.
var (template, statusHint) = decision.Status == AuthorizationStatus.Unauthenticated
    ? (AuthorizationMessageTemplates.AuthenticationRequired, 401)
    : (AuthorizationMessageTemplates.AccessDenied, 403);

var localizer = context.RequestServices.GetService<IMessageLocalization>();
var message = localizer is null ? template : localizer[template];

context.Response = new PipelineResponse
{
    StatusHint = statusHint,
    Errors = [new PipelineMessage(template, message)],
};
```

## The middleware owns the response

The provider returns only an [`AuthorizationStatus`](../../Abstractions/docs/4.AuthorizationStatus.md). The
middleware turns a non-allowed status into the actual response: it picks the stable code and localization
template (see [Message templates](3.MessageTemplates.md)), resolves the text, and sets `StatusHint`.

| Decision | Code / template | `StatusHint` |
| --- | --- | --- |
| `Unauthenticated` | `AUTHENTICATION_REQUIRED` | `401` |
| `Denied` | `ACCESS_DENIED` | `403` |

## Localization

The message is localized through [`IMessageLocalization`](../../../../Modules/Localization/Abstraction/docs/1.IMessageLocalization.md)
when one is registered in DI (resolved from `context.RequestServices` with `GetService`, so it is optional).
When no localizer is registered, the template key itself is used as the fallback message text — the same
pattern the [exception-handling middleware](../../../../Modules/Middlewares/ExceptionHandlingMiddleware/docs/1.ExceptionHandlingMiddleware.md)
uses. The `PipelineMessage.Code` is always the stable template key.

## No security options

The middleware holds **no** configuration. Whether a request is anonymous or permitted is decided entirely by
the provider, so there is no second place that could accidentally weaken access rules.

## Placement

Register services with [`AddRaccoonLandAuthorization`](2.Registration.md), then add it to the pipeline:

```csharp
pipeline.UseMiddleware<AuthorizationMiddleware>();
```

Place it **after** exception handling and **before** validation and the terminal endpoint, on both the command
and the query pipeline. The middleware is a **singleton** and **stateless**; the provider is resolved per
request from `PipelineContext.RequestServices`.

## Run-time dependencies

| Dependency | Required | Role |
| --- | --- | --- |
| `IAuthorizationProvider` | Yes | The active authorization decision-maker (registered by a provider package). |
| `IMessageLocalization` | No | Localizes the message template when registered; otherwise the template key is used as text. |
