# 2. Registration

Register the authorization middleware, then add it to the request pipeline.

- **Package:** `RaccoonLand.Modules.Security.Authorization.Middlewares`
- **Namespace:** `RaccoonLand.Modules.Security.Authorization.Middlewares`

## Surface

```csharp
public static IServiceCollection AddRaccoonLandAuthorization(this IServiceCollection services);
```

Registers `AuthorizationMiddleware` as a singleton (via `TryAddSingleton`).

## Usage

```csharp
// 1) Register the middleware
services.AddRaccoonLandAuthorization();

// 2) Register an IAuthorizationProvider (for example the claim-based provider) — required
services.AddRaccoonLandClaimAuthorization(options =>
{
    options.AllowAnonymous("MyApp.People.Queries.SearchPeopleQuery");
    options.RequireClaim("MyApp.People.Commands.CreatePersonCommand", "permission", "people.write");
});

// 3) Add it to the pipelines
services.AddRaccoonLandRequestProcessing(
    configureCommandPipeline: pipeline =>
    {
        pipeline.UseMiddleware<ExceptionHandlingMiddleware>();
        pipeline.UseMiddleware<AuthorizationMiddleware>();
        pipeline.UseMiddleware<FluentValidationMiddleware>();
    },
    configureQueryPipeline: pipeline =>
    {
        pipeline.UseMiddleware<ExceptionHandlingMiddleware>();
        pipeline.UseMiddleware<AuthorizationMiddleware>();
        pipeline.UseMiddleware<FluentValidationMiddleware>();
    },
    typeof(CreatePersonCommand).Assembly);
```

## Provider is mandatory

An `IAuthorizationProvider` **must** be registered separately — the middleware package does not pick a
provider and does **not** validate presence at startup. If no provider is registered, the failure appears on
the first request that hits `AuthorizationMiddleware` as a DI resolution exception
(`GetRequiredService<IAuthorizationProvider>`), not as `Denied`.

See also: [AuthorizationMiddleware](1.AuthorizationMiddleware.md).
