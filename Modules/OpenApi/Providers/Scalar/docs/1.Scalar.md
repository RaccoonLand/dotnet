# 1. Scalar provider

`RaccoonLand.Modules.OpenApi.Scalar` renders the **Scalar** API reference for the OpenAPI document produced by
`RaccoonLand.Modules.OpenApi`. It is a UI provider only — it does **not** generate the document.

- **Package:** `RaccoonLand.Modules.OpenApi.Scalar`
- **Namespace:** `RaccoonLand.Modules.OpenApi.Scalar`
- **Depends on:** `Scalar.AspNetCore`, `RaccoonLand.Modules.OpenApi.Abstractions`

## How it works

The provider reads `IOptions<OpenApiDocumentOptions>` (registered by the core's `AddRaccoonLandOpenApi`) and
maps the Scalar reference (`MapScalarApiReference`), pointing it at the configured document route via
`WithOpenApiRoutePattern` and setting the title via `WithTitle`.

```
OpenApiDocumentOptions.RoutePattern → "/openapi/{documentName}.json" → Scalar reference
```

Scalar's default expected route (`/openapi/{documentName}.json`) already matches the native
`Microsoft.AspNetCore.OpenApi` route, so the default configuration works out of the box.

## Composition requirement

This package only supplies the UI. The host must also reference and register the core package
(`RaccoonLand.Modules.OpenApi`) so the document exists and `OpenApiDocumentOptions` is configured.

See [Registration](2.Registration.md).
