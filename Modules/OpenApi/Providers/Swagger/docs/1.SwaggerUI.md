# 1. Swagger UI provider

`RaccoonLand.Modules.OpenApi.Swagger` renders **Swagger UI** for the OpenAPI document produced by
`RaccoonLand.Modules.OpenApi`. It is a UI provider only — it does **not** generate the document.

- **Package:** `RaccoonLand.Modules.OpenApi.Swagger`
- **Namespace:** `RaccoonLand.Modules.OpenApi.Swagger`
- **Depends on:** `Swashbuckle.AspNetCore.SwaggerUI`, `RaccoonLand.Modules.OpenApi.Abstractions`

## How it works

The provider reads `IOptions<OpenApiDocumentOptions>` (registered by the core's `AddRaccoonLandOpenApi`) to
discover the document route and title, then mounts Swagger UI pointing at that JSON endpoint. It uses only
`Swashbuckle.AspNetCore.SwaggerUI` — not SwaggerGen — because the document is generated by the native
`Microsoft.AspNetCore.OpenApi` pipeline.

```
OpenApiDocumentOptions.RoutePattern → "/openapi/v1.json" → Swagger UI endpoint
```

## Composition requirement

This package only supplies the UI. The host must also reference and register the core package
(`RaccoonLand.Modules.OpenApi`) so the document exists and `OpenApiDocumentOptions` is configured. Without it
the UI has no document to display.

See [Registration](2.Registration.md).
