# 2. OpenApiSecurityOptions

`OpenApiSecurityOptions` describes the JWT Bearer security scheme that the core package adds to the OpenAPI
document so a UI (Swagger UI / Scalar) shows an Authorize dialog. It only **documents** the scheme — it does
not configure authentication on the server (see `RaccoonLand.Modules.Security.Authentication` for that).

- **Package:** `RaccoonLand.Modules.OpenApi.Abstractions`
- **Namespace:** `RaccoonLand.Modules.OpenApi.Abstractions`

## Surface

```csharp
public sealed class OpenApiSecurityOptions
{
    public bool EnableJwtBearer { get; set; }
    public string SchemeName { get; set; } = "Bearer";
    public string BearerFormat { get; set; } = "JWT";
    public string Description { get; set; } =
        "JWT Authorization header using the Bearer scheme. Example: \"Bearer {token}\"";
    public bool ApplyGlobally { get; set; } = true;
}
```

| Member | Description |
| --- | --- |
| `EnableJwtBearer` | When `true`, a JWT Bearer security scheme is added to the document. |
| `SchemeName` | Security scheme name registered in the document components (default `Bearer`). Must be non-empty when `EnableJwtBearer` is `true`. |
| `BearerFormat` | Token format hint shown in the UI (default `JWT`). |
| `Description` | Human-readable description shown in the UI Authorize dialog. |
| `ApplyGlobally` | When `true` (default), a global security requirement referencing the scheme is added so every operation requires the token in the UI. |

## Validation

When JWT Bearer is enabled, `AddRaccoonLandOpenApi` rejects a null/empty/whitespace `SchemeName`. The document
transformer also fails fast if a component with the same name is already present (no silent overwrite).

## Relationship to authentication

This type produces only OpenAPI metadata. Enabling it does not validate tokens. Pair it with real
authentication middleware if the API must reject unauthenticated requests.
