# 1. OpenApiDocumentOptions

`OpenApiDocumentOptions` is the shared configuration contract for the OpenAPI module family. It is bound from
the `OpenApi` configuration section by the core package and read by the UI provider packages (Swagger UI /
Scalar) so the UI knows where the document is served and how to title it.

- **Package:** `RaccoonLand.Modules.OpenApi.Abstractions`
- **Namespace:** `RaccoonLand.Modules.OpenApi.Abstractions`
- **Dependencies:** none (plain options; no ASP.NET Core or OpenAPI library reference)

## Surface

```csharp
public sealed class OpenApiDocumentOptions
{
    public const string SectionName = "OpenApi";

    public bool Enabled { get; set; } = true;
    public string DocumentName { get; set; } = "v1";
    public string Title { get; set; } = "API";
    public string Version { get; set; } = "v1";
    public string? Description { get; set; }
    public string RoutePattern { get; set; } = "/openapi/{documentName}.json";
    public OpenApiSecurityOptions Security { get; set; } = new();
}
```

| Member | Description |
| --- | --- |
| `SectionName` | Default configuration section name (`OpenApi`). |
| `Enabled` | When `false`, document generation and mapping become no-ops (options snapshot is still registered). |
| `DocumentName` | OpenAPI document name; must be non-empty. Substituted for the `{documentName}` token. |
| `Title` | Document title shown in the UI. |
| `Version` | OpenAPI document version label. |
| `Description` | Optional API description. |
| `RoutePattern` | Route where the generated JSON is served. **Must** contain `{documentName}` so `AddOpenApi(DocumentName)`, `MapOpenApi(RoutePattern)`, and UI providers stay aligned. |
| `Security` | JWT Bearer security definition — see [OpenApiSecurityOptions](2.OpenApiSecurityOptions.md). Must not be `null`. |

## DocumentName / RoutePattern contract

`AddRaccoonLandOpenApi` validates at registration:

- `DocumentName` and `RoutePattern` are non-empty.
- `RoutePattern` contains the literal `{documentName}` token (ordinal).

This prevents combinations such as `DocumentName = "v2"` with a hard-coded `/openapi/v1.json` route that would
leave the UI pointing at the wrong document.

## Why it lives in Abstractions

The UI provider packages need only the document route and title — not the document generator. Keeping the
options in a dependency-free package lets Swagger UI and Scalar reference a small, stable contract while the
core package (which depends on `Microsoft.AspNetCore.OpenApi`) owns generation.

## Example configuration

```json
{
  "OpenApi": {
    "Enabled": true,
    "DocumentName": "v1",
    "Title": "Capability Centric Sample API",
    "Version": "v1",
    "Description": "Sample API built on RaccoonLand.",
    "RoutePattern": "/openapi/{documentName}.json",
    "Security": { "EnableJwtBearer": true }
  }
}
```
