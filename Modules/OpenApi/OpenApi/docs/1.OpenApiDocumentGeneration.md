# 1. OpenAPI document generation

`RaccoonLand.Modules.OpenApi` is the **core** of the OpenAPI module family. It generates the OpenAPI document
on top of the built-in .NET 10 `Microsoft.AspNetCore.OpenApi` package (`AddOpenApi` / `MapOpenApi`) and serves
the JSON that the UI providers (Swagger UI / Scalar) consume. It does **not** render a UI itself.

- **Package:** `RaccoonLand.Modules.OpenApi`
- **Namespace:** `RaccoonLand.Modules.OpenApi`
- **Depends on:** `Microsoft.AspNetCore.OpenApi` (10.0.x), `RaccoonLand.Modules.OpenApi.Abstractions`

## What it does

| Concern | Behaviour |
| --- | --- |
| Document generation | Registers a document named `OpenApiDocumentOptions.DocumentName` via `AddOpenApi`. |
| Document info | A document transformer sets `Info.Title`, `Info.Version`, and `Info.Description` from options. |
| Security (optional) | When `Security.EnableJwtBearer` is `true`, adds a JWT Bearer scheme — see [BearerSecurityScheme](2.BearerSecurityScheme.md). |
| Endpoint | `MapRaccoonLandOpenApi()` serves the JSON at `OpenApiDocumentOptions.RoutePattern`. |
| Disable switch | When `Enabled` is `false`, both registration and mapping are no-ops. |

## Generation uses the native .NET 10 pipeline

The document is produced by `Microsoft.AspNetCore.OpenApi`, not Swashbuckle. This keeps generation aligned
with the framework and lets any UI consume the standard `/openapi/{documentName}.json` document.

```
AddRaccoonLandOpenApi(configuration)   → AddOpenApi(documentName, transformers…)
MapRaccoonLandOpenApi()                → MapOpenApi(routePattern)   // /openapi/v1.json
```

## Choosing a UI

The core serves only JSON. Add one (or both) provider packages for a UI:

- `RaccoonLand.Modules.OpenApi.Scalar` → `MapRaccoonLandScalar()`
- `RaccoonLand.Modules.OpenApi.Swagger` → `UseRaccoonLandSwaggerUI()`

See [Registration](3.Registration.md) for the full wiring.
