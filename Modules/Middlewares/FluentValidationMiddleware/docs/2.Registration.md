# 2. Registration (AddRaccoonLandFluentValidation)

`AddRaccoonLandFluentValidation` registers the middleware in DI so it can be added to a request pipeline with
`UseMiddleware<FluentValidationMiddleware>()`.

- **Package:** `RaccoonLand.Modules.Middlewares.FluentValidationMiddleware`
- **Namespace:** `RaccoonLand.Modules.Middlewares.FluentValidationMiddleware`

## Surface

```csharp
public static IServiceCollection AddRaccoonLandFluentValidation(this IServiceCollection services);
```

## What it registers

| Service | Lifetime | Description |
| --- | --- | --- |
| `FluentValidationMiddleware` | Singleton | Stateless; resolved once at pipeline build time. Validators are discovered from `PipelineContext.RequestServices` per request. |

## Full setup example

Validators are **not** registered by this extension — use FluentValidation's own DI helpers:

```csharp
builder.Services.AddRaccoonLandFluentValidation();
builder.Services.AddValidatorsFromAssembly(typeof(Program).Assembly);

builder.Services.AddRaccoonLandRequestProcessing(
    configureCommandPipeline: pipeline => pipeline
        .UseMiddleware<ExceptionHandlingMiddleware>()
        .UseMiddleware<FluentValidationMiddleware>(),
    configureQueryPipeline: pipeline => pipeline
        .UseMiddleware<ExceptionHandlingMiddleware>(),
    typeof(Program).Assembly);
```

## Package dependencies

| Package | Purpose |
| --- | --- |
| `FluentValidation` | Validator interfaces and execution. |
| `RaccoonLand.Core.RequestProcessing.Abstractions` | `IPipelineMiddleware`, `PipelineContext`, `PipelineResponse`. |

No ASP.NET Core or MessageLocalization packages are required by this middleware.

See also: [FluentValidationMiddleware](1.FluentValidationMiddleware.md).
