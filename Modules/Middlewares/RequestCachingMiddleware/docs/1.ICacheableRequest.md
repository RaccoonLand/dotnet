# 1. ICacheableRequest

`ICacheableRequest` is the opt-in marker for CQRS requests whose **response** can be cached by
[`RequestCachingMiddleware`](2.RequestCachingMiddleware.md). Feature modules that define queries reference this
package only — no pipeline, `IDistributedCache`, or middleware dependencies.

- **Package:** `RaccoonLand.Modules.Middlewares.RequestCaching.Abstraction`
- **Namespace:** `RaccoonLand.Modules.Middlewares.RequestCaching.Abstraction`
- **Folder:** `src/Abstraction/`

## Surface

```csharp
public interface ICacheableRequest
{
    /// <summary>
    /// Stable cache key for this request instance. Namespaced by request type in the middleware.
    /// </summary>
    string GetCacheKey();
}
```

## Usage

Implement on a query (or any `IRequest<TResponse>`) together with your CQRS marker:

```csharp
using RaccoonLand.Core.RequestProcessing.Abstractions.Cqrs;
using RaccoonLand.Modules.Middlewares.RequestCaching.Abstraction;

public sealed record GetProductByIdQuery(Guid Id)
    : IQuery<GetProductByIdResponse>, ICacheableRequest
{
    public string GetCacheKey() => Id.ToString();
}
```

## Contract

| Responsibility | Owner |
| --- | --- |
| Build a stable per-instance key | Request author (`GetCacheKey`) |
| Namespace key by request type | `RequestCachingMiddleware` |
| Resolve TTL per request category | [`RequestCachingOptions`](3.RequestCachingOptions.md) |
| Serialize/deserialize `PipelineResponse` | `RequestCachingMiddleware` (`System.Text.Json`) |

Requests that do **not** implement `ICacheableRequest` pass through the middleware unchanged.

See also: [RequestCachingMiddleware](2.RequestCachingMiddleware.md), [Registration](4.Registration.md).
