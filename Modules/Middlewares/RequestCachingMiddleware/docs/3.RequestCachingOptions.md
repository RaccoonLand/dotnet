# 3. RequestCachingOptions

Configuration for per-category cache lifetimes used by
[`RequestCachingMiddleware`](2.RequestCachingMiddleware.md).

- **Package:** `RaccoonLand.Modules.Middlewares.RequestCachingMiddleware`
- **Namespace:** `RaccoonLand.Modules.Middlewares.RequestCachingMiddleware`

## Surface

```csharp
public sealed class RequestCachingOptions
{
    public const string SectionName = "RequestCaching";

    public RequestCacheEntryOptions Default { get; set; } = new();

    public IDictionary<string, RequestCacheEntryOptions> Overrides { get; set; }
        = new Dictionary<string, RequestCacheEntryOptions>(StringComparer.OrdinalIgnoreCase);
}

public sealed class RequestCacheEntryOptions
{
    public TimeSpan Duration { get; set; } = TimeSpan.FromMinutes(5);
}
```

## Validation

`AddRaccoonLandRequestCaching` validates on start that `Default.Duration` and every `Overrides` entry
`Duration` is **greater than** `TimeSpan.Zero`. Invalid configuration fails fast at host startup.

## Override resolution

Override keys are matched against the request type's **full name** (`Namespace.TypeName`), using the same
longest-prefix-wins rule as logging categories:

| Request type full name | Override key | Matches? |
| --- | --- | --- |
| `MyApp.Features.Products.GetProductByIdQuery` | `MyApp.Features.Products` | Yes (prefix) |
| `MyApp.Features.Products.GetProductByIdQuery` | `MyApp.Features.Products.GetProductByIdQuery` | Yes (exact, wins over shorter prefix) |
| `MyApp.Features.Orders.GetOrderQuery` | `MyApp.Features.Products` | No |

When no override matches, `Default` is used.

The middleware reads options via `IOptionsMonitor<RequestCachingOptions>.CurrentValue`, so reloads of the
bound configuration are visible without restarting the process (subject to the host's options reload setup).

## Expiration semantics

`Duration` maps to **absolute** expiration (`AbsoluteExpirationRelativeToNow`). Sliding expiration is intentionally
not used: cache entries are written **only on a miss**, so repeated reads never extend TTL.

## appsettings example

```json
{
  "RequestCaching": {
    "Default": { "Duration": "00:05:00" },
    "Overrides": {
      "MyApp.Features.Products": { "Duration": "00:00:30" },
      "MyApp.Features.Products.GetProductByIdQuery": { "Duration": "00:02:00" }
    }
  }
}
```

Bind via [`AddRaccoonLandRequestCaching`](4.Registration.md) with `IConfiguration`, or configure in code.

See also: [RequestCachingMiddleware](2.RequestCachingMiddleware.md), [Registration](4.Registration.md).
