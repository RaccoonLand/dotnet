# 2. RequestCachingMiddleware

`RequestCachingMiddleware` is a class-based [`IPipelineMiddleware`](../../../Core/RequestProcessing/RequestProcessing.Abstractions/docs/4.PipelineDelegate.md)
that adds **read-through** response caching for requests implementing
[`ICacheableRequest`](1.ICacheableRequest.md).

- **Package:** `RaccoonLand.Modules.Middlewares.RequestCachingMiddleware`
- **Namespace:** `RaccoonLand.Modules.Middlewares.RequestCachingMiddleware`

## Host independence

The middleware has **no dependency on ASP.NET Core**. It reads and writes via `IDistributedCache` and assigns
`context.Response` on a cache hit. Any host (API, worker) that runs the RaccoonLand pipeline can use it.

## Behaviour

1. If the request does not implement `ICacheableRequest`, call `next` and return.
2. Resolve whether the request implements `IRequest<TResponse>`. If not (e.g. void command), call `next` and return.
3. Build cache key: `{RequestType.FullName}:{GetCacheKey()}`.
4. **Hit** — deserialize a non-null `PipelineResponse`, set `context.Response`, **skip** the rest of the pipeline.
   A null deserialize result is treated as a **miss** (logged as a warning) and the corrupt entry is removed
   best-effort so later requests do not keep re-reading it.
5. **Miss** — call `next`; if `context.Response` is non-null **and** has no `Errors`, serialize and store with
   absolute expiration from [`RequestCachingOptions`](3.RequestCachingOptions.md). Responses with errors are
   not cached so transient failures are not replayed.

Cache infrastructure failures (read, write, or corrupt-entry removal) are logged as warnings and the request
continues as if caching were disabled (**fail-open**). `OperationCanceledException` for the request cancellation
token is **rethrown** and is not swallowed by fail-open.

## Storage

Uses [`IDistributedCache`](https://learn.microsoft.com/en-us/dotnet/api/microsoft.extensions.caching.distributed.idistributedcache).
Register any provider in DI (`AddDistributedMemoryCache`, Redis, SQL Server, …).

Serialization uses `System.Text.Json` for the full [`PipelineResponse`](../../../Core/RequestProcessing/RequestProcessing.Abstractions/docs/7.PipelineResponse.md) envelope (including `Result`, `Errors`, and `Warnings`).

## Pipeline placement

Register with [`AddRaccoonLandRequestCaching`](4.Registration.md), then:

```csharp
pipeline.UseMiddleware<RequestCachingMiddleware>();
```

Place it **before** handlers that perform expensive reads and **after** middleware that must always run (for
example validation on cache misses only — on a hit validation is skipped). Typical order for read queries:

```
ExceptionHandling → FluentValidation → RequestCaching → Handler
```

On a cache hit, downstream middleware and the handler are not invoked.

## Dependencies at run time

| Dependency | Required | Role |
| --- | --- | --- |
| `IDistributedCache` | Yes | Cache storage (consumer registers). |
| `IOptionsMonitor<RequestCachingOptions>` | Yes | Default and per-category TTL (`CurrentValue` per use). |
| `ILogger<RequestCachingMiddleware>` | No | Warning logs on cache failures / null deserialize. |

The middleware is registered as a **singleton**. It is **stateless** with respect to request data; TTL is read
from `IOptionsMonitor.CurrentValue` so configuration reloads apply. See the RaccoonLand pipeline middleware
contract in [PipelineDelegate & IPipelineMiddleware](../../../Core/RequestProcessing/RequestProcessing.Abstractions/docs/4.PipelineDelegate.md).

See also: [ICacheableRequest](1.ICacheableRequest.md), [RequestCachingOptions](3.RequestCachingOptions.md), [Registration](4.Registration.md).
