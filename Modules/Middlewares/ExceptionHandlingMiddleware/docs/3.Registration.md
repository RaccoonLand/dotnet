# 3. Registration (AddRaccoonLandExceptionHandling)

`AddRaccoonLandExceptionHandling` registers the middleware and its options in DI so it can be added to a
request pipeline with `UseMiddleware<ExceptionHandlingMiddleware>()`.

- **Package:** `RaccoonLand.Modules.Middlewares.ExceptionHandlingMiddleware`
- **Namespace:** `RaccoonLand.Modules.Middlewares.ExceptionHandlingMiddleware`

## Surface

```csharp
public static IServiceCollection AddRaccoonLandExceptionHandling(
    this IServiceCollection services,
    Action<ExceptionHandlingOptions>? configure = null);
```

| Parameter | Description |
| --- | --- |
| `services` | The service collection. |
| `configure` | Optional callback to register custom handlers on [`ExceptionHandlingOptions`](2.ExceptionHandlingOptions.md). |

## What it registers

| Service | Lifetime | Description |
| --- | --- | --- |
| `IOptions<ExceptionHandlingOptions>` | Options pattern | Holds custom handler registrations. |
| `ExceptionHandlingMiddleware` | Singleton | Stateless; resolved once at pipeline build time from `ApplicationServices`. Request-scoped services (e.g. `IMessageLocalization`) are obtained from `PipelineContext.RequestServices` inside `InvokeAsync`. |

## Full setup example

```csharp
// 1. Register middleware services
builder.Services.AddRaccoonLandExceptionHandling(options =>
{
    options.On<MyCustomException>(HandleMyCustomException);
});

// 2. Add to command/query pipeline (via RequestProcessing)
builder.Services.AddRaccoonLandRequestProcessing(
    configureCommandPipeline: pipeline => pipeline
        .UseMiddleware<ExceptionHandlingMiddleware>(),
    configureQueryPipeline: pipeline => pipeline
        .UseMiddleware<ExceptionHandlingMiddleware>(),
    typeof(Program).Assembly);
```
