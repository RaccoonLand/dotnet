# 1. ExceptionHandlingMiddleware

`ExceptionHandlingMiddleware` is a class-based [`IPipelineMiddleware`](../../../Core/RequestProcessing/RequestProcessing.Abstractions/docs/4.PipelineDelegate.md)
that intercepts exceptions thrown by downstream pipeline components and converts handled ones into a
[`PipelineResponse`](../../../Core/RequestProcessing/RequestProcessing.Abstractions/docs/7.PipelineResponse.md)
error envelope on `context.Response`.

- **Package:** `RaccoonLand.Modules.Middlewares.ExceptionHandlingMiddleware`
- **Namespace:** `RaccoonLand.Modules.Middlewares.ExceptionHandlingMiddleware`

## Host independence

Unlike the legacy implementation, this middleware has **no dependency on ASP.NET Core**. It does not set HTTP
status codes or write `ProblemDetails` — it only assigns `context.Response`. A host adapter (API, worker) maps
that envelope to transport-specific output (for example HTTP 400 when `Errors` is non-empty).

## Resolution order

When `next(context)` throws:

1. **Custom handlers** — registered via [`ExceptionHandlingOptions.On<T>`](2.ExceptionHandlingOptions.md), in
   registration order. The first handler that returns `true` wins.
2. **`DomainException`** — built-in handling: localizes each [`DomainError`](../../../Core/Domain/docs/7.DomainError.md)
   in `exception.Errors` through
   [`IMessageLocalization`](../../Localization/Abstraction/docs/1.IMessageLocalization.md) when available,
   otherwise uses each error's `Message` as fallback text. Sets `context.Response.Errors` with one
   [`PipelineMessage`](../../../Core/RequestProcessing/RequestProcessing.Abstractions/docs/7.PipelineResponse.md)
   per domain error.
3. **Any other exception** — **rethrown**. There is no generic catch-all handler and no HTTP 500 fallback.

```csharp
catch (Exception exception)
{
    if (await TryHandleWithCustomAsync(context, exception)) return;

    if (exception is DomainException domainException)
    {
        HandleDomainException(context, domainException);
        return;
    }

    throw;
}
```

## Adding to a pipeline

Register services with [`AddRaccoonLandExceptionHandling`](3.Registration.md), then:

```csharp
pipeline.UseMiddleware<ExceptionHandlingMiddleware>();
```

Place it **outermost** (first in the pipeline) so it wraps all downstream middleware and the terminal endpoint.

The middleware is registered as a **singleton** and is expected to be **stateless** — see the RaccoonLand
pipeline middleware contract in
[PipelineDelegate & IPipelineMiddleware](../../../Core/RequestProcessing/RequestProcessing.Abstractions/docs/4.PipelineDelegate.md).
`IMessageLocalization` is resolved from `context.RequestServices` at run time, not injected as a scoped
constructor dependency.

## Dependencies at run time

| Dependency | Required | Role |
| --- | --- | --- |
| `IOptions<ExceptionHandlingOptions>` | Yes | Custom handler registrations. |
| `IMessageLocalization` | No | Localizes each `DomainError.Message` template key when registered in DI. |

See also: [ExceptionHandlingOptions](2.ExceptionHandlingOptions.md), [Registration](3.Registration.md).
