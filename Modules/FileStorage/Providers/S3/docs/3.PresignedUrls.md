# 3. PresignedUrls

Use `IFileUrlSigner` for direct client uploads and downloads:

```csharp
var signer = serviceProvider.GetRequiredService<IFileUrlSigner>();

var write = await signer.GetWriteUrlAsync(new SignedWriteUrlRequest
{
    ContentType = "image/png",
}, cancellationToken);

var read = await signer.GetReadUrlAsync(new SignedReadUrlRequest
{
    Key = write.File.Key,
}, cancellationToken);
```

Store `write.File` (`FileRef`) in your application database after the client finishes uploading to the signed URL.

**S3 GET vs PUT signing:** write URLs include `Content-Type` in the signature (client must send that header). Read URLs do not — typical download clients omit `Content-Type`, which would break a signed-header lock with `403 SignatureDoesNotMatch`. `SignedReadUrlRequest.ContentType` is ignored by the S3 signer; the returned read `FileRef` contains only `Key`.

For large files, combine this with `IMultipartUploadCoordinator`.
