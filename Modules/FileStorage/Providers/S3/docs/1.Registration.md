# 1. Registration

Register HTTP-based S3 storage:

```csharp
services.AddRaccoonLandS3FileStorage(configuration);
```

This registers:

- `IFileStorage`
- `IFileUrlSigner`
- `IMultipartUploadCoordinator`

Configuration:

```json
{
  "FileStorage:S3": {
    "Bucket": "raccoonland-files",
    "Region": "us-east-1",
    "Flavor": "Aws",
    "Credentials": {
      "AccessKeyId": "your-access-key",
      "SecretAccessKey": "your-secret-key"
    }
  }
}
```

MinIO example:

```json
{
  "FileStorage:S3": {
    "Bucket": "files",
    "Region": "us-east-1",
    "Endpoint": "https://localhost:9000",
    "Flavor": "Minio",
    "Credentials": {
      "AccessKeyId": "minio",
      "SecretAccessKey": "minio123"
    }
  }
}
```

## Put behaviour notes

- **`PutMode.CreateOnly`** on `IFileStorage.PutAsync` uses the S3 precondition header `If-None-Match: *` so create races resolve at the service (HTTP 412 → `FileAlreadyExistsStorageException`). No separate HEAD check is performed.
- **CreateOnly applies only to simple PUT.** Multipart (`IMultipartUploadCoordinator`) does not take `PutMode` and does not send `If-None-Match`. Initiate/`Complete` do not provide the same atomic create guarantee. If the object must not already exist, the application should generate a new key, or call `ExistsAsync` before `InitiateAsync` as a best-effort check (still racy).
- **`ContentLength`** is required for simple PUT and multipart part uploads when the stream is not seekable. Prefer setting `PutFileRequest.ContentLength` / `UploadPartRequest.ContentLength`; seekable streams use remaining length automatically. A seekable stream whose position is past `Length` fails validation instead of uploading an empty object.
- **`CompleteAsync` `FileRef`** typically omits `ContentType` and `Length` — S3 Complete Multipart does not return them, and `CompleteMultipartUploadRequest` does not accept a content type (MIME is locked at Initiate). Persist content type from Initiate in application state, and/or call `IFileStorage.GetMetadataAsync` when needed.

See also: [S3Flavor](2.S3Flavor.md), [PresignedUrls](3.PresignedUrls.md).
