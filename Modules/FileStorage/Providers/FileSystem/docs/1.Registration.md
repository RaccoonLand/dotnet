# 1. Registration

Register local disk file storage as the active `IFileStorage` provider.

- **Package:** `RaccoonLand.Modules.FileStorage.FileSystem`
- **Namespace:** `Microsoft.Extensions.DependencyInjection` (extension methods)
- **Source:** `src/DependencyInjection/FileSystemFileStorageServiceCollectionExtensions.cs`

## Surface

```csharp
public static IServiceCollection AddRaccoonLandFileSystemStorage(
    this IServiceCollection services,
    IConfiguration configuration,
    string sectionName = FileSystemStorageOptions.SectionName,
    string sharedSectionName = FileStorageOptions.SectionName);

public static IServiceCollection AddRaccoonLandFileSystemStorage(
    this IServiceCollection services,
    IConfiguration configuration,
    Action<FileSystemStorageOptions> configure,
    string sectionName = FileSystemStorageOptions.SectionName,
    string sharedSectionName = FileStorageOptions.SectionName);
```

| Parameter | Default | Binds |
|-----------|---------|-------|
| `sectionName` | `FileStorage:FileSystem` | `FileSystemStorageOptions` |
| `sharedSectionName` | `FileStorage` | `FileStorageOptions` (shared guardrails) |

Custom section names:

```csharp
services.AddRaccoonLandFileSystemStorage(
    configuration,
    sectionName: "App:Uploads:FileSystem",
    sharedSectionName: "App:Uploads");
```

## What gets registered

| Service | Implementation |
|---------|----------------|
| `IFileStorage` | `FileSystemFileStorage` (singleton) |
| `FileStorageOptions` | Bound from `FileStorage` section |
| `FileSystemStorageOptions` | Bound from `FileStorage:FileSystem` section |

**Not registered:** `IFileUrlSigner`, `IMultipartUploadCoordinator`. Use the [S3 provider](../S3/docs/1.Registration.md) for direct client uploads.

## Configuration

```json
{
  "FileStorage": {
    "MaxUploadBytes": 104857600,
    "AllowedContentTypes": [ "image/jpeg", "image/png", "application/pdf" ]
  },
  "FileStorage:FileSystem": {
    "RootPath": "C:\\storage\\files"
  }
}
```

Linux / container example:

```json
{
  "FileStorage:FileSystem": {
    "RootPath": "/var/lib/app/files"
  }
}
```

## Code-first configuration

```csharp
builder.Services.AddRaccoonLandFileSystemStorage(builder.Configuration, options =>
{
    options.RootPath = Path.Combine(builder.Environment.ContentRootPath, "uploads");
});
```

## When to use

| Scenario | Recommendation |
|----------|----------------|
| Local development | FileSystem |
| Single-node on-prem with shared disk | FileSystem |
| Production multi-instance | S3 or shared volume with care |
| Direct browser upload without proxy | S3 (presigned URLs) |

## Application usage

Use the same `IFileStorage` contract as other providers. Prefer [FileStoragePutHelper](../../Abstractions/docs/11.FilePutConstraints.md) for per-endpoint content type guardrails.

```csharp
builder.Services.AddRaccoonLandFileSystemStorage(builder.Configuration);
```

See also: [Storage layout](2.StorageLayout.md), [Configuration and limitations](3.ConfigurationAndLimitations.md).
