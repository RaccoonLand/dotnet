# 2. Schema

The FILESTREAM table requires a pre-created FILESTREAM filegroup. Example DBA script:

```sql
ALTER DATABASE AppDb ADD FILEGROUP FileStreamGroup CONTAINS FILESTREAM;

ALTER DATABASE AppDb
ADD FILE (
    NAME = FileStreamData,
    FILENAME = N'D:\SqlData\AppDbFileStream'
) TO FILEGROUP FileStreamGroup;
```

Table:

```sql
CREATE TABLE [app].[FileBlobStreams] (
    [Key] NVARCHAR(128) NOT NULL PRIMARY KEY,
    [BlobId] UNIQUEIDENTIFIER ROWGUIDCOL NOT NULL UNIQUE DEFAULT NEWSEQUENTIALID(),
    [Content] VARBINARY(MAX) FILESTREAM NULL,
    [ContentType] NVARCHAR(128) NULL,
    [Length] BIGINT NOT NULL,
    [ChecksumSha256] NVARCHAR(64) NULL,
    [CreatedAtUtc] DATETIMEOFFSET NOT NULL,
    [Metadata] NVARCHAR(MAX) NULL
) FILESTREAM_ON [FileStreamGroup];
```

Default `SchemaName` is `dbo`. Set both `SchemaName` and `TableName` in configuration to control the qualified object name.

When `EnsureSchema` is `true`, the provider creates the schema (if missing) and table only when the FILESTREAM filegroup already exists.

When `EnsureSchema` is `true` but the connection lacks DDL permission or the filegroup is missing, the provider raises `FileStorageConfigurationException` with an actionable message instead of surfacing a raw `SqlException`. That configuration failure is cached so later requests do not repeat the DDL attempt.

## Put modes

| Mode | Behaviour |
|------|-----------|
| `CreateOnly` | Insert a new row; fail when the key exists |
| `Overwrite` | Rewrite the FILESTREAM payload in place; fail when the key is missing |
| `Upsert` | Insert when missing; otherwise rewrite in place |

`CreatedAtUtc` is set only on insert. Overwrite and upsert updates preserve the original creation timestamp, matching the FileSystem and SqlServer VARBINARY providers.

Concurrent upserts on the same new key may hit a primary-key conflict; the provider retries as an in-place update.
