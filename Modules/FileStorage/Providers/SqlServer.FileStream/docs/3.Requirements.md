# 3. Requirements

| Requirement | Details |
| --- | --- |
| Host OS | Windows only |
| SQL Server | FILESTREAM feature enabled |
| Filegroup | FILESTREAM filegroup created by a DBA |
| Transactions | Reads and writes use `SqlFileStream` inside SQL transactions |
| URL signing | Not supported — use S3 provider for direct client upload |

## Operational notes

Put operations insert a row, stream bytes through `SqlFileStream`, then update length and checksum inside one SQL transaction. If the application crashes before `Commit`, SQL Server rolls the transaction back and the row disappears. FILESTREAM may still leave orphaned container files on disk until SQL Server's own FILESTREAM garbage collection runs during checkpoint/maintenance. Monitor database FILESTREAM disk usage over time; this is inherent to FILESTREAM rather than a provider defect.

Applications still persist provider-neutral `FileRef` values only.
