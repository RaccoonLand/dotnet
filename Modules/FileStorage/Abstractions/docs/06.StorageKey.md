# 6. StorageKey

`StorageKey` generates and validates opaque storage keys used by all providers.

- **Package:** `RaccoonLand.Modules.FileStorage.Abstractions`
- **Namespace:** `RaccoonLand.Modules.FileStorage.Abstractions`
- **Source:** `src/Keys/StorageKey.cs`

## Surface

```csharp
public static partial class StorageKey
{
    public const int MaxLength = 128;
    public const int MinLength = 8;

    public static string Generate();
    public static bool IsValid(string? key);
    public static string Normalize(string key);
    public static string NormalizeOrGenerate(string? key);
}
```

## Key format

- Prefix: `file_`
- Generated keys use GUID version 7 (`file_<32 hex chars>`)
- Allowed characters: `a-z`, `A-Z`, `0-9`, `.`, `_`, `-`
- Length: 8–128 characters
- Path traversal (`..`) is rejected

## Usage

**Let the provider generate**

```csharp
var result = await fileStorage.PutAsync(new PutFileRequest
{
    Key = null,  // provider calls StorageKey.NormalizeOrGenerate
    Content = stream,
});
```

**Caller-supplied key**

```csharp
var key = StorageKey.Normalize(userSuppliedKey);
```

**Validation only**

```csharp
if (!StorageKey.IsValid(key))
{
    throw new FileStorageValidationException("Invalid file key.");
}
```

## Design principles

1. **Opaque identifiers.** Keys are not file paths, URLs, or human-readable names.
2. **Provider-neutral.** The same validation rules apply regardless of backend.
3. **Safe characters only.** Prevents injection into filesystem sharding or SQL identifiers inside providers.

Invalid keys raise [FileStorageValidationException](8.Exceptions.md).
