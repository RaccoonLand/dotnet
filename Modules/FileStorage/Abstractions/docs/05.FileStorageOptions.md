# 5. FileStorageOptions

`FileStorageOptions` holds shared limits and signed-URL defaults for all file storage providers.

- **Package:** `RaccoonLand.Modules.FileStorage.Abstractions`
- **Namespace:** `RaccoonLand.Modules.FileStorage.Abstractions`
- **Source:** `src/Configuration/FileStorageOptions.cs`

## Surface

```csharp
public sealed class FileStorageOptions
{
    public const string SectionName = "FileStorage";

    public long? MaxUploadBytes { get; set; }
    public HashSet<string>? AllowedContentTypes { get; set; }
    public int MaxMetadataEntries { get; set; } = 20;
    public int MaxMetadataValueLength { get; set; } = 256;
    public TimeSpan DefaultSignedUrlExpiry { get; set; } = TimeSpan.FromMinutes(15);
    public TimeSpan MaxSignedUrlExpiry { get; set; } = TimeSpan.FromHours(24);
}
```

## Configuration section

Bind the shared section once; each provider extension method reads it:

```json
{
  "FileStorage": {
    "MaxUploadBytes": 104857600,
    "AllowedContentTypes": [ "image/jpeg", "image/png", "application/pdf" ],
    "MaxMetadataEntries": 20,
    "MaxMetadataValueLength": 256,
    "DefaultSignedUrlExpiry": "00:15:00",
    "MaxSignedUrlExpiry": "24:00:00"
  }
}
```

Provider-specific settings are nested under the same root — see [Providers](10.Providers.md).

## Properties

| Property | Default | Purpose |
|----------|---------|---------|
| `MaxUploadBytes` | `null` (unlimited) | Rejects uploads exceeding the limit via [FileStorageGuards](../src/Validation/FileStorageGuards.cs) |
| `AllowedContentTypes` | `null` (all allowed) | Global infrastructure ceiling. Per-endpoint narrowing uses [FilePutConstraints](11.FilePutConstraints.md). |
| `MaxMetadataEntries` | `20` | Caps user metadata dictionary size |
| `MaxMetadataValueLength` | `256` | Caps each metadata value length |
| `DefaultSignedUrlExpiry` | 15 minutes | Used when signed URL requests omit `Expiry` |
| `MaxSignedUrlExpiry` | 24 hours | Upper bound for requested signed URL lifetime |

## Validation

Providers call `FileStorageGuards.ValidatePutRequest` and `FileStorageGuards.ResolveExpiry` before reaching storage. Violations raise [FileStorageValidationException](8.Exceptions.md).

## Design note

Keep business rules (who may upload what) in the application layer. Use these options for **infrastructure guardrails** shared across all providers.

See also: [IFileStorage](1.IFileStorage.md), [IFileUrlSigner](3.IFileUrlSigner.md), [Providers](10.Providers.md).
