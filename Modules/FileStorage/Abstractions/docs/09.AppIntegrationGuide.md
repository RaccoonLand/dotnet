# 9. App integration guide

How consuming applications should wire and use the FileStorage module.

- **Package:** `RaccoonLand.Modules.FileStorage.Abstractions` (+ one provider package)

## Architecture

```text
┌─────────────────────────────────────┐
│  Application (API / worker)         │
│  • Authorization & ownership        │
│  • Persist FileRef.Key in DB        │
└──────────────┬──────────────────────┘
               │ IFileStorage (+ optional IFileUrlSigner, IMultipartUploadCoordinator)
┌──────────────▼──────────────────────┐
│  Provider (FileSystem / SqlServer / │
│  SqlServer.FileStream / S3)         │
└─────────────────────────────────────┘
```

## Step 1 — Choose a provider

| Scenario | Provider package |
|----------|-----------------|
| Local dev / single-node | `RaccoonLand.Modules.FileStorage.FileSystem` |
| Small blobs, no extra infra | `RaccoonLand.Modules.FileStorage.SqlServer` |
| Large files, Windows + DBA setup | `RaccoonLand.Modules.FileStorage.SqlServer.FileStream` |
| Production, CDN, direct client upload | `RaccoonLand.Modules.FileStorage.S3` |

Details: [Providers](10.Providers.md).

## Step 2 — Register DI

```csharp
// Pick one:
builder.Services.AddRaccoonLandFileSystemStorage(builder.Configuration);
// builder.Services.AddRaccoonLandSqlServerFileStorage(builder.Configuration);
// builder.Services.AddRaccoonLandSqlServerFileStreamStorage(builder.Configuration);
// builder.Services.AddRaccoonLandS3FileStorage(builder.Configuration);
```

Each extension binds [FileStorageOptions](5.FileStorageOptions.md) from the `FileStorage` section and provider-specific nested settings.

## Step 3 — Model ownership in your domain

```csharp
public sealed class Attachment
{
    public Guid Id { get; set; }
    public Guid OwnerUserId { get; set; }
    public string FileKey { get; set; } = default!;  // FileRef.Key only
    public string OriginalFileName { get; set; } = default!;
}
```

Never store bucket names, connection strings, or filesystem paths in domain entities.

## Step 4 — Authorize before storage calls

```csharp
public async Task<IResult> Download(Guid attachmentId, CancellationToken ct)
{
    var attachment = await db.Attachments.FindAsync(attachmentId);
    if (attachment is null || attachment.OwnerUserId != currentUser.Id)
        return Results.NotFound();

    await using var read = await fileStorage.OpenReadAsync(
        new OpenReadRequest { Key = attachment.FileKey }, ct);

    return Results.File(read.Content, contentType: read.File.ContentType);
}
```

The module intentionally has no concept of users, tenants, or roles.

## Step 5 — Upload patterns

**Recommended — use the put helper with per-operation constraints:**

```csharp
var request = FileStoragePutHelper.CreateRequest(
    file.OpenReadStream(),
    file.ContentType,
    FilePutConstraints.Images,           // this endpoint: images only
    contentLength: file.Length,
    options: fileStorageOptions.Value);  // optional early validation against global config

var put = await fileStorage.PutAsync(request, ct);
```

See [FilePutConstraints and FileStoragePutHelper](11.FilePutConstraints.md) for profiles, custom allowlists, and per-operation size limits.

**Server-side proxy (manual request):**

```csharp
var put = await fileStorage.PutAsync(new PutFileRequest
{
    Content = file.OpenReadStream(),
    ContentType = file.ContentType,
    ContentLength = file.Length,
    Mode = PutMode.CreateOnly,
}, ct);

await db.Attachments.AddAsync(new Attachment
{
    FileKey = put.File.Key,
    OwnerUserId = currentUser.Id,
}, ct);
```

**Direct client upload (S3 only)**

1. Authorize the user.
2. Build the request with `FileStoragePutHelper.CreateSignedWriteRequest` or `CreateMultipartInitRequest`.
3. Call `IFileUrlSigner.GetWriteUrlAsync` or `IMultipartUploadCoordinator.InitiateAsync`.
4. After client upload completes, persist `FileRef.Key` from the result.

For very large files, use [IMultipartUploadCoordinator](4.IMultipartUploadCoordinator.md) with `CreateMultipartInitRequest`.

## Step 6 — Error handling

Map [Exceptions](8.Exceptions.md) to application responses after authorization. Retry only `FileStorageUnavailableException`.

## Step 7 — Testing

- **Unit tests:** mock `IFileStorage`.
- **Integration tests:** use FileSystem provider with a temp directory, or S3 with MinIO/Testcontainers.
- **Do not** switch providers per request in production — pick one per deployment.
