# 7. PutMode

`PutMode` controls overwrite behaviour when putting a file at an existing key.

- **Package:** `RaccoonLand.Modules.FileStorage.Abstractions`
- **Namespace:** `RaccoonLand.Modules.FileStorage.Abstractions`
- **Source:** `src/Models/PutMode.cs`

## Surface

```csharp
public enum PutMode
{
    CreateOnly,
    Overwrite,
    Upsert,
}
```

## Values

| Value | Behaviour |
|-------|-----------|
| `CreateOnly` | Fail with [FileAlreadyExistsStorageException](8.Exceptions.md) when the key exists. **Default.** |
| `Overwrite` | Replace the existing object. Fail when the key is missing (provider-dependent). |
| `Upsert` | Create or replace without distinguishing the case. |

## Usage

```csharp
await fileStorage.PutAsync(new PutFileRequest
{
    Key = existingKey,
    Content = stream,
    Mode = PutMode.Overwrite,
});
```

## Design guidance

- Prefer **`CreateOnly`** for user uploads — generate a new key per upload to avoid accidental overwrites.
- Use **`Overwrite`** only when your domain explicitly models file replacement at a stable key.
- Use **`Upsert`** sparingly; explicit create vs replace semantics are easier to reason about in application code.

See also: [IFileStorage](1.IFileStorage.md), [StorageKey](6.StorageKey.md), [Exceptions](8.Exceptions.md).
