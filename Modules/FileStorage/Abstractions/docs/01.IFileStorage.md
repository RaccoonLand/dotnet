# 1. IFileStorage

`IFileStorage` is the single contract applications depend on for blob storage operations.

- **Package:** `RaccoonLand.Modules.FileStorage.Abstractions`
- **Namespace:** `RaccoonLand.Modules.FileStorage.Abstractions`
- **Source:** `src/Contracts/IFileStorage.cs`

## Surface

```csharp
public interface IFileStorage
{
    Task<PutFileResult> PutAsync(PutFileRequest request, CancellationToken cancellationToken = default);
    Task<OpenReadResult> OpenReadAsync(OpenReadRequest request, CancellationToken cancellationToken = default);
    Task DeleteAsync(DeleteFileRequest request, CancellationToken cancellationToken = default);
    Task<bool> ExistsAsync(ExistsFileRequest request, CancellationToken cancellationToken = default);
    Task<FileMetadata?> GetMetadataAsync(GetMetadataRequest request, CancellationToken cancellationToken = default);
}
```

## Operations

| Method | Purpose | Typical exceptions |
|--------|---------|-------------------|
| `PutAsync` | Stream content into storage | `FileAlreadyExistsStorageException`, `FileStorageValidationException` |
| `OpenReadAsync` | Return a readable stream | `FileNotFoundStorageException` |
| `DeleteAsync` | Remove an object by key | `FileNotFoundStorageException` when `IgnoreNotFound` is false |
| `ExistsAsync` | Check key presence without opening content | — |
| `GetMetadataAsync` | Read metadata without opening content | Returns `null` when missing |

## Design principles

1. **Provider-agnostic.** Applications persist [FileRef](2.FileRef.md), not bucket names or physical paths.
2. **Streaming-first.** All reads and writes operate on `Stream`. Prefer passing `ContentLength` on uploads when known.
3. **Authorization stays in the app.** The module does not enforce business permissions. Verify ownership before calling storage.
4. **Optional capabilities.** URL signing and multipart uploads are separate interfaces — see [IFileUrlSigner](3.IFileUrlSigner.md) and [IMultipartUploadCoordinator](4.IMultipartUploadCoordinator.md).
5. **Single active provider.** Register exactly one provider package at startup. Switching providers is a deployment concern, not a runtime selection.

## Typical upload flow

```csharp
await using var upload = await fileStorage.PutAsync(new PutFileRequest
{
    Content = request.Body,
    ContentType = request.ContentType,
    ContentLength = request.ContentLength,
    Mode = PutMode.CreateOnly,
}, cancellationToken);

var fileRef = upload.File;
// Persist fileRef.Key (and optional metadata) in your application database.
```

## Typical read flow

```csharp
await using var read = await fileStorage.OpenReadAsync(
    new OpenReadRequest { Key = storedKey },
    cancellationToken);

await read.Content.CopyToAsync(response.Body, cancellationToken);
```

Always dispose `OpenReadResult` — providers may hold open connections or file handles until disposal.
