# 11. FilePutConstraints and FileStoragePutHelper

Per-operation upload guardrails for application endpoints.

- **Package:** `RaccoonLand.Modules.FileStorage.Abstractions`
- **Namespace:** `RaccoonLand.Modules.FileStorage.Abstractions`
- **Source:** `src/Models/FilePutConstraints.cs`, `src/Validation/FileStoragePutHelper.cs`

## Problem

[FileStorageOptions](5.FileStorageOptions.md) defines **global infrastructure guardrails** shared by all providers. Applications also need **per-endpoint allowlists** — for example avatars accept only images while document endpoints accept PDF.

Business rules (who may upload) stay in the app. Content-type and size checks are technical guardrails that should not clutter endpoint code.

## Two layers

| Layer | Where | Purpose |
|-------|-------|---------|
| Global | `FileStorageOptions` in config | Infrastructure ceiling for the whole deployment |
| Per-operation | `FilePutConstraints` via helper | What this specific upload endpoint allows |

Both layers apply. A content type must pass **each** configured allowlist. Size limits use the **stricter** (minimum) of global and per-operation values.

## FilePutConstraints

```csharp
public sealed record FilePutConstraints
{
    public required IReadOnlySet<string> AllowedContentTypes { get; init; }
    public long? MaxUploadBytes { get; init; }

    public static FilePutConstraints For(params string[] contentTypes);
    public static FilePutConstraints Images { get; }
    public static FilePutConstraints PdfDocuments { get; }
}
```

MIME comparison is **case-insensitive**.

## FileStoragePutHelper

Three factory methods cover all upload entry points:

### PutAsync (server proxy)

```csharp
public static PutFileRequest CreateRequest(
    Stream content,
    string contentType,
    FilePutConstraints constraints,
    PutMode mode = PutMode.CreateOnly,
    string? key = null,
    long? contentLength = null,
    IReadOnlyDictionary<string, string>? metadata = null,
    FileStorageOptions? options = null);
```

### GetWriteUrlAsync (direct client upload via presigned URL)

```csharp
public static SignedWriteUrlRequest CreateSignedWriteRequest(
    string contentType,
    FilePutConstraints constraints,
    string? key = null,
    TimeSpan? expiry = null,
    long? maxSizeBytes = null,
    FileStorageOptions? options = null);
```

Sets `MaxSizeBytes` to the effective cap when omitted — useful as a client upload limit hint.

### InitiateAsync (multipart upload)

```csharp
public static InitiateMultipartUploadRequest CreateMultipartInitRequest(
    string contentType,
    FilePutConstraints constraints,
    string? key = null,
    IReadOnlyDictionary<string, string>? metadata = null,
    FileStorageOptions? options = null);
```

All helpers:

1. Normalize and validate content type against operation constraints
2. Validate against global `FileStorageOptions` when supplied (optional early fail in the app)
3. Apply effective size limits where applicable
4. Populate `AllowedContentTypes` and `MaxUploadBytes` on the request for provider re-validation

Providers call `FileStorageGuards.ValidatePutRequest`, `ValidateSignedWriteRequest`, or `ValidateMultipartInitRequest` — defense in depth.

## Application usage

### Server-side Put

```csharp
var request = FileStoragePutHelper.CreateRequest(
    file.OpenReadStream(),
    file.ContentType,
    FilePutConstraints.Images,
    contentLength: file.Length,
    options: options.Value);

var result = await storage.PutAsync(request, ct);
```

### Presigned write URL (S3)

```csharp
var signer = serviceProvider.GetRequiredService<IFileUrlSigner>();

var request = FileStoragePutHelper.CreateSignedWriteRequest(
    contentType: "image/png",
    constraints: FilePutConstraints.Images,
    expiry: TimeSpan.FromMinutes(10),
    options: options.Value);

var signed = await signer.GetWriteUrlAsync(request, ct);
// Return signed.Url to the client; persist signed.File.Key after upload completes
```

### Multipart initiate (S3, large files)

```csharp
var coordinator = serviceProvider.GetRequiredService<IMultipartUploadCoordinator>();

var request = FileStoragePutHelper.CreateMultipartInitRequest(
    contentType: "video/mp4",
    constraints: FilePutConstraints.For("video/mp4", "video/webm") with { MaxUploadBytes = 500 * 1024 * 1024 },
    options: options.Value);

var session = await coordinator.InitiateAsync(request, ct);
// Return session.Key and session.UploadId to the client for part uploads
```

Custom allowlist:

```csharp
var constraints = FilePutConstraints.For("application/pdf", "image/jpeg");
```

| Request | Fields |
|---------|--------|
| `PutFileRequest` | `AllowedContentTypes`, `MaxUploadBytes` |
| `SignedWriteUrlRequest` | `AllowedContentTypes`, `MaxUploadBytes`, `MaxSizeBytes` |
| `InitiateMultipartUploadRequest` | `AllowedContentTypes`, `MaxUploadBytes` |

## Configuration example

```json
{
  "FileStorage": {
    "MaxUploadBytes": 104857600,
    "AllowedContentTypes": [ "image/jpeg", "image/png", "image/webp", "application/pdf", "video/mp4" ]
  }
}
```

Global list = superset of what any endpoint may allow. Each endpoint narrows further via `FilePutConstraints`.
