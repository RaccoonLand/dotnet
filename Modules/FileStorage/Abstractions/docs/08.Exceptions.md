# 8. Exceptions

File storage operations raise typed exceptions derived from `FileStorageException`.

- **Package:** `RaccoonLand.Modules.FileStorage.Abstractions`
- **Namespace:** `RaccoonLand.Modules.FileStorage.Abstractions`
- **Source:** `src/Exceptions/FileStorageExceptions.cs`

## Hierarchy

```text
FileStorageException (abstract)
├── FileNotFoundStorageException
├── FileAlreadyExistsStorageException
├── FileAccessDeniedStorageException
├── FileStorageUnavailableException      (IsRetryable = true)
├── FileStorageConfigurationException
├── FileStorageValidationException
└── FileStorageNotSupportedException
```

## Types

| Exception | When raised | Retry? |
|-----------|-------------|--------|
| `FileNotFoundStorageException` | Key does not exist on read/delete | No |
| `FileAlreadyExistsStorageException` | `PutMode.CreateOnly` and key exists | No |
| `FileAccessDeniedStorageException` | Provider denied access (credentials, ACL) | No |
| `FileStorageUnavailableException` | Transient provider/network failure | Yes |
| `FileStorageConfigurationException` | Invalid or missing provider configuration at startup or first use | No |
| `FileStorageValidationException` | Request failed validation ([FileStorageGuards](../src/Validation/FileStorageGuards.cs), [StorageKey](6.StorageKey.md)) | No |
| `FileStorageNotSupportedException` | Capability not implemented by active provider (e.g. signing on FileSystem) | No |

## Base type

```csharp
public abstract class FileStorageException : Exception
{
    public virtual bool IsRetryable => false;
}
```

Use `IsRetryable` in Polly policies or background workers when wrapping storage calls.

## Handling guidance

```csharp
try
{
    await fileStorage.OpenReadAsync(new OpenReadRequest { Key = key }, ct);
}
catch (FileNotFoundStorageException)
{
    return Results.NotFound();
}
catch (FileStorageValidationException ex)
{
    return Results.BadRequest(ex.Message);
}
catch (FileStorageUnavailableException) when (attempt < maxAttempts)
{
    await Task.Delay(backoff, ct);
    // retry
}
```

**Do not** map storage exceptions directly to HTTP responses without authorization checks — a `FileNotFoundStorageException` may leak existence of keys if returned before access control.
