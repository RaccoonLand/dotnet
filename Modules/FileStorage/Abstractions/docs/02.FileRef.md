# 2. FileRef

`FileRef` is the provider-neutral reference applications store after upload.

- **Package:** `RaccoonLand.Modules.FileStorage.Abstractions`
- **Namespace:** `RaccoonLand.Modules.FileStorage.Abstractions`
- **Source:** `src/Models/FileRef.cs`

## Surface

```csharp
public sealed record FileRef
{
    public required string Key { get; init; }
    public string? Version { get; init; }
    public long? Length { get; init; }
    public string? ContentType { get; init; }
    public string? ChecksumSha256 { get; init; }
    public DateTimeOffset? CreatedAtUtc { get; init; }
}
```

## Fields

| Field | Description |
|-------|-------------|
| `Key` | Opaque storage identifier. Validated by [StorageKey](6.StorageKey.md) when supplied by callers. |
| `Version` | Provider-specific concurrency token (ETag, row version, etc.). |
| `Length` | Stored size in bytes when known. |
| `ContentType` | MIME type when known or inferred. |
| `ChecksumSha256` | Lowercase hex SHA-256 when computed by the provider. |
| `CreatedAtUtc` | Object creation timestamp when available. |

## Persistence rules

1. **Store `Key` only** in application tables. Do not persist bucket names, SQL table names, filesystem paths, or provider names alongside business entities unless you are building a multi-provider migration tool.
2. **Ownership belongs in the app.** Tenancy, user IDs, access policies, and display names live in your domain model — not inside `FileRef`.
3. **Treat `Version` as opaque.** Use it for optimistic concurrency or cache invalidation when the provider supplies it; do not parse provider-specific formats in application code.
4. **Optional fields may be null.** Not every provider populates checksums or creation timestamps on every operation.

## Related types

- [FileMetadata](1.IFileStorage.md) wraps `FileRef` with user metadata dictionaries returned by `GetMetadataAsync`.
- [PutFileResult](1.IFileStorage.md) and [SignedUrlResult](3.IFileUrlSigner.md) carry a `FileRef` after writes.
