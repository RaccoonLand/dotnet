# 10. Providers

Available file storage provider packages and their registration entry points.

## Package overview

| Package | NuGet id | `IFileStorage` | `IFileUrlSigner` | `IMultipartUploadCoordinator` |
|---------|----------|----------------|------------------|-------------------------------|
| FileSystem | `RaccoonLand.Modules.FileStorage.FileSystem` | Yes | — | — |
| SqlServer | `RaccoonLand.Modules.FileStorage.SqlServer` | Yes | — | — |
| SqlServer.FileStream | `RaccoonLand.Modules.FileStorage.SqlServer.FileStream` | Yes | — | — |
| S3 | `RaccoonLand.Modules.FileStorage.S3` | Yes | Yes | Yes |

All providers depend on `RaccoonLand.Modules.FileStorage.Abstractions` and honour [FileStorageOptions](5.FileStorageOptions.md).

## Registration

```csharp
services.AddRaccoonLandFileSystemStorage(configuration);
services.AddRaccoonLandSqlServerFileStorage(configuration);
services.AddRaccoonLandSqlServerFileStreamStorage(configuration);
services.AddRaccoonLandS3FileStorage(configuration);
```

Register **exactly one** provider per application.

## Configuration sections

### FileSystem

```json
{
  "FileStorage": {
    "FileSystem": {
      "RootPath": "C:\\storage\\files"
    }
  }
}
```

Local disk with sharded directory layout. Best for development and single-node deployments.

### SqlServer (VARBINARY)

```json
{
  "FileStorage": {
    "SqlServer": {
      "ConnectionString": "...",
      "TableName": "FileStorageObjects",
      "EnsureSchema": true
    }
  }
}
```

Stores full blob in `VARBINARY(MAX)`. Simple and portable; loads entire content into memory — not suitable for large files.

### SqlServer.FileStream

```json
{
  "FileStorage": {
    "SqlServer": {
      "FileStream": {
        "ConnectionString": "...",
        "SchemaName": "dbo",
        "TableName": "FileStorageFileStreamObjects",
        "FileGroupName": "FileStreamData",
        "ContentColumnName": "Content",
        "EnsureSchema": false
      }
    }
  }
}
```

- Target: `net10.0-windows`
- Requires Windows and a pre-provisioned FILESTREAM filegroup
- True streaming via `SqlFileStream`
- See provider docs under `Modules/FileStorage/Providers/SqlServer.FileStream/docs/`

### S3

```json
{
  "FileStorage": {
    "S3": {
      "Bucket": "my-bucket",
      "Region": "eu-west-1",
      "Endpoint": "https://s3.amazonaws.com",
      "Flavor": "Aws",
      "Credentials": {
        "AccessKeyId": "...",
        "SecretAccessKey": "..."
      },
      "Compatibility": {
        "ForcePathStyle": false
      }
    }
  }
}
```

`Flavor` presets: `Aws`, `Minio`, `GenericS3Compatible`. Supports presigned URLs and multipart uploads.

## Choosing a provider

| Need | Recommendation |
|------|----------------|
| Zero external dependencies | FileSystem or SqlServer |
| Large files on Windows + SQL Server | SqlServer.FileStream |
| Cloud-native, direct client transfers | S3 |
| Azure Blob Storage | Future separate provider (not S3-compatible) |
