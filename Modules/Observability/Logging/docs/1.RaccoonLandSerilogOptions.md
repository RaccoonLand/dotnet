# 1. RaccoonLandSerilogOptions

`RaccoonLandSerilogOptions` controls RaccoonLand-specific Serilog **enrichments**. Sinks, minimum levels, and
filters remain in the standard `Serilog` configuration section and are supplied by the consumer as separate
NuGet packages.

- **Package:** `RaccoonLand.Modules.Observability.Logging.Serilog`
- **Namespace:** `RaccoonLand.Modules.Observability.Logging.Serilog.Configuration`

## Surface

```csharp
public sealed class RaccoonLandSerilogOptions
{
    public const string SectionName = "RaccoonLandSerilog";

    public string ApplicationName { get; set; }
    public bool EnrichWithApplicationName { get; set; } = true;
    public bool EnrichWithEnvironmentName { get; set; } = true;
    public bool EnrichWithMachineName { get; set; } = true;
    public bool EnrichWithTraceId { get; set; } = true;
}
```

| Member | Description |
| --- | --- |
| `ApplicationName` | Logical name on every log event. When empty, falls back to `IHostEnvironment.ApplicationName`. |
| `EnrichWithApplicationName` | Adds `ApplicationName` property. |
| `EnrichWithEnvironmentName` | Adds `EnvironmentName` property (Development, Production, …). |
| `EnrichWithMachineName` | Adds `MachineName` property. |
| `EnrichWithTraceId` | Adds `TraceId` from `Activity.Current` when present (see [TraceIdEnricher](2.TraceIdEnricher.md)). |

## Configuration example

```json
{
  "RaccoonLandSerilog": {
    "ApplicationName": "Ordering.Api",
    "EnrichWithApplicationName": true,
    "EnrichWithEnvironmentName": true,
    "EnrichWithMachineName": true,
    "EnrichWithTraceId": true
  },
  "Serilog": {
    "Using": [ "Serilog.Sinks.Console" ],
    "MinimumLevel": "Information",
    "WriteTo": [ { "Name": "Console" } ]
  }
}
```

## Snapshot (no live reload)

Options are bound **once** when Serilog is configured by `UseRaccoonLandSerilog`. Later changes to
`RaccoonLandSerilog` in a reloadable `IConfiguration` do **not** update these enrichments unless the host
rebuilds the logger. Sinks/levels under the standard `Serilog` section follow Serilog's own configuration
reload behaviour, which is separate from this options object.

## Property collision policy

RaccoonLand enrichments are registered **before** `ReadFrom.Configuration`. That registration **order** is a
fact of this package; it is **not** a promise that later configuration “overrides” earlier properties.

Collision behaviour follows **Serilog enricher semantics**, which the consumer must account for when extending
the pipeline:

- Static enrichments (`ApplicationName`, `EnvironmentName`, `MachineName`) use Serilog's `Enrich.WithProperty`
  (add-if-absent on the event).
- `TraceId` uses `AddPropertyIfAbsent` (see [TraceIdEnricher](2.TraceIdEnricher.md)).

Therefore:

- A later enricher that also uses add-if-absent will **leave** an existing RaccoonLand property unchanged.
- A later enricher (or sink/formatter) that **forces** a property or renames fields can still change what the
  backend sees — that is the consumer's policy, not an automatic override from `ReadFrom.Configuration` alone.
- Treat property names from this package as a **contract to map or preserve**, not as values that configuration
  order will replace by default.

`ReadFrom.Configuration` can still **extend** the pipeline (more enrichers, sinks, filters). Remapping for a
backend belongs in the sink, formatter, or ingestion pipeline.

A missing or mistyped `sectionName` binds to default option values (all enrichments on, empty
`ApplicationName`) with **no** error. Only a null/whitespace `sectionName` argument is rejected.

See also: [TraceIdEnricher](2.TraceIdEnricher.md), [Registration](3.Registration.md).
