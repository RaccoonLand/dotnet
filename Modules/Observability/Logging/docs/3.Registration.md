# 3. Registration (UseRaccoonLandSerilog)

`UseRaccoonLandSerilog` wires Serilog into any `IHostBuilder` (ASP.NET Core web app or generic worker host).
It applies RaccoonLand enrichments, then reads the standard `Serilog` section from configuration.

- **Package:** `RaccoonLand.Modules.Observability.Logging.Serilog`
- **Namespace:** `RaccoonLand.Modules.Observability.Logging.Serilog.Hosting`

## Surface

```csharp
public static IHostBuilder UseRaccoonLandSerilog(
    this IHostBuilder hostBuilder,
    IConfiguration? configuration = null,
    string sectionName = RaccoonLandSerilogOptions.SectionName,
    Action<RaccoonLandSerilogOptions>? configureOptions = null);
```

| Parameter | Description |
| --- | --- |
| `hostBuilder` | The host builder (`WebApplicationBuilder.Host` or `Host.CreateDefaultBuilder`). |
| `configuration` | Optional configuration root; when `null`, uses `HostBuilderContext.Configuration`. |
| `sectionName` | Configuration section for [`RaccoonLandSerilogOptions`](1.RaccoonLandSerilogOptions.md) (default `RaccoonLandSerilog`). Must be non-whitespace; a missing/mistyped section falls back to default options silently. |
| `configureOptions` | Optional code-based customization after binding from configuration. |

## Setup order

Call **early** in `Program.cs`, before `Build()`:

```csharp
using RaccoonLand.Modules.Observability.Logging.Serilog.Hosting;

var builder = WebApplication.CreateBuilder(args);

builder.Host.UseRaccoonLandSerilog();

var app = builder.Build();
```

Generic worker host (`IHostBuilder`):

```csharp
using Microsoft.Extensions.Hosting;
using RaccoonLand.Modules.Observability.Logging.Serilog.Hosting;

var host = Host.CreateDefaultBuilder(args)
    .UseRaccoonLandSerilog()
    .ConfigureServices(services =>
    {
        // … worker services
    })
    .Build();

await host.RunAsync();
```

> The extension targets `IHostBuilder`. Prefer `builder.Host.UseRaccoonLandSerilog()` for
> `WebApplicationBuilder`, or `Host.CreateDefaultBuilder(...).UseRaccoonLandSerilog()` for workers.
> `Host.CreateApplicationBuilder` returns `HostApplicationBuilder`, which is not an `IHostBuilder` and does
> not expose this extension directly.

## Snapshot

`RaccoonLandSerilogOptions` is read when the Serilog logger is built. It is a **snapshot** — runtime reloads of
that section do not change RaccoonLand enrichments without rebuilding the logger. See
[RaccoonLandSerilogOptions](1.RaccoonLandSerilogOptions.md).

## Sinks (consumer responsibility)

This package does **not** ship sinks. Install sink packages in the application project, for example:

```xml
<PackageReference Include="Serilog.Sinks.Console" Version="6.0.0" />
```

Then configure them under the `Serilog` section in `appsettings.json` (see
[RaccoonLandSerilogOptions](1.RaccoonLandSerilogOptions.md)).

## What the extension configures

1. RaccoonLand enrichments from `RaccoonLandSerilog` (application name, environment, machine, trace id).
   Property collisions follow Serilog semantics and the consumer's pipeline policy — not a built-in “override”
   guarantee; see [property collision policy](1.RaccoonLandSerilogOptions.md#property-collision-policy).
2. `ReadFrom.Configuration` — levels, sinks, filters from `Serilog` section.
3. `ReadFrom.Services` — Serilog services registered in DI.

## Package dependencies

| Package | Purpose |
| --- | --- |
| `Serilog.AspNetCore` | `UseSerilog` host integration. |
| `Serilog.Settings.Configuration` | `ReadFrom.Configuration`. |

> No `FrameworkReference` to `Microsoft.AspNetCore.App` — works for API and worker hosts alike.
