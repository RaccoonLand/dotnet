# 2. TraceIdEnricher

`TraceIdEnricher` is an internal Serilog enricher that adds the current distributed-tracing identifier to every
log event when an `Activity` is active.

- **Package:** `RaccoonLand.Modules.Observability.Logging.Serilog`
- **Namespace:** `RaccoonLand.Modules.Observability.Logging.Serilog.Enrichers`

## Behaviour

When `System.Diagnostics.Activity.Current` is not null, the enricher adds a log property:

| Property | Source |
| --- | --- |
| `TraceId` | `Activity.Current.TraceId` |

The name is fixed as `TraceId` (PascalCase). The enricher uses Serilog's `AddPropertyIfAbsent`, so under that
semantics it does not replace an existing `TraceId` already on the event. Whether another enricher later wins
depends on **that** enricher's Serilog behaviour — this package does not define a global override policy.

When no activity is active, nothing is added — the enricher is a no-op.

## Backend / sink naming

If your sink or log backend expects a different field name (`trace_id`, `otelTraceId`, …), map or rename in the
sink configuration, formatter, or backend ingestion pipeline. This package does not rename `TraceId` per
destination.

## Enable / disable

Controlled by [`RaccoonLandSerilogOptions.EnrichWithTraceId`](1.RaccoonLandSerilogOptions.md) (default `true`).
Registered automatically by [`UseRaccoonLandSerilog`](3.Registration.md).

## Why TraceId

Correlating logs with ASP.NET Core request tracing, OpenTelemetry spans, and downstream services requires a
stable identifier on every log line. `TraceId` aligns with W3C trace context and is the same value exposed on
`Activity.Current` for the current scope.

See also: [RaccoonLandSerilogOptions](1.RaccoonLandSerilogOptions.md).
