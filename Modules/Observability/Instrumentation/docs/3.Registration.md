# 3. Registration

Register the instrumentation middleware — configuring its toggles in code or binding them from
`IConfiguration` — then add it to the pipelines.

- **Package:** `RaccoonLand.Modules.Observability.Instrumentation`
- **Namespace:** `Microsoft.Extensions.DependencyInjection` (so `AddRaccoonLandPipelineInstrumentation` is available wherever DI is configured)

## Surface

```csharp
public static IServiceCollection AddRaccoonLandPipelineInstrumentation(
    this IServiceCollection services,
    Action<InstrumentationOptions>? configure = null);

public static IServiceCollection AddRaccoonLandPipelineInstrumentation(
    this IServiceCollection services,
    IConfiguration configuration,
    string sectionName = InstrumentationOptions.SectionName);
```

Registers:

| Service | Lifetime | Notes |
| --- | --- | --- |
| `InstrumentationOptions` | options (`IOptions` / `IOptionsMonitor`) | configured in code or bound from `configuration`; validated on start |
| `IValidateOptions<InstrumentationOptions>` | singleton | `InstrumentationOptionsValidator` |
| `PipelineInstrumentationMiddleware` | singleton | `TryAddSingleton`; reads `IOptionsMonitor` per request |

Reload: when bound to a reloadable `IConfiguration` (for example `appsettings.json` with reload on change),
updates to the section apply to **subsequent** requests. Code-only `Configure(...)` is fixed unless you also
register a change token source.

## 1. Register and place the middleware

```csharp
services.AddRaccoonLandPipelineInstrumentation();

services.AddRaccoonLandRequestProcessing(
    configureCommandPipeline: p => p.UseMiddleware<PipelineInstrumentationMiddleware>(),
    configureQueryPipeline:   p => p.UseMiddleware<PipelineInstrumentationMiddleware>());
```

```json
{
  "Observability": {
    "Instrumentation": {
      "EnableTracing": true,
      "EnableMetrics": true,
      "EnableLogging": true,
      "RequestNameInMetrics": "FullName"
    }
  }
}
```

`RequestNameInMetrics` accepts `None`, `Name`, or `FullName` (default).

Invalid values are rejected by options validation (`InstrumentationOptionsValidator`):

- **Host startup** (`ValidateOnStart`): an initially invalid configuration fails the host so the app does not
  run with unknown instrumentation settings.
- **Runtime reload**: observability must not break the business pipeline. If a reload produces invalid options,
  reading `IOptionsMonitor` can throw `OptionsValidationException`; the middleware catches that, keeps the last
  known-good snapshot, and continues the request. Separately, if an unsupported enum value somehow reaches
  metric tagging, the request-name tag falls back to `FullName` instead of throwing.

## 2. Export the telemetry (the host's job)

The framework only emits; the host registers the well-known source and meter with the OpenTelemetry SDK and
chooses exporters:

```csharp
builder.Services.AddOpenTelemetry()
    .WithTracing(t => t.AddSource(RaccoonLandTelemetry.ActivitySourceName).AddOtlpExporter())
    .WithMetrics(m => m.AddMeter(RaccoonLandTelemetry.MeterName).AddOtlpExporter());
```

Logs flow through the standard `ILogger`; pair this with the
`RaccoonLand.Modules.Observability.Logging.Serilog` package (or any logging provider) to ship them.

See also: [PipelineInstrumentationMiddleware](2.PipelineInstrumentationMiddleware.md), [RaccoonLandTelemetry](1.RaccoonLandTelemetry.md).
