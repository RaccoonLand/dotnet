# 2. PipelineInstrumentationMiddleware

A single `IPipelineMiddleware` that instruments every command/query with the three observability pillars at
once, so the pipeline is only walked once:

- **Tracing** — starts a span `"{Kind} {RequestName}"` on the shared `ActivitySource`, tags it, records the
  outcome, and on an exception sets the span status to `Error` and attaches the exception.
- **Metrics** — increments an in-flight up/down counter for the duration, then records a request count and a
  duration histogram tagged by kind, name and outcome.
- **Logging** — opens an `ILogger` scope (`CorrelationId`, `UserId`, `TenantId`, `TraceId`) so **every** log
  written during the request carries them, and writes a start (`Debug`) and finish (`Information`/`Warning`/
  `Error`) entry.

- **Package:** `RaccoonLand.Modules.Observability.Instrumentation`
- **Namespace:** `RaccoonLand.Modules.Observability.Instrumentation.Diagnostics`

## Why this is the framework's job (not the host's)

The CQRS pipeline (`PipelineContext`, `IPipelineMiddleware`) and the notion of a correlation id are framework
abstractions, so instrumenting them belongs to the framework. The host remains responsible for the transport
concerns: installing the OpenTelemetry SDK and exporters, configuring sampling and resource attributes
(`service.name`, `service.version`), wiring Serilog sinks, and mapping endpoints. HTTP-server logging/metrics
and runtime/EF Core/HttpClient instrumentation are likewise the host's job.

## Behavior

- Each pillar can be disabled independently via `InstrumentationOptions` (`EnableTracing`, `EnableMetrics`,
  `EnableLogging`).
- The middleware is a stateless singleton. `ILogger` is injected (singleton-safe); the request-scoped
  `ICurrentExecutionContext` is resolved per request from `PipelineContext.RequestServices`.
- Exceptions are re-thrown after being recorded, so the rest of the pipeline (for example the exception-handling
  middleware) behaves exactly as before.

## Placement

Add it **first** (outermost) so it measures the whole pipeline, including downstream middleware:

```csharp
services.AddRaccoonLandRequestProcessing(
    configureCommandPipeline: pipeline =>
    {
        pipeline.UseMiddleware<PipelineInstrumentationMiddleware>();
        // … other middleware (validation, authorization, …)
    },
    configureQueryPipeline: pipeline =>
    {
        pipeline.UseMiddleware<PipelineInstrumentationMiddleware>();
        // …
    });
```
