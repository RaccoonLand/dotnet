# 1. RaccoonLandTelemetry

The well-known `ActivitySource` and `Meter` the pipeline instrumentation emits through. The framework only
**produces** telemetry; it never configures exporters. This keeps the package exporter-agnostic — the host
chooses the destination (OTLP, Jaeger, Prometheus, …).

- **Package:** `RaccoonLand.Modules.Observability.Instrumentation`
- **Namespace:** `RaccoonLand.Modules.Observability.Instrumentation.Telemetry`

## Names

| Member | Value | Used for |
| --- | --- | --- |
| `ActivitySourceName` | `RaccoonLand.RequestProcessing` | pipeline spans |
| `MeterName` | `RaccoonLand.RequestProcessing` | pipeline metrics |

## Metrics

| Instrument | Type | Unit | Tags |
| --- | --- | --- | --- |
| `raccoonland.requests.count` | Counter`<long>` | `{request}` | kind, name, outcome |
| `raccoonland.requests.duration` | Histogram`<double>` | `ms` | kind, name, outcome |
| `raccoonland.requests.active` | UpDownCounter`<long>` | `{request}` | kind |

`outcome` is one of `success`, `failure` (a domain failure — `PipelineResponse.Errors` is non-empty) or
`exception` (an unhandled exception bubbled out of the pipeline).

## Span tags

| Tag | Source |
| --- | --- |
| `raccoonland.request.kind` | `Command` / `Query` |
| `raccoonland.request.name` | `request.GetType().FullName` |
| `raccoonland.request.outcome` | `success` / `failure` / `exception` |
| `enduser.id` | `ICurrentExecutionContext.UserId` |
| `raccoonland.tenant.id` | `ICurrentExecutionContext.TenantId` |
| `raccoonland.correlation_id` | `ICurrentExecutionContext.CorrelationId` |

A span's status is set to `Error` only on an exception; expected domain failures keep an unset status and are
distinguished by the `outcome` tag.

## Host wiring (the host's job)

```csharp
builder.Services.AddOpenTelemetry()
    .WithTracing(t => t
        .AddSource(RaccoonLandTelemetry.ActivitySourceName)
        .AddOtlpExporter())
    .WithMetrics(m => m
        .AddMeter(RaccoonLandTelemetry.MeterName)
        .AddPrometheusExporter());
```
