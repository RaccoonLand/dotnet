# 4. OutboxPendingMessage

`OutboxPendingMessage` is the envelope of a buffered outbox message — what an [`IOutboxWriter`](2.IOutboxWriter.md)
implementation holds between `Enqueue` and flush. It shares the same shape as domain/service outbox rows.

- **Package:** `RaccoonLand.Modules.Persistence.Outbox.Abstraction`
- **Namespace:** `RaccoonLand.Modules.Persistence.Outbox.Abstraction`

## Surface

```csharp
public sealed record OutboxPendingMessage
{
    public required Guid Id { get; init; }
    public required string EventType { get; init; }
    public string? AggregateType { get; init; }
    public Guid? AggregateBusinessKey { get; init; }
    public required string Payload { get; init; }
    public string? CreatedBy { get; init; }
    public required DateTimeOffset OccurredOnUtc { get; init; }
}
```

| Member | Description |
| --- | --- |
| `Id` | Message identifier (idempotency key); assigned by the framework. |
| `EventType` | From options or CLR-name fallback — see [OutboxEnqueueOptions](3.OutboxEnqueueOptions.md). |
| `AggregateType` | Optional correlation: aggregate type name. |
| `AggregateBusinessKey` | Optional correlation: aggregate business key. |
| `Payload` | **JSON text** already produced by the writer. Always treat as final JSON; never serialize again on flush. |
| `CreatedBy` | Acting user when known; `null` = no user context (background/system/unavailable). Allowed. |
| `OccurredOnUtc` | Enqueue timestamp; **UTC with zero offset** (implementations use e.g. `DateTimeOffset.UtcNow`). |

## Payload JSON contract

- Property name remains `Payload` (JSON content is documented here; renaming to `PayloadJson` was deferred to
  avoid a breaking change).
- Value is always a string containing JSON.
- Polymorphic type handling is defined by the implementation serializer (for example
  `JsonSerializerDefaults.Web` in the SQL Server writer).
- Flush paths copy `Payload` to storage as-is.

## Role

This is an **in-memory**, storage-agnostic envelope. Consumers of the abstraction normally never construct this
type directly — they enqueue a payload through [`IOutboxWriter`](2.IOutboxWriter.md).

### Mapping note (SQL Server implementation — not part of this package)

The SQL Server Commands module maps each `OutboxPendingMessage` to a table row and inserts it on the shared
connection/transaction during `SaveChanges`. See that module's outbox interceptor docs for physical details.

See also: [IOutboxWriter](2.IOutboxWriter.md), [OutboxEnqueueOptions](3.OutboxEnqueueOptions.md).
