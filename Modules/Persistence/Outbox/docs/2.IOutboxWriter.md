# 2. IOutboxWriter

`IOutboxWriter` is the **request-scoped** collector applications use to enqueue outbox messages during command
handling. Buffered messages are flushed by the implementation atomically with the transaction when
`SaveChanges` completes.

- **Package:** `RaccoonLand.Modules.Persistence.Outbox.Abstraction`
- **Namespace:** `RaccoonLand.Modules.Persistence.Outbox.Abstraction`

## Surface

```csharp
public interface IOutboxWriter
{
    void Enqueue<TOutbox>(object payload, OutboxEnqueueOptions? options = null)
        where TOutbox : IOutbox;
}
```

| Parameter | Description |
| --- | --- |
| `TOutbox` | The target channel marker ([`IOutbox`](1.IOutbox.md)). Must be registered. |
| `payload` | The message body; serialized to JSON when flushed. |
| `options` | Optional correlation metadata ([`OutboxEnqueueOptions`](3.OutboxEnqueueOptions.md)). |

## Semantics

These are the guarantees the abstraction expects every implementation to honor:

- **Buffered, not immediate:** `Enqueue` only records the message in a request-scoped buffer. Nothing is
  written until `SaveChanges` succeeds, so a rolled-back command writes no outbox rows.
- **Transactional:** on flush, rows are written on the same connection/transaction as the entity changes —
  all-or-nothing.
- **Unregistered channel:** enqueueing for a channel that was not registered throws `InvalidOperationException`.

> How a channel maps to physical storage is **implementation-specific** and is not part of this abstraction.
> The SQL Server implementation maps each channel to a table — see the SQL Server Commands module docs for
> [channel options](../../SqlServer/Commands/docs/4.OutboxChannelOptions.md) and the
> [channel registry](../../SqlServer/Commands/docs/5.IOutboxChannelRegistry.md).

## Usage

```csharp
public sealed class PlaceOrderEndpoint(IOutboxWriter outbox, ShopCommandDbContext db)
    : IEndpoint<PlaceOrderCommand>
{
    public async Task<Result> ExecuteAsync(PlaceOrderCommand request, CancellationToken cancellationToken)
    {
        var order = Order.Place(request);
        db.Orders.Add(order);

        outbox.Enqueue<IQueueOutbox>(new OrderPlacedMessage(order.Id));

        await db.SaveChangesAsync(cancellationToken); // order + outbox row committed together
        return Result.Success();
    }
}
```

See also: [OutboxEnqueueOptions](3.OutboxEnqueueOptions.md), [OutboxPendingMessage](4.OutboxPendingMessage.md).
