# 3. OutboxEnqueueOptions

`OutboxEnqueueOptions` carries optional correlation metadata supplied when calling
[`IOutboxWriter.Enqueue`](2.IOutboxWriter.md).

- **Package:** `RaccoonLand.Modules.Persistence.Outbox.Abstraction`
- **Namespace:** `RaccoonLand.Modules.Persistence.Outbox.Abstraction`

## Surface

```csharp
public sealed class OutboxEnqueueOptions
{
    public string? EventType { get; init; }
    public string? AggregateType { get; init; }
    public Guid? AggregateBusinessKey { get; init; }
}
```

| Member | Description |
| --- | --- |
| `EventType` | Overrides the default event type (the payload CLR type name). |
| `AggregateType` | Optional aggregate type name for correlation. |
| `AggregateBusinessKey` | Optional aggregate business key for correlation. |

## Framework-assigned fields

The identity fields of a message — `Id`, `CreatedBy`, `OccurredOnUtc` — are **always assigned by the
framework** when the message is buffered and cannot be set through these options. This keeps message identity
and provenance consistent across channels.

## Example

```csharp
outbox.Enqueue<IQueueOutbox>(
    new OrderPlacedMessage(order.Id),
    new OutboxEnqueueOptions
    {
        EventType = "order.placed.v1",
        AggregateType = nameof(Order),
        AggregateBusinessKey = order.BusinessKey,
    });
```

See also: [IOutboxWriter](2.IOutboxWriter.md), [OutboxPendingMessage](4.OutboxPendingMessage.md).
