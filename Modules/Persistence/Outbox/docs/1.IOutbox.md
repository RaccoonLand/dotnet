# 1. IOutbox

`IOutbox` is the marker interface that identifies a **logical outbox channel**. Each channel maps to an
implementation-defined destination; a message enqueued for a channel is written there when the ambient unit of
work flushes (typically `SaveChanges`).

- **Package:** `RaccoonLand.Modules.Persistence.Outbox.Abstraction`
- **Namespace:** `RaccoonLand.Modules.Persistence.Outbox.Abstraction`

## Surface

```csharp
public interface IOutbox;
```

The interface carries **no members**. It exists only to identify a channel at compile time; routing to physical
storage is resolved by the **implementation** at registration time.

## Defining channels

There are **no built-in channel markers** — the application defines the channels it needs:

```csharp
// Application-defined channels
public interface IQueueOutbox : IOutbox;
public interface IFullTextSearchOutbox : IOutbox;
```

Then enqueue against the marker:

```csharp
outboxWriter.Enqueue<IQueueOutbox>(new OrderPlacedMessage(orderId));
```

## Registration contract (expected of implementations)

| Situation | Expected behaviour |
| --- | --- |
| Channel not registered | `IOutboxWriter.Enqueue` throws `InvalidOperationException` (fail on enqueue, not silently on flush). |
| Duplicate registration | Implementation-defined (replace, ignore, or throw). Documented by the concrete package. |
| When errors surface | Prefer fail at registration or enqueue time, not after the business transaction has partially completed. |

## Why marker interfaces

- They make the target channel **explicit and type-checked** at the call site.
- They keep storage configuration out of calling code — it lives in the implementation, registered once at
  startup.

## Storage configuration lives in the implementation

This abstraction intentionally says nothing about *where* a channel is stored. Per-channel options and the
channel registry are **implementation concerns**.

### Example (SQL Server implementation — not part of this package)

```csharp
// From RaccoonLand.Modules.Persistence.SqlServer.Commands — illustrative only
services.AddRaccoonLandOutbox<IQueueOutbox>(o => o.Table = "QueueOutbox");
services.AddRaccoonLandOutbox<IFullTextSearchOutbox>(o => o.Table = "FullTextSearchOutbox");
```

See also that package's channel options / registry docs. Do not treat `AddRaccoonLandOutbox` or `Table` as APIs
of this abstraction package.
