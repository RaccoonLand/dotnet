# 1. IOutbox

`IOutbox` is the marker interface that identifies a **logical outbox channel**. Each channel maps to its own
outbox table; a message enqueued for a channel is written to that channel's table on `SaveChanges`.

- **Package:** `RaccoonLand.Modules.Persistence.Outbox.Abstraction`
- **Namespace:** `RaccoonLand.Modules.Persistence.Outbox.Abstraction`

## Surface

```csharp
public interface IOutbox;
```

The interface carries **no members**. It exists only to identify a channel at compile time; routing to physical
storage is resolved by the **implementation** at registration time (for the SQL Server module via
`AddRaccoonLandOutbox<TOutbox>()`).

## Defining channels

There are **no built-in channel markers** — the application defines the channels it needs. Declare an interface
per channel and register it with its table:

```csharp
// Application-defined channels
public interface IQueueOutbox : IOutbox;
public interface IFullTextSearchOutbox : IOutbox;
```

```csharp
services.AddRaccoonLandOutbox<IQueueOutbox>(o => o.Table = "QueueOutbox");
services.AddRaccoonLandOutbox<IFullTextSearchOutbox>(o => o.Table = "FullTextSearchOutbox");
```

Then enqueue against the marker:

```csharp
outboxWriter.Enqueue<IQueueOutbox>(new OrderPlacedMessage(orderId));
```

## Why marker interfaces

- They make the target channel **explicit and type-checked** at the call site.
- They keep storage configuration (table/schema/database for SQL, topic for a queue, …) out of the calling
  code — it lives in the implementation, registered once at startup.

## Storage configuration lives in the implementation

This abstraction intentionally says nothing about *where* a channel is stored. Per-channel storage options and
the channel registry are **implementation concerns** so that different backends (SQL Server, Oracle, a queue,
…) can model them differently. For the SQL Server backend see
[`OutboxChannelOptions`](../../SqlServer/Commands/docs/4.OutboxChannelOptions.md) and
[`IOutboxChannelRegistry`](../../SqlServer/Commands/docs/5.IOutboxChannelRegistry.md).

See also: [IOutboxWriter](2.IOutboxWriter.md), [OutboxEnqueueOptions](3.OutboxEnqueueOptions.md).
