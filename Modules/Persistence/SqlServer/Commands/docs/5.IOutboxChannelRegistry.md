# 5. IOutboxChannelRegistry

`IOutboxChannelRegistry` maps registered [`IOutbox`](../../Outbox/docs/1.IOutbox.md) channel markers to their
[`OutboxChannelOptions`](4.OutboxChannelOptions.md). Because the options are SQL Server-specific, the registry
that returns them is an implementation concern and lives in this package (not in the abstraction).

- **Package:** `RaccoonLand.Modules.Persistence.SqlServer.Commands`
- **Namespace:** `RaccoonLand.Modules.Persistence.SqlServer.Commands.Outbox`

## Surface

```csharp
public interface IOutboxChannelRegistry
{
    OutboxChannelOptions? Get<TOutbox>() where TOutbox : IOutbox;
    OutboxChannelOptions? Get(Type channelType);
    IReadOnlyCollection<Type> Channels { get; }
}
```

| Member | Description |
| --- | --- |
| `Get<TOutbox>()` | Options for the channel marker `TOutbox`, or `null` when not registered. |
| `Get(Type)` | Non-generic lookup by channel marker type. |
| `Channels` | All registered channel marker types. |

## Implementation and lifetime

The concrete `OutboxChannelRegistry` is a **singleton** populated at startup from the
`AddRaccoonLandOutbox<TOutbox>()` calls. It is read on the `SaveChanges` path by
[`OutboxWriterSaveChangesInterceptor`](6.OutboxWriterSaveChangesInterceptor.md) to resolve the destination
table for each buffered message, and by [`OutboxWriter`](6.OutboxWriterSaveChangesInterceptor.md) to validate a
channel on `Enqueue`.

## Usage

Application code rarely calls the registry directly — it enqueues through
[`IOutboxWriter`](../../Outbox/docs/2.IOutboxWriter.md) and the implementation consults the registry. Direct
lookups are useful for diagnostics:

```csharp
foreach (var channel in registry.Channels)
{
    var options = registry.Get(channel);
    logger.LogInformation("Outbox channel {Channel} -> {Table}", channel.Name, options!.QualifiedTableName);
}
```

See also: [OutboxChannelOptions](4.OutboxChannelOptions.md), [Registration](7.Registration.md).
