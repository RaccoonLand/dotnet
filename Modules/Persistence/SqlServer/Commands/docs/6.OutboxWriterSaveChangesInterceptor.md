# 6. OutboxWriterSaveChangesInterceptor (message outbox)

`OutboxWriterSaveChangesInterceptor` is the SQL Server implementation of the
[`IOutboxWriter`](../../Outbox/docs/2.IOutboxWriter.md) abstraction. It flushes the request-scoped writer buffer
into registered channel tables with Dapper on the **same connection and transaction** as `SaveChanges`.

- **Package:** `RaccoonLand.Modules.Persistence.SqlServer.Commands`
- **Namespace:** `RaccoonLand.Modules.Persistence.SqlServer.Commands.Interceptors`

## Two different outboxes

| Interceptor | Source of rows | Destination |
| --- | --- | --- |
| [`OutboxSaveChangesInterceptor`](3.OutboxSaveChangesInterceptor.md) | Domain/service **events** on `IAggregateRoot`. | Single events table. |
| `OutboxWriterSaveChangesInterceptor` | Messages from `IOutboxWriter.Enqueue<TOutbox>(...)`. | Per-channel tables. |

## Flush and cleanup

| Hook | Behaviour |
| --- | --- |
| `SavedChangesAsync` | `GetPendingBatches()` → INSERT → `MarkFlushed` (messages leave the pending set but are kept until commit). |
| `TransactionCommitted` | `ClearFlushedOnCommit()` — drop flushed messages. |
| `TransactionRolledBack` | `RestoreFlushedOnRollback()` — return flushed messages to pending for retry. |

Only **async** `SaveChangesAsync` is supported.

## Collaborators

- **`OutboxWriter`** — scoped `IOutboxWriter`. Prefer explicit versioned `EventType`; otherwise payload CLR
  `Name` is a rename-fragile fallback.
- **`OutboxChannelRegistry`** — singleton registry from `AddRaccoonLandOutbox<TOutbox>()`.
- **`OutboxRow`** — Dapper INSERT shape.

## Table contract

```
Id, EventType, AggregateType, AggregateBusinessKey, Payload, CreatedBy, OccurredOnUtc, CreatedOnUtc
```
