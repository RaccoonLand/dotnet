# 6. OutboxWriterSaveChangesInterceptor (message outbox)

`OutboxWriterSaveChangesInterceptor` is the SQL Server implementation of the
[`IOutboxWriter`](../../Outbox/docs/2.IOutboxWriter.md) abstraction. It drains the request-scoped writer buffer
and inserts each enqueued message into its registered channel table with Dapper, on the **same connection and
transaction** as `SaveChanges`.

- **Package:** `RaccoonLand.Modules.Persistence.SqlServer.Commands`
- **Namespace:** `RaccoonLand.Modules.Persistence.SqlServer.Commands.Interceptors`

## Two different outboxes

This module ships **two** outbox interceptors that must not be confused:

| Interceptor | Source of rows | Destination |
| --- | --- | --- |
| [`OutboxSaveChangesInterceptor`](3.OutboxSaveChangesInterceptor.md) | Domain/service **events** raised by `IAggregateRoot` entities tracked by EF Core. | Single events table ([`OutboxOptions`](3.OutboxSaveChangesInterceptor.md)). |
| `OutboxWriterSaveChangesInterceptor` | Messages explicitly **enqueued** via `IOutboxWriter.Enqueue<TOutbox>(...)`. | Per-channel tables resolved from the [channel registry](5.IOutboxChannelRegistry.md). |

## Collaborators

- **`OutboxWriter`** — the request-scoped `IOutboxWriter` implementation. `Enqueue<TOutbox>` validates the
  channel against the registry, serializes the payload to JSON, assigns framework fields (`Id` via
  `Guid.CreateVersion7()`, `OccurredOnUtc`, and `CreatedBy` from
  [`ICurrentExecutionContext`](../../../../Core/ExecutionContext/docs/1.ICurrentExecutionContext.md)), and
  buffers an [`OutboxPendingMessage`](../../Outbox/docs/4.OutboxPendingMessage.md). `Drain()` groups the buffer
  per channel.
- **`OutboxChannelRegistry`** — the singleton [`IOutboxChannelRegistry`](5.IOutboxChannelRegistry.md) populated
  at startup from `AddRaccoonLandOutbox<TOutbox>()` calls.
- **`OutboxRow`** — the row shape mapped to channel-table columns for the Dapper INSERT.

## Flush flow (`SavedChangesAsync`)

1. `Drain()` the writer; return early if nothing is buffered.
2. Grab the **current** `DbConnection` and `CurrentTransaction` from the context (set up by
   [`BaseCommandDbContext`](1.BaseCommandDbContext.md)).
3. For each channel batch, look up [`OutboxChannelOptions`](4.OutboxChannelOptions.md) (throws if the channel is
   unregistered) and `INSERT` the rows into `QualifiedTableName`.

Because the insert reuses the EF Core transaction, the entity changes and the outbox rows commit
atomically — a rolled-back command writes no outbox rows.

## Table contract

Each channel table must expose the columns used by the INSERT:

```
Id, EventType, AggregateType, AggregateBusinessKey, Payload, CreatedBy, OccurredOnUtc, CreatedOnUtc
```

`CreatedBy` is stamped by the writer from
[`ICurrentExecutionContext`](../../../../Core/ExecutionContext/docs/1.ICurrentExecutionContext.md): it is set to
`UserId` when the context is available, otherwise `null` (when no implementation is registered, the writer falls
back to `NullCurrentExecutionContext`).

See registration in [7. Registration](7.Registration.md).
