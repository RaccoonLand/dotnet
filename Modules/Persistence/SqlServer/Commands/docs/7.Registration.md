# 7. Registration

This module exposes two registration surfaces:

- `AddRaccoonLandCommandInterceptors` — audit/concurrency + domain/service **event** outbox.
- `AddRaccoonLandOutbox` / `AddRaccoonLandOutbox<TOutbox>` + `AddRaccoonLandOutboxInterceptor` — the
  `IOutboxWriter` **message** outbox.

- **Package:** `RaccoonLand.Modules.Persistence.SqlServer.Commands`
- **Namespace:** `RaccoonLand.Modules.Persistence.SqlServer.Commands`

## AddRaccoonLandCommandInterceptors

```csharp
public static DbContextOptionsBuilder AddRaccoonLandCommandInterceptors(
    this DbContextOptionsBuilder builder,
    OutboxOptions outboxOptions,
    ICurrentExecutionContext? currentExecutionContext = null);
```

| Parameter | Description |
| --- | --- |
| `outboxOptions` | Where domain/service events are written ([`OutboxOptions`](3.OutboxSaveChangesInterceptor.md)). |
| `currentExecutionContext` | Execution context for the [audit interceptor](2.AuditSaveChangesInterceptor.md). When `null`, falls back to `NullCurrentExecutionContext.Instance`. |

Interceptors are added in this order:

1. [`AuditSaveChangesInterceptor`](2.AuditSaveChangesInterceptor.md) — runs on `SavingChanges`.
2. [`OutboxSaveChangesInterceptor`](3.OutboxSaveChangesInterceptor.md) — runs on `SavedChanges`.

### Execution context fallback

The audit interceptor needs the acting user from
[`ICurrentExecutionContext`](../../../../Core/ExecutionContext/docs/1.ICurrentExecutionContext.md). If a host
has not implemented it, pass `null` (or omit the argument) and the registration substitutes
[`NullCurrentExecutionContext.Instance`](../../../../Core/ExecutionContext/docs/2.NullCurrentExecutionContext.md):
auditing still records timestamps with a `null` user instead of throwing.

```csharp
services.AddDbContext<ShopCommandDbContext>((sp, options) =>
{
    options.UseSqlServer(connectionString);
    options.AddRaccoonLandCommandInterceptors(
        new OutboxOptions { Schema = "dbo", Table = "OutboxEvent" },
        sp.GetService<ICurrentExecutionContext>());
});
```

## Message outbox (IOutboxWriter)

The message outbox is opt-in and independent of the event outbox. Register the infrastructure, declare one or
more channels, and add the interceptor. The channel storage type ([`OutboxChannelOptions`](4.OutboxChannelOptions.md))
and the [`IOutboxChannelRegistry`](5.IOutboxChannelRegistry.md) are SQL Server-specific and provided by **this**
package.

```csharp
// 1. Application-defined channel markers (this module ships none on purpose)
public interface IQueueOutbox : IOutbox;

// 2. Register channels (infrastructure is wired up automatically)
services.AddRaccoonLandOutbox<IQueueOutbox>(o =>
{
    o.Schema = "messaging";
    o.Table  = "QueueOutbox";
});

// 3. Add the writer interceptor to the DbContext (scoped -> use the (sp, options) overload)
services.AddDbContext<ShopCommandDbContext>((sp, options) =>
{
    options.UseSqlServer(connectionString);
    options.AddRaccoonLandCommandInterceptors(
        new OutboxOptions { Table = "OutboxEvent" },
        sp.GetService<ICurrentExecutionContext>());
    options.AddRaccoonLandOutboxInterceptor(sp);
});
```

| Member | Description |
| --- | --- |
| `AddRaccoonLandOutbox()` | Registers the singleton channel registry, the scoped `OutboxWriter`/`IOutboxWriter`, and the scoped `OutboxWriterSaveChangesInterceptor`. |
| `AddRaccoonLandOutbox<TOutbox>(configure)` | Registers a channel marker and its `OutboxChannelOptions`; calls the base registration automatically. |
| `AddRaccoonLandOutboxInterceptor(serviceProvider)` | Resolves the scoped writer interceptor and attaches it to the `DbContextOptionsBuilder`. Must be called from a scoped provider (the `(sp, options)` overload of `AddDbContext`). |

> `IFullTextSearchOutbox` and `IQueueOutbox` are **not** provided by this package — define the channel markers
> your application needs (any interface inheriting [`IOutbox`](../../Outbox/docs/1.IOutbox.md)).
