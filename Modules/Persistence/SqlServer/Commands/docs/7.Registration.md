# 7. Registration

This module exposes two registration surfaces:

- `AddRaccoonLandCommandInterceptors` — audit/concurrency + domain/service **event** outbox.
- `AddRaccoonLandOutbox` / `AddRaccoonLandOutbox<TOutbox>` + `AddRaccoonLandOutboxInterceptor` — the
  `IOutboxWriter` **message** outbox.

- **Package:** `RaccoonLand.Modules.Persistence.SqlServer.Commands`
- **Namespace:** `RaccoonLand.Modules.Persistence.SqlServer.Commands`

## AddRaccoonLandCommandInterceptors

```csharp
public static DbContextOptionsBuilder AddRaccoonLandCommandInterceptors(
    this DbContextOptionsBuilder builder,
    OutboxOptions outboxOptions,
    ICurrentExecutionContext? currentExecutionContext = null);
```

### Interceptor order (contract)

Order is **required** and fixed by this method:

1. [`AuditSaveChangesInterceptor`](2.AuditSaveChangesInterceptor.md) — `SavingChanges` (stamps `IAuditable`).
2. [`OutboxSaveChangesInterceptor`](3.OutboxSaveChangesInterceptor.md) — `SavedChangesAsync` +
   `TransactionCommitted` (reads `ModifiedBy`/`CreatedBy`, writes event outbox, clears events after commit).

Reordering (or inserting another interceptor that clears audit fields) can produce wrong/`null` `CreatedBy` on
outbox event rows.

### Async-only save path

Outbox interceptors override **`SavedChangesAsync`** (and transaction commit/rollback). Synchronous
`SaveChanges` is blocked by [`BaseCommandDbContext`](1.BaseCommandDbContext.md). Always use
`SaveChangesAsync`.

```csharp
services.AddDbContext<ShopCommandDbContext>((sp, options) =>
{
    options.UseSqlServer(connectionString);
    options.AddRaccoonLandCommandInterceptors(
        new OutboxOptions { Schema = "dbo", Table = "OutboxEvent" },
        sp.GetService<ICurrentExecutionContext>());
});
```

## Message outbox (IOutboxWriter)

Register after command interceptors so audit still runs first for the event outbox; the writer interceptor is
independent but should be added in the same `AddDbContext` callback:

```csharp
services.AddRaccoonLandOutbox<IQueueOutbox>(o =>
{
    o.Schema = "messaging";
    o.Table  = "QueueOutbox";
});

services.AddDbContext<ShopCommandDbContext>((sp, options) =>
{
    options.UseSqlServer(connectionString);
    options.AddRaccoonLandCommandInterceptors(
        new OutboxOptions { Table = "OutboxEvent" },
        sp.GetService<ICurrentExecutionContext>());
    options.AddRaccoonLandOutboxInterceptor(sp); // after command interceptors
});
```

| Member | Description |
| --- | --- |
| `AddRaccoonLandOutbox()` | Registry + scoped writer + scoped writer interceptor. |
| `AddRaccoonLandOutbox<TOutbox>(configure)` | Registers a channel; calls base registration. |
| `AddRaccoonLandOutboxInterceptor(serviceProvider)` | Attach scoped writer interceptor (use `(sp, options)`). |

## External transactions

If the host begins its own EF transaction, see [BaseCommandDbContext](1.BaseCommandDbContext.md) for caller
duties (commit, rollback covering Dapper INSERTs, no early dispose, own retry policy).

See also: [BaseCommandDbContext](1.BaseCommandDbContext.md),
[OutboxWriterSaveChangesInterceptor](6.OutboxWriterSaveChangesInterceptor.md).
