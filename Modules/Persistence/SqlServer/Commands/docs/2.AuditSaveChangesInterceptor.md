# 2. AuditSaveChangesInterceptor

`AuditSaveChangesInterceptor` stamps audit fields on `IAuditable` entities and regenerates the concurrency
token on modified `IAggregateRoot`s during save. The acting user comes from
[`ICurrentExecutionContext`](../../../../Core/ExecutionContext/docs/1.ICurrentExecutionContext.md).

- **Package:** `RaccoonLand.Modules.Persistence.SqlServer.Commands`
- **Namespace:** `RaccoonLand.Modules.Persistence.SqlServer.Commands.Interceptors`

## Behaviour

Runs on `SavingChanges` / `SavingChangesAsync` (before the database write):

| Entry state | Action |
| --- | --- |
| `Added` + `IAuditable` | `SetCreatedAudit(utcNow, user)` |
| `Modified` + `IAuditable` | `SetModifiedAudit(utcNow, user)` |
| `Modified` + `IAggregateRoot` | `RegenerateConcurrencyToken()` |

## Resolving the user from ICurrentExecutionContext

```csharp
protected virtual string? GetCurrentUser()
    => _executionContext.IsAvailable ? _executionContext.UserId : null;
```

- When the execution context **is available**, the user is `ICurrentExecutionContext.UserId`.
- When it is **not** available, the user is `null` (the row is still stamped with the timestamp).

## Fallback when no execution context is implemented

The constructor accepts an optional context and falls back to
[`NullCurrentExecutionContext`](../../../../Core/ExecutionContext/docs/2.NullCurrentExecutionContext.md):

```csharp
public AuditSaveChangesInterceptor(ICurrentExecutionContext? executionContext = null)
    => _executionContext = executionContext ?? NullCurrentExecutionContext.Instance;
```

This means a host that has **not** implemented `ICurrentExecutionContext` still gets a working interceptor: it
records audit timestamps with a `null` user instead of failing. The fallback is wired in the
[registration helper](4.Registration.md).

## Customizing

Override `GetCurrentUser()` in a derived interceptor to change how the acting user is determined (for example
to combine `UserId` with `TenantId`, or to substitute a system user for background work).

See also: [Registration](4.Registration.md), [BaseCommandDbContext](1.BaseCommandDbContext.md).
