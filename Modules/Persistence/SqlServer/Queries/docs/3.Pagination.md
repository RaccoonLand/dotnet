# 3. Pagination (Queryable extensions)

EF Core helpers for the common read-side pagination pattern used with
[`PagedQuery<TResponse>`](../../../Core/RequestProcessing/RequestProcessing.Abstractions/docs/9.Pagination.md)
and [`PageResponse<TItem>`](../../../Core/RequestProcessing/RequestProcessing.Abstractions/docs/9.Pagination.md).

- **Package:** `RaccoonLand.Modules.Persistence.SqlServer.Queries`
- **Namespace:** `RaccoonLand.Modules.Persistence.SqlServer.Queries`

This package does **not** reference the CQRS abstractions. Endpoints map `PagedList<T>` to their
`PageResponse<TItem>` type.

## Types

```csharp
public readonly record struct PageRequest(int Page, int PageSize, bool IncludeTotalCount);

public sealed record PagedList<T>
{
    public IReadOnlyList<T> Items { get; init; } = [];
    public int? TotalCount { get; init; }
    public int Page { get; init; }
    public int PageSize { get; init; }
}
```

## Extensions

```csharp
public static class QueryablePaginationExtensions
{
    public static PageRequest Normalize(
        int page, int pageSize, bool includeTotalCount, int maxPageSize = 100);

    public static IQueryable<T> ApplyPaging<T>(this IQueryable<T> query, PageRequest request);

    public static Task<PagedList<T>> ToPagedListAsync<T>(
        this IQueryable<T> query,
        int page,
        int pageSize,
        bool includeTotalCount = false,
        int maxPageSize = 100,
        CancellationToken cancellationToken = default);
}
```

| Method | Purpose |
| --- | --- |
| `Normalize` | Clamps `page` to at least `1` and `pageSize` to `[1, maxPageSize]`. |
| `ApplyPaging` | Applies `Skip`/`Take` for a normalized `PageRequest`. |
| `ToPagedListAsync` | Optional `CountAsync`, then fetches the page. |

## Usage

Apply filters, **ordering**, and projection on the query first, then paginate:

```csharp
var paged = await query
    .Where(x => x.Status == status)
    .OrderBy(x => x.LastName)
    .Select(x => new PersonListItem { Id = x.Id, LastName = x.LastName })
    .ToPagedListAsync(
        request.Page,
        request.PageSize,
        request.IncludeTotalCount,
        maxPageSize: 100,
        cancellationToken);

return Result<SearchPeopleResponse>.Success(new SearchPeopleResponse
{
    Items = paged.Items,
    TotalCount = paged.TotalCount,
    Page = paged.Page,
    PageSize = paged.PageSize,
});
```

## Rules

1. **Ordering is caller responsibility** â€” the extension does not impose sort order.
2. **`IncludeTotalCount == false`** â€” no `CountAsync` is executed.
3. **`IncludeTotalCount == true`** â€” count runs on the filtered query before `Skip`/`Take`.
4. **`maxPageSize`** defaults to `100`. Validators may clamp earlier; this is a safe fallback in the handler.
5. **No `AsNoTracking()`** â€” `BaseQueryDbContext` already disables tracking.
