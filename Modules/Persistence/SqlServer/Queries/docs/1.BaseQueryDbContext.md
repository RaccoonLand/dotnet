# 1. BaseQueryDbContext

`BaseQueryDbContext` is the base class for the **read side** of the CQRS split ‚Äî the query database context. It
enforces that the read model is never used to write and is optimized for read throughput.

- **Package:** `RaccoonLand.Modules.Persistence.SqlServer.Queries`
- **Namespace:** `RaccoonLand.Modules.Persistence.SqlServer.Queries`

## Surface

```csharp
public abstract class BaseQueryDbContext : DbContext
{
    protected BaseQueryDbContext(DbContextOptions options);
}
```

## Rules it enforces

1. **Read-only.** All four `SaveChanges` / `SaveChangesAsync` overloads throw `InvalidOperationException`. This
   is a fail-fast guard: any attempt to persist through the query context is a programming error ‚Äî writes must
   go through the command side (`BaseCommandDbContext`).
2. **No change tracking.** The constructor sets:
   - `ChangeTracker.QueryTrackingBehavior = QueryTrackingBehavior.NoTracking` ‚Äî queries return untracked
     instances, avoiding identity-map and snapshot overhead.
   - `ChangeTracker.AutoDetectChangesEnabled = false` ‚Äî no change detection passes.

   Together these reduce allocations and CPU for read-heavy workloads.

## Usage

Derive a concrete read context and expose query models as `DbSet<T>`:

```csharp
public sealed class ShopQueryDbContext(DbContextOptions<ShopQueryDbContext> options)
    : BaseQueryDbContext(options)
{
    public DbSet<OrderReadModel> Orders => Set<OrderReadModel>();

    protected override void OnModelCreating(ModelBuilder modelBuilder)
    {
        modelBuilder.Entity<OrderReadModel>().ToTable("Orders");
    }
}
```

```csharp
services.AddDbContext<ShopQueryDbContext>(options =>
    options.UseSqlServer(readConnectionString));
```

Because tracking is off, there is no need to call `AsNoTracking()` on individual queries. If a specific query
must be tracked (rare on the read side), opt in explicitly with `AsTracking()`.

## Why a separate read context

Splitting reads from writes lets the query side use plain, query-friendly models
([`QueryAggregateRoot<TKey>`](2.QueryAggregateRoot.md)) with navigation properties across aggregates, without
dragging in the command-side invariants, interceptors, or transactions.

See also: [QueryAggregateRoot](2.QueryAggregateRoot.md).
