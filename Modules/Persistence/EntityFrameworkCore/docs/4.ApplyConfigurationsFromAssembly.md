# 4. ApplyConfigurationsFromAssembly

`ModelBuilder` extension that applies only the `IEntityTypeConfiguration<TEntity>` implementations in an
assembly that also implement the chosen marker interface.

- **Package:** `RaccoonLand.Modules.Persistence.EntityFrameworkCore`
- **Namespace:** `RaccoonLand.Modules.Persistence.EntityFrameworkCore.Extensions`

## Signature

```csharp
public static class ModelBuilderConfigurationExtensions
{
    public static void ApplyConfigurationsFromAssembly<TMarker>(
        this ModelBuilder modelBuilder,
        Assembly assembly)
        where TMarker : class;
}
```

| Parameter | Purpose |
| --- | --- |
| `TMarker` | `ICommandEntityConfiguration` or `IQueryEntityConfiguration` (or any future marker you define). |
| `assembly` | Assembly to scan — typically `typeof(MyDbContext).Assembly`. |

## Discovery rules

A type is applied when **all** of the following hold:

| Rule | Detail |
| --- | --- |
| Public concrete class | Abstract types, interfaces, and non-public types are skipped. |
| Implements `TMarker` | Must be assignable to the marker passed as the generic argument. |
| Exactly one `IEntityTypeConfiguration<TEntity>` | Zero closed generics (marker only) or more than one → `InvalidOperationException`. Split multi-entity mappings into separate classes; do not leave orphan markers. |
| Exclusive CQRS marker | Implementing both `ICommandEntityConfiguration` and `IQueryEntityConfiguration` → `InvalidOperationException`. |
| Public parameterless constructor | Instantiated with `Activator.CreateInstance`. Constructor injection / DI is **not** supported. |

Each matching type is applied through the same `ModelBuilder.ApplyConfiguration<TEntity>` path EF Core uses
internally. Failures inside `Configure` are rethrown as `InvalidOperationException` that names the
configuration type and entity type (inner exception preserved).

## Multiple assemblies

The API scans **one** assembly per call. If configurations live in several assemblies, call the extension
once per assembly from `OnModelCreating`:

```csharp
modelBuilder.ApplyConfigurationsFromAssembly<ICommandEntityConfiguration>(typeof(ShopCommandDbContext).Assembly);
modelBuilder.ApplyConfigurationsFromAssembly<ICommandEntityConfiguration>(typeof(PeopleCommandConfigs).Assembly);
```

## Command DbContext

```csharp
using Microsoft.EntityFrameworkCore;
using RaccoonLand.Modules.Persistence.EntityFrameworkCore.Configuration;
using RaccoonLand.Modules.Persistence.EntityFrameworkCore.Extensions;
using RaccoonLand.Modules.Persistence.SqlServer.Commands;

public sealed class ShopCommandDbContext(DbContextOptions<ShopCommandDbContext> options)
    : BaseCommandDbContext(options)
{
    public DbSet<Product> Products => Set<Product>();

    protected override void OnModelCreating(ModelBuilder modelBuilder)
    {
        base.OnModelCreating(modelBuilder);
        modelBuilder.ApplyConfigurationsFromAssembly<ICommandEntityConfiguration>(
            typeof(ShopCommandDbContext).Assembly);
    }
}
```

## Query DbContext

```csharp
using Microsoft.EntityFrameworkCore;
using RaccoonLand.Modules.Persistence.EntityFrameworkCore.Configuration;
using RaccoonLand.Modules.Persistence.EntityFrameworkCore.Extensions;
using RaccoonLand.Modules.Persistence.SqlServer.Queries;

public sealed class ShopQueryDbContext(DbContextOptions<ShopQueryDbContext> options)
    : BaseQueryDbContext(options)
{
    public DbSet<ProductListItem> Products => Set<ProductListItem>();

    protected override void OnModelCreating(ModelBuilder modelBuilder)
    {
        base.OnModelCreating(modelBuilder);
        modelBuilder.ApplyConfigurationsFromAssembly<IQueryEntityConfiguration>(
            typeof(ShopQueryDbContext).Assembly);
    }
}
```

## Comparison with EF Core built-in

| | `ModelBuilder.ApplyConfigurationsFromAssembly(assembly)` | RaccoonLand `ApplyConfigurationsFromAssembly<TMarker>(assembly)` |
| --- | --- | --- |
| Filter | All `IEntityTypeConfiguration<>` in the assembly | Only configurations that also implement `TMarker` |
| CQRS split | Manual predicates or separate assemblies | Marker interfaces on the configuration class |
| Provider | EF Core built-in | This package (`Microsoft.EntityFrameworkCore` only) |

## Registration

No DI registration is required. Reference the package from the project that contains your `DbContext` and
configuration classes, then call the extension from `OnModelCreating`.
