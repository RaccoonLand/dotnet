# 6. Message resolution (SqlServerMessageLocalization)

`SqlServerMessageLocalization` is the `IMessageLocalization` implementation. It resolves and formats messages
**entirely from the in‑memory store** — resolution never touches the database. Misses are recorded for the
background worker to persist later.

- **Package:** `RaccoonLand.Modules.MessageLocalization.SQLServer`
- **Namespace:** `RaccoonLand.Modules.MessageLocalization.SQLServer`
- **Visibility:** `internal` (consumed only through `IMessageLocalization`).

## Resolution pipeline

For each lookup the localizer:

1. **Determines the culture** in this order:
   explicit culture argument → `ICurrentCultureProvider.GetCurrentCulture()` → configured `DefaultCulture`.
2. **Resolves the template** for that culture via the culture fallback chain (below).
3. **Resolves each parameter** following the [parameter convention](../../Abstraction/docs/1.IMessageLocalization.md#parameter-convention):
   `RawValue` → literal; `string` → recursively resolved as a key; anything else → literal.
4. **Formats** the template with `string.Format(culture, template, args)`.

```csharp
public string this[string messageTemplate, params object?[] parameters]
    => Format(ResolveCulture(explicitCulture: null), messageTemplate, parameters);

public string this[CultureInfo culture, string messageTemplate, params object?[] parameters]
    => Format(ResolveCulture(culture), messageTemplate, parameters);
```

## Culture fallback chain

A key is looked up against the requested culture, then each of its parents (e.g. `en-US` → `en`), and finally
the configured default culture if it differs:

```
en-US  →  en  →  (DefaultCulture, e.g. en-US)
```

The first culture in the chain that contains the key wins.

## Graceful degradation

The localizer is designed to **never throw** on the request path:

| Situation | Behavior |
| --- | --- |
| Key missing in every culture | Returns the **key itself** (a stable, meaningful string) and, if `AutoInsertMissingKeys` is on, reports it to the [`MissingKeyTracker`](4.Storage.md). |
| Template/arguments mismatch (`FormatException`) | Returns the **unformatted template** instead of throwing. |
| Invalid `DefaultCulture` name | Falls back to `CultureInfo.InvariantCulture`. |

## Example

```csharp
// Current culture comes from ICurrentCultureProvider (e.g. Accept-Language), else DefaultCulture.
string message = localizer[ValidationErrors.NOT_FOUND, Translation.COMPANY];

// "COMPANY" is itself resolved as a key in the same culture before being inserted.
```

## Design notes

- Reads are lock‑free: they hit an atomically swapped snapshot (see [Storage](4.Storage.md)).
- Missing‑key reporting is fire‑and‑forget; nothing on the request path waits on the database.
