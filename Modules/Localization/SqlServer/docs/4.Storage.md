# 7. Storage (in-memory snapshot)

The storage types keep all translations in memory so the request path never hits the database. They live in
the `RaccoonLand.Modules.MessageLocalization.SQLServer.Storage` namespace and are `internal`.

## LocalizationEntry

A single localization row loaded from the database: a value for a `(Culture, Key)` pair.

```csharp
internal sealed record LocalizationEntry(string Culture, string Key, string Value);
```

## MessageLocalizationStore

A thread‑safe, **lock‑free** snapshot of every translation for the current application. The full set is loaded
at startup and **atomically swapped** on each periodic refresh, so reads are always consistent without
locking.

```csharp
internal sealed class MessageLocalizationStore
{
    public int CultureCount { get; }
    public void Replace(IEnumerable<LocalizationEntry> entries);
    public bool TryGet(string culture, string key, out string value);
}
```

| Member | Description |
| --- | --- |
| `Replace(entries)` | Builds a fresh nested dictionary and atomically replaces the current snapshot. |
| `TryGet(culture, key, out value)` | Exact lookup for a `(culture, key)` pair. Culture matching is case‑insensitive; key matching is case‑sensitive. |
| `CultureCount` | Number of cultures in the current snapshot. |

Internals:

- Outer dictionary keyed by **culture name** with `StringComparer.OrdinalIgnoreCase`.
- Inner dictionary keyed by **message/parameter key** with `StringComparer.Ordinal`.
- The snapshot field is `volatile`; `Replace` swaps the whole reference so in‑flight reads either see the old
  or the new snapshot, never a partial one.

## MissingKeyTracker

Collects keys that were requested but missing, so reads never block on the database. The localizer just
records the miss; the [refresh worker](6.RefreshService.md) drains and persists them on its next cycle.

```csharp
internal sealed record MissingKey(string Culture, string Key);

internal sealed class MissingKeyTracker
{
    public void Report(string culture, string key);
    public IReadOnlyCollection<MissingKey> Drain();
    public void Requeue(IEnumerable<MissingKey> keys);
}
```

| Member | Description |
| --- | --- |
| `Report(culture, key)` | Records a missing key, deduplicated until the next drain. |
| `Drain()` | Under a shared lock, copies and clears the pending set. |
| `Requeue(keys)` | Puts keys back into the pending set (used when persist fails after a drain). Input is materialized before the lock. |

Internals: a `HashSet<MissingKey>` guarded by a single lock shared by `Report`, `Drain`, and `Requeue`, so a
report cannot land on a drained instance.

## Design notes

- Splitting reads (`Store`) from miss‑tracking (`Tracker`) keeps the hot path allocation‑light and entirely
  in memory.
- Because `Replace` swaps a complete snapshot, an admin edit or a newly persisted placeholder becomes visible
  all at once on the next refresh — there is never a half‑updated view.
