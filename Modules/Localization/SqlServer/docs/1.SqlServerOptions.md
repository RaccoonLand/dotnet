# 4. MessageLocalizationSqlServerOptions

Configuration for the SQL Server implementation of [`IMessageLocalization`](../../Abstraction/docs/1.IMessageLocalization.md).
Typically bound from a configuration section named `MessageLocalization`.

- **Package:** `RaccoonLand.Modules.MessageLocalization.SQLServer`
- **Namespace:** `RaccoonLand.Modules.MessageLocalization.SQLServer.Configuration`

## Surface

| Property | Type | Default | Description |
| --- | --- | --- | --- |
| `SectionName` (const) | `string` | `"MessageLocalization"` | Default configuration section name. |
| `ConnectionString` | `string` | `""` (**required**) | Connection string to the shared localization database. |
| `ServiceName` | `string` | `""` (**required**) | Current microservice — a row in `Localization.Services`. |
| `ApplicationName` | `string` | `""` (**required**) | Current application within the service — a row in `Localization.Applications`. |
| `DefaultCulture` | `string` | `"en-US"` | Culture used when neither the caller nor the `ICurrentCultureProvider` specifies one. |
| `SchemaName` | `string` | `"Localization"` | Schema containing the localization tables. |
| `ServicesTableName` | `string` | `"Services"` | Services table name. |
| `ApplicationsTableName` | `string` | `"Applications"` | Applications table name. |
| `MessageLocalizationsTableName` | `string` | `"MessageLocalizations"` | Messages table name. |
| `AutoCreateTables` | `bool` | `false` | Create schema/tables on startup if missing (needs DDL permissions). |
| `RefreshInterval` | `TimeSpan` | `00:05:00` | How often the in‑memory snapshot is reloaded. `<= 0` disables periodic refresh. |
| `AutoRegisterApplication` | `bool` | `true` | Create the service/application rows on startup if missing. |
| `AutoInsertMissingKeys` | `bool` | `true` | Persist missing keys as placeholders for later admin translation. |

The three required properties are enforced by an options validation predicate registered by the module (see
[Registration](2.Registration.md)); validation runs on first resolve.

## Example (`appsettings.json`)

```json
{
  "MessageLocalization": {
    "ConnectionString": "Server=.;Database=SharedLocalization;Trusted_Connection=True;TrustServerCertificate=True",
    "ServiceName": "Ordering",
    "ApplicationName": "Ordering.Api",
    "DefaultCulture": "en-US",
    "RefreshInterval": "00:05:00",
    "AutoCreateTables": false,
    "AutoRegisterApplication": true,
    "AutoInsertMissingKeys": true
  }
}
```

## Operational guidance

- **Production:** prefer `AutoCreateTables = false` and apply the
  [schema script](5.Repository.md#schema-script) with a privileged account, leaving the runtime account with
  only DML rights.
- **Local/dev:** `AutoCreateTables = true` lets the app bootstrap its own schema.
- The table/schema name overrides let multiple environments share a database while customizing object names.
