# 5. Registration (DI)

`MessageLocalizationSqlServerServiceCollectionExtensions` wires the SQL Server implementation into the
dependency‑injection container.

- **Package:** `RaccoonLand.Modules.MessageLocalization.SQLServer`
- **Namespace:** `Microsoft.Extensions.DependencyInjection` (so the extension is discoverable wherever you
  already use DI)

## Surface

```csharp
public static IServiceCollection AddRaccoonLandMessageLocalizationSqlServer(
    this IServiceCollection services,
    IConfiguration configuration,
    string sectionName = MessageLocalizationSqlServerOptions.SectionName);

public static IServiceCollection AddRaccoonLandMessageLocalizationSqlServer(
    this IServiceCollection services,
    Action<MessageLocalizationSqlServerOptions> configure);
```

| Overload | Use when |
| --- | --- |
| `(IConfiguration, sectionName)` | Binding options from a configuration section (defaults to `MessageLocalization`). |
| `(Action<Options>)` | Configuring options in code. |

Both overloads register an options validation predicate that requires `ConnectionString`, `ServiceName` and
`ApplicationName` to be non‑empty.

## What gets registered

`AddCore` registers everything as singletons (using `TryAdd*`, so your own registrations win) plus the
background refresh worker:

| Service | Lifetime | Notes |
| --- | --- | --- |
| `MessageLocalizationStore` | Singleton | In‑memory snapshot ([Storage](4.Storage.md)). |
| `MissingKeyTracker` | Singleton | Records cache misses ([Storage](4.Storage.md)). |
| `MessageLocalizationRepository` | Singleton | Dapper data access ([Repository](5.Repository.md)). |
| `ICurrentCultureProvider` → `NullCurrentCultureProvider` | Singleton (`TryAdd`) | Default; override with your own. |
| `IMessageLocalization` → `SqlServerMessageLocalization` | Singleton (`TryAdd`) | The localizer ([Resolution](3.MessageResolution.md)). |
| `MessageLocalizationRefreshService` | Hosted service | Initial load + periodic refresh ([Refresh service](6.RefreshService.md)). |

Because the culture provider and the localizer use `TryAddSingleton`, registering your own implementation
**before** calling `AddRaccoonLandMessageLocalizationSqlServer` overrides the defaults.

## Example

```csharp
// Bind from configuration.
builder.Services.AddRaccoonLandMessageLocalizationSqlServer(builder.Configuration);

// Or configure in code.
builder.Services.AddRaccoonLandMessageLocalizationSqlServer(options =>
{
    options.ConnectionString = "Server=.;Database=SharedLocalization;Trusted_Connection=True;TrustServerCertificate=True";
    options.ServiceName = "Ordering";
    options.ApplicationName = "Ordering.Api";
    options.AutoCreateTables = true;
});

// Provide the request culture (overrides the Null default).
builder.Services.AddHttpContextAccessor();
builder.Services.AddSingleton<ICurrentCultureProvider, HeaderCultureProvider>();
```
