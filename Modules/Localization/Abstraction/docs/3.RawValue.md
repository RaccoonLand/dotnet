# 3. RawValue

`RawValue` is a small `readonly struct` that wraps a **string** which must be inserted into a message
**exactly as provided**, bypassing the localizer's key‑resolution step.

- **Package:** `RaccoonLand.Modules.MessageLocalization.Abstraction`
- **Namespace:** `RaccoonLand.Modules.MessageLocalization.Abstraction`

## Why it exists

The default [parameter convention](1.IMessageLocalization.md#parameter-convention) is: a `string` parameter is
a localization **key** and is resolved before insertion, while any non‑string value is a **literal**. That
convention is convenient for nested keys, but it gets in the way when you genuinely want to insert a raw
string — for example a person's name — that must **not** be looked up as a key. `RawValue` marks such a
string as a literal.

Only strings need this escape hatch. Numbers, dates, and other non‑string values are already treated as
literals and should be passed directly (not wrapped in `Raw`).

## Surface

```csharp
public readonly struct RawValue
{
    public string? Value { get; }
    public static RawValue Raw(string? value);
    public override string ToString();
}
```

| Member | Description |
| --- | --- |
| `Raw(string?)` | Factory that wraps a string as a literal. The only way to construct a `RawValue`. `null` is allowed and means an empty inserted literal. |
| `Value` | The wrapped literal string (`null` → empty text). |
| `ToString()` | Returns `Value` or `string.Empty` when `Value` is `null`. |

## Example

```csharp
using static RaccoonLand.Modules.MessageLocalization.Abstraction.RawValue;

// Without Raw: "احمد" would be treated as a localization key and resolved (wrong).
// With Raw: it is inserted verbatim into the resolved template.
var welcome = localizer.Get(Messages.WELCOME, Raw("احمد"));

// Numbers/dates are already literals — do not wrap them:
var age = localizer.Get(Messages.AGE_BETWEEN, 10, 60);
```

## Null

`Raw(null)` is valid. Implementations insert `string.Empty`, same as a bare `null` element in the
`params` array. Prefer omitting the argument when you do not need a placeholder.

## Design notes

- The constructor is private; use `Raw(...)` so intent is explicit at the call site.
- Implementations detect `RawValue` first in their parameter‑resolution switch, before the `string` branch,
  so a wrapped string is always treated as a literal.
