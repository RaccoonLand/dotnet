# 3. RawValue

`RawValue` is a small `readonly struct` that wraps a parameter value which must be inserted into a message
**exactly as provided**, bypassing the localizer's key‑resolution step.

- **Package:** `RaccoonLand.Modules.MessageLocalization.Abstraction`
- **Namespace:** `RaccoonLand.Modules.MessageLocalization.Abstraction`

## Why it exists

The default [parameter convention](1.IMessageLocalization.md#parameter-convention) is: a `string` parameter is
a localization **key** and is resolved before insertion, while any non‑string value is a **literal**. That
convention is convenient for nested keys, but it gets in the way when you genuinely want to insert a raw
string — for example a person's name — that must **not** be looked up as a key. `RawValue` marks such a value
as a literal.

## Surface

```csharp
public readonly struct RawValue
{
    public object? Value { get; }
    public static RawValue Raw(object? value);
    public override string? ToString();
}
```

| Member | Description |
| --- | --- |
| `Raw(object?)` | Factory that wraps `value` as a literal. The only way to construct a `RawValue`. |
| `Value` | The wrapped literal value the implementation inserts as‑is. |
| `ToString()` | Returns `Value?.ToString()`. |

## Example

```csharp
using static RaccoonLand.Modules.MessageLocalization.Abstraction.RawValue;

// Without Raw: "احمد" would be treated as a localization key and resolved (wrong).
// With Raw: it is inserted verbatim into the resolved template.
var welcome = localizer[Messages.WELCOME, Raw("احمد")];
```

## Design notes

- The constructor is private; use `Raw(...)` so intent is explicit at the call site.
- Implementations detect `RawValue` first in their parameter‑resolution switch, before the `string` branch,
  so a wrapped string is always treated as a literal.
