# 1. IMessageLocalization

`IMessageLocalization` is the central contract of the Localization module. It resolves and formats
user‑facing messages at runtime so that **no message is hard‑coded** in the application — every template
(and, where applicable, the values inserted into it) can be edited at runtime by an administrator.

- **Package:** `RaccoonLand.Modules.MessageLocalization.Abstraction`
- **Namespace:** `RaccoonLand.Modules.MessageLocalization.Abstraction`
- **Analogy:** like `IDistributedCache`, this package ships **only the contract**; the actual resolution
  (database, cache, resource files, …) comes from a separate implementation package such as the
  SQL Server module.

## Surface

```csharp
public interface IMessageLocalization
{
    string this[string messageTemplate, params object?[] parameters] { get; }
    string this[CultureInfo culture, string messageTemplate, params object?[] parameters] { get; }
}
```

| Member | Description |
| --- | --- |
| `this[string, params object?[]]` | Resolves the template for the **current culture** and formats it with the optional parameters. |
| `this[CultureInfo, string, params object?[]]` | Resolves the template for an **explicit culture** and formats it with the optional parameters. |

The "current culture" is provided by an [`ICurrentCultureProvider`](2.ICurrentCultureProvider.md); when the
provider returns `null` the implementation falls back to its configured default culture.

## Parameter convention

Both indexers accept a `params object?[]` of values that are inserted into the resolved template via
`string.Format`. The convention for each value is:

| Value type | Treated as |
| --- | --- |
| `string` | A **localization key** — resolved through the localizer before insertion. |
| Non‑string (`int`, `DateTime`, …) | A **literal** — inserted as‑is. |
| [`RawValue`](3.RawValue.md) | A **literal string** — inserted verbatim, bypassing key resolution. |

This makes nested localization natural: a message template can reference other keys as parameters, and they
are all resolved in the same culture.

## Example

```csharp
using static RaccoonLand.Modules.MessageLocalization.Abstraction.RawValue;

// "COMPANY" is resolved as a key; 10 and 60 are literals.
var notFound = localizer[ValidationErrors.NOT_FOUND, Translation.COMPANY];
var ageRange = localizer[ValidationErrors.AGE_BETWEEN, 10, 60];

// Insert a literal name that must NOT be looked up as a key.
var welcome = localizer[Messages.WELCOME, Raw("Raccoon")];

// Resolve for an explicit culture.
var en = localizer[CultureInfo.GetCultureInfo("en-US"), ValidationErrors.NOT_FOUND, Translation.COMPANY];
```

## Design notes

- The contract is intentionally tiny and dependency‑free so any layer (domain, application, API) can depend
  on it without pulling in a storage technology.
- Resolution is expected to be cheap and synchronous; implementations should serve reads from memory rather
  than hitting a database on the request path (see [Message resolution](../../SqlServer/docs/3.MessageResolution.md)).
