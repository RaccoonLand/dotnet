# 2. ICurrentCultureProvider

`ICurrentCultureProvider` supplies the culture for the current operation when the caller does **not** pass
one explicitly to [`IMessageLocalization`](1.IMessageLocalization.md). It is the seam that lets the localizer
stay ignorant of *how* a request's culture is determined.

- **Package:** `RaccoonLand.Modules.MessageLocalization.Abstraction`
- **Namespace:** `RaccoonLand.Modules.MessageLocalization.Abstraction`

## Surface

```csharp
public interface ICurrentCultureProvider
{
    CultureInfo? GetCurrentCulture();
}
```

| Member | Description |
| --- | --- |
| `GetCurrentCulture()` | Returns the culture for the current operation, or `null` to use the configured default culture. |

## NullCurrentCultureProvider

A default implementation that never supplies a culture, so the implementation always falls back to its
configured default. Implementation packages register it as the default; consumers can replace it with their
own provider.

```csharp
public sealed class NullCurrentCultureProvider : ICurrentCultureProvider
{
    public static readonly NullCurrentCultureProvider Instance = new();
    public CultureInfo? GetCurrentCulture() => null;
}
```

The SQL Server module registers `NullCurrentCultureProvider` with `TryAddSingleton`, so simply registering
your own `ICurrentCultureProvider` **before** calling the module's `Add…` method (or overriding afterwards)
wins.

## Typical API implementation

In a web API the culture usually comes from a request header (for example `Accept-Language` or a custom
header) read from the current `HttpContext`:

```csharp
internal sealed class HeaderCultureProvider(IHttpContextAccessor accessor) : ICurrentCultureProvider
{
    public CultureInfo? GetCurrentCulture()
    {
        var header = accessor.HttpContext?.Request.Headers.AcceptLanguage.ToString();
        if (string.IsNullOrWhiteSpace(header))
        {
            return null; // fall back to the configured default
        }

        try { return CultureInfo.GetCultureInfo(header.Split(',')[0].Trim()); }
        catch (CultureNotFoundException) { return null; }
    }
}

// Registration (override the module default):
services.AddHttpContextAccessor();
services.AddSingleton<ICurrentCultureProvider, HeaderCultureProvider>();
```

## Design notes

- Returning `null` is a first‑class, expected outcome — it means "I have no opinion, use the default".
- The provider is resolved as a singleton; if it depends on per‑request state it must do so through an
  accessor (such as `IHttpContextAccessor`) rather than capturing it.
