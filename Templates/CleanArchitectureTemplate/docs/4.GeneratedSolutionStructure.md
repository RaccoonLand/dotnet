# 4. Generated Solution Structure

## Top-level layout

```text
{ProjectName}/
├── global.json
├── nuget.config
├── Directory.Build.props
├── Directory.Packages.props
├── {ProjectName}.slnx
└── src/
    ├── Domain/
    ├── Application/
    ├── Infrastructure/Persistence/SqlServer/
    │   ├── Commands/
    │   └── Queries/
    └── Hosting/API/
```

## Domain project

```text
src/Domain/
├── People/
│   ├── Domain/
│   └── Shared/
└── Shared/
    └── Localizations/
```

## Application project

```text
src/Application/
├── Abstractions/Persistence/
│   ├── ICommandDbContext.cs
│   └── IQueryDbContext.cs
└── People/
    ├── Commands/
    ├── Queries/
    └── Queries/ReadModels/
```

## Infrastructure projects

```text
src/Infrastructure/Persistence/SqlServer/
├── Commands/
│   ├── {CommandDbContextName}.cs
│   ├── DependencyInjection/
│   └── People/PersonConfiguration.cs
└── Queries/
    ├── {QueryDbContextName}.cs
    ├── DependencyInjection/
    └── People/PersonReadModelConfiguration.cs
```

## Host project

```text
src/Hosting/API/
├── Controllers/People/
├── Hosting.cs
├── Program.cs
└── appsettings.json
```

## DbContext placement

| DbContext | File | Responsibility |
|-----------|------|----------------|
| `{ProjectName}CommandDbContext` or custom name | `Infrastructure/Persistence/SqlServer/Commands/{Name}.cs` | Writes, migrations, outbox interceptors |
| `{ProjectName}QueryDbContext` or custom name | `Infrastructure/Persistence/SqlServer/Queries/{Name}.cs` | Read-only queries via `IQueryDbContext` |

## People capability

| Layer | Key types |
|-------|-----------|
| Domain | `Person`, `FirstName`, `LastName` |
| Application commands | `CreatePersonCommand`, `CreatePersonEndpoint` |
| Application queries | `GetPersonByIdQuery`, `SearchPeopleQuery`, `PersonReadModel` |
| Infrastructure | `PersonConfiguration`, `PersonReadModelConfiguration` |

## HTTP routes

| Route | Request |
|-------|---------|
| `POST /api/people` | `CreatePersonCommand` |
| `GET /api/people/{id}` | `GetPersonByIdQuery` |
| `GET /api/people/Search` | `SearchPeopleQuery` |

## First run checklist

1. Update SQL connection strings in `appsettings.json`.
2. Create the initial migration (see [6. Create Initial Migration](6.CreateInitialMigration.md)).
3. Apply the migration.
4. Run the host:

```bash
dotnet run --project src/Hosting/API/{ProjectName}.Hosting.API.csproj
```

5. Open Scalar:

```text
/scalar/v1
```
