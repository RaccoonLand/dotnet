# 5. Packaging and Publishing

## Packaging project

The template is packed from:

```text
Templates/CleanArchitectureTemplate/RaccoonLand.Templates.CleanArchitecture.csproj
```

Important properties:

| Property | Value |
|----------|-------|
| `PackageType` | `Template` |
| `PackageId` | `RaccoonLand.Templates.CleanArchitecture` |
| `Version` | `1.0.0` (from repository `Directory.Build.props`) |

## Build the nupkg

From the repository root:

```bash
dotnet pack Templates/CleanArchitectureTemplate/RaccoonLand.Templates.CleanArchitecture.csproj -c Release -o ./artifacts/templates
```

Expected output:

```text
artifacts/templates/RaccoonLand.Templates.CleanArchitecture.1.0.0.nupkg
```

## Push to local BaGet feed

```bash
dotnet nuget push ./artifacts/templates/RaccoonLand.Templates.CleanArchitecture.1.0.0.nupkg -s https://localhost:5001/v3/index.json
```

No API key is required for the local feed.

## Install from the local feed

If the feed is configured in `nuget.config`:

```bash
dotnet new install RaccoonLand.Templates.CleanArchitecture
```

## Validation workflow

1. Pack the template.
2. Push to `https://localhost:5001/v3/index.json`.
3. Install the template.
4. Generate a sample project with custom names.
5. Build the generated solution.
6. Create the initial EF migration.
7. Optionally apply the migration to a local database.

Example validation:

```bash
dotnet new raccoonland-clean-architecture `
  -n Validation.App `
  -o ./tmp/Validation.App `
  --CommandDbContextName ValidationCommandDbContext `
  --QueryDbContextName ValidationQueryDbContext

dotnet build ./tmp/Validation.App/Validation.App.slnx

dotnet ef migrations add Init `
  --project ./tmp/Validation.App/src/Infrastructure/Persistence/SqlServer/Commands/CleanArchitectureTemplate.Persistence.SqlServer.Commands.csproj `
  --startup-project ./tmp/Validation.App/src/Hosting/API/CleanArchitectureTemplate.Hosting.API.csproj `
  --context ValidationCommandDbContext `
  --output-dir Migrations
```

## Notes

- Repository root keeps only packaging + docs; shippable files live under `content/`.
- Template content excludes `bin/`, `obj/`, `.vs/`, and `.template.config` from generated output.
- The generated project does not contain a prebuilt EF Core migration; consumers create the first migration after project creation.
- Generated projects include their own `Directory.Packages.props` so they can be opened outside the monorepo.
- RaccoonLand framework packages are still resolved from the configured local feed through `nuget.config`.

## Local source build

To validate template source before packing:

```bash
dotnet build Templates/CleanArchitectureTemplate/content/CleanArchitectureTemplate.slnx
```
