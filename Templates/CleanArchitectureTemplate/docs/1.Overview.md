# 1. Overview

`RaccoonLand.Templates.CleanArchitecture` is a `dotnet new` project template that scaffolds a layered clean architecture RaccoonLand application.

## What it generates

| Area | Content |
|------|---------|
| Domain project | `{ProjectName}.Domain` |
| Application project | `{ProjectName}.Application` with `ICommandDbContext` and `IQueryDbContext` |
| Infrastructure projects | `{ProjectName}.Persistence.SqlServer.Commands` and `{ProjectName}.Persistence.SqlServer.Queries` |
| Host project | `{ProjectName}.Hosting.API` |
| Capability | `People` with `FirstName` and `LastName` |
| Endpoints | `CreatePerson`, `GetPersonById`, `SearchPeople` |
| Persistence | SQL Server command/query DbContexts ready for you to create the initial EF Core migration |
| Infrastructure | OpenAPI, Scalar, Serilog, observability, validation, request caching, exception handling |

## Design goals

1. Keep the generated solution small and easy to extend.
2. Preserve Clean Architecture layer boundaries (Domain → Application → Infrastructure → Hosting).
3. Expose persistence through `ICommandDbContext` and `IQueryDbContext` abstractions in Application.
4. Allow project and DbContext names to be customized at creation time.
5. Ship as a standalone solution with its own `global.json`, `Directory.Packages.props`, and `nuget.config`.

## Template identity

| Property | Value |
|----------|-------|
| Package ID | `RaccoonLand.Templates.CleanArchitecture` |
| Template short name | `raccoonland-clean-architecture` |
| Source name token | `CleanArchitectureTemplate` |
| Command DbContext token | `TemplateCommandDbContext` |
| Query DbContext token | `TemplateQueryDbContext` |

## Repository layout

The template source in the monorepo is split into packaging metadata and shippable content:

```text
CleanArchitectureTemplate/
├── RaccoonLand.Templates.CleanArchitecture.csproj   # NuGet packaging only
├── docs/                                            # maintainer documentation
└── content/                                         # everything shipped by dotnet new
    ├── .template.config/template.json
    ├── CleanArchitectureTemplate.slnx
    ├── global.json
    ├── nuget.config
    ├── Directory.Build.props
    ├── Directory.Packages.props
    └── src/
        ├── Domain/
        ├── Application/
        ├── Infrastructure/Persistence/SqlServer/
        └── Hosting/API/
```

Only `content/` is packed into the template NuGet. The generated project layout matches `content/` (without `.template.config`).
