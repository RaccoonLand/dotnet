# 6. Create Initial Migration

The template intentionally does **not** ship with a prebuilt EF Core migration.  
After generating a project, create the first migration for your own project name and DbContext names.

## Prerequisites

- The project was created successfully with `dotnet new raccoonland-capability`
- SQL connection strings are configured in `src/{ProjectName}.Hosting.API/appsettings.json`
- NuGet restore succeeds

## Default command

If you did not override the command DbContext name:

```bash
dotnet ef migrations add Init --project src/{ProjectName}/{ProjectName}.csproj --startup-project src/{ProjectName}.Hosting.API/{ProjectName}.Hosting.API.csproj --context {SafeProjectName}CommandDbContext --output-dir Shared/Persistence/Commands/Migrations
```

## Custom DbContext example

If you created the project like this:

```bash
dotnet new raccoonland-capability -n Acme.HumanResources -o ./Acme.HumanResources --CommandDbContextName AcmeHumanResourcesCommandDbContext --QueryDbContextName AcmeHumanResourcesQueryDbContext
```

Run:

```bash
dotnet ef migrations add Init --project src/Acme.HumanResources/Acme.HumanResources.csproj --startup-project src/Acme.HumanResources.Hosting.API/Acme.HumanResources.Hosting.API.csproj --context AcmeHumanResourcesCommandDbContext --output-dir Shared/Persistence/Commands/Migrations
```

## Apply the migration

```bash
dotnet ef database update --project src/{ProjectName}/{ProjectName}.csproj --startup-project src/{ProjectName}.Hosting.API/{ProjectName}.Hosting.API.csproj
```

## Expected result

The command creates files under:

```text
src/{ProjectName}/Shared/Persistence/Commands/Migrations/
```

Usually:

- `yyyyMMddHHmmss_Init.cs`
- `yyyyMMddHHmmss_Init.Designer.cs`
- `{CommandDbContextName}ModelSnapshot.cs`

## Notes

- Use the **command** DbContext, not the query DbContext.
- The query DbContext is read-only and is not intended for migrations.
- If multiple DbContexts exist in the future, always pass `--context`.
